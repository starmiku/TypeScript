# 代码组织

代码组织指的是如何将代码拆分、分组和管理，使其结构清晰、易于维护和复用。

## 模块

模块系统是现代TypeScript开发的基础，通过模块系统，可以将代码组织成可重用的单元，并且可以控制代码的可见性和访问权限。

TypeScript模块使用`export`关键字来导出模块中的内容，使用`import`关键字来导入模块中的内容。

### 模块导出

使用`export`关键字可以将模块中的类、接口、函数、变量等导出，使其可以在其他模块中使用。

```ts
// 导出类
export class MyClass {
    // 类的定义
}

// 导出接口
export interface MyInterface {
    // 接口的定义
}

// 导出函数
export function myFunction() {
    // 函数的定义
}

// 导出变量
export const myVariable = 42;

// 导出多个内容
export { MyClass, MyInterface, myFunction, myVariable };

// 重命名导出
export { MyClass as MyRenamedClass };
```

接口和类型在编译时会被完全擦除，不产生实际JavaScript代码，但它们仍然需要用`export`导出才能被其他模块引用。

### 模块导入

使用`import`关键字可以将其他模块中的内容导入到当前模块中。

```ts
// 命名导入：从模块中导入指定的内容
import { name, age, greet } from "./user";

// 默认导入：导入模块的默认导出
import User from "./user";

// 全部导入：将模块所有导出放入一个对象
import * as UserModule from "./user";

// 重命名导入：避免命名冲突
import { greet as sayHello } from "./user";

// 使用导入的内容
console.log(greet("World"));
console.log(sayHello("TypeScript"));
```

导入路径可以是相对路径（如`./user`）或通过 tsconfig.json 配置的路径别名（如`@/utils`）。相对路径是相对于当前文件的路径，路径别名则是相对于 `baseUrl` 配置的项目根目录。

### 默认导出

每个模块可以有一个默认导出，默认导出使用`export default`关键字定义，导入时不需要使用花括号。

Math.ts

```ts
// 默认导出：一个模块只能有一个默认导出
export default function add(a: number, b: number): number {
    return a + b;
}

// 可以和其他导出混合使用
export function multiply(a: number, b: number): number {
    return a * b;
}
```

Main.ts

```ts
// 导入默认导出
import add from "./Math";

// 导入其他导出
import { multiply } from "./Math";

// 使用导入的内容
console.log(add(2, 3)); // 输出：5
console.log(multiply(2, 3)); // 输出：6
```

### 重新导出

重新导出用于聚合多个模块的内容，或将一个模块的导出暴露给另一个模块。

```ts
// 从其他模块重新导出指定内容
export { name, age } from "./user";

// 重新导出默认导出（需要重命名）
export { default as User } from "./user";

// 重新导出所有内容
export * from "./math";
```

重新导出的作用是转发模块的导出，使其他模块无需知道原始模块路径就能使用这些内容；当前模块如果既想转发又想自己使用，则需要先用 `import` 导入，再 `export` 转发。

#### 模块解析策略

TypeScript提供了多种模块解析策略，用于查找导入的模块，可以在tsconfig.json文件中配置。

```json
{
    "compilerOptions": {
        // Node 解析策略
        // 遵循 Node.js 的模块解析规则
        "moduleResolution": "node16",
        // 可选：
        // bundler（现代前端项目首选）
        // Node16/NodeNext（Node.js项目首选）
        // Classic/Node(早期版本使用，不建议)

        // base URL：设置基础路径
        // 所有非相对路径导入都基于此路径解析
        "baseUrl": "./src",

        // 路径映射：为导入路径设置别名
        "paths": {
            // @ 开头的导入映射到 src 目录
            "@/*": ["./*"],
            // @components 开头的导入映射到 components 目录
            "@components/*": ["./components/*"]
        }
    }
}
```

### 动态导入

动态导入使用import()函数，可以在运行时动态加载模块，返回一个Promise对象。

```ts
// 动态导入 - 懒加载
// import() 返回一个 Promise
async function loadMath() {
    // 动态导入模块，只有执行到这里才会加载
    var math = await import("./math");

    // math.default 是默认导出的函数
    console.log("动态加法: " + math.default(1, 2));
}

// 调用懒加载函数
loadMath();

// 条件导入：根据条件动态加载不同模块
async function loadFeature(enable: boolean) {
    if (enable) {
        // 只有条件满足时才加载模块
        var feature = await import("./feature");
        feature.run();
    }
}

// 根据条件加载
loadFeature(true); 
```

## 命名空间（不推荐）

命名空间最明确的一个目的就是解决重名问题，命名空间定义了标识符的可见范围，一个标识符可在多个命名空间中定义，它在不同命名空间中的含义是互不相干的。

TypeScript中命名空间使用`namespace`关键字来定义，语法格式如下：

```ts
namespace SomeNameSpaceName { 
   export interface ISomeInterfaceName { 
      // 接口定义 
   } 
   export class SomeClassName { 
      // 类定义 
   } 
}
```

以上定义了一个命名空间`SomeNameSpaceName`，如果需要在外部调用SomeNameSpaceName中的接口和类，则需要在类和接口添加`export`关键字。

如果一个命名空间在一个单独的TypeScript文件中，则应使用三斜杠///引用它，语法格式如下：

```ts
/// <reference path="SomeFileName.ts" />
```

### 实例

IShape.ts

```ts
namespace Drawing { 
    export interface IShape { 
        draw(): void; 
    }
}
```

Circle.ts

```ts
/// <reference path = "IShape.ts" /> 
namespace Drawing { 
    export class Circle implements IShape { 
        public draw(): void { 
            console.log("Circle is drawn"); 
        }  
    }
}
```

Triangle.ts

```ts
/// <reference path = "IShape.ts" />
namespace Drawing {
    export class Triangle implements IShape {
        public draw(): void {
            console.log("Triangle is drawn");
        }
    }
}
```

Main.ts

```ts
/// <reference path = "IShape.ts" />
/// <reference path = "Circle.ts" />
/// <reference path = "Triangle.ts" />
function drawAllShapes(shape: Drawing.IShape): void {
    shape.draw();
}
drawAllShapes(new Drawing.Circle()); // 输出：Circle is drawn
drawAllShapes(new Drawing.Triangle()); // 输出：Triangle is drawn
```

### 嵌套命名空间

命名空间可以嵌套，嵌套命名空间的使用方法和普通命名空间一样，只要在嵌套的命名空间中添加`export`关键字即可。

```ts
namespace namespace_name1 { 
    export namespace namespace_name2 {
        export class class_name {    } 
    } 
}
```

#### 嵌套实例

Invoice.ts

```ts
namespace Runoob { 
   export namespace invoiceApp { 
      export class Invoice { 
         public calculateDiscount(price: number) { 
            return price * 0.40; 
         } 
      } 
   } 
}
```

Main.ts

```ts
/// <reference path = "Invoice.ts" />
let invoice = new Runoob.invoiceApp.Invoice();
console.log(invoice.calculateDiscount(500)); // 输出：200
```

## 环境声明

### 声明文件

声明文件用于描述非TypeScript代码的接口和类型，使得TypeScript可以正确地识别和使用这些代码。

声明文件使用`.d.ts`后缀，包含`declare`关键字，用于声明全局变量、函数、类、接口等。

```ts
// 声明全局变量
declare var $: any;

// 声明全局函数
declare function greet(name: string): void;

// 声明全局类
declare class Greeter {
    constructor(greeting: string);
    greet(): string;
}
```

### 声明合并

声明合并用于将多个声明合并为一个声明。

#### 自身合并

接口、命名空间、枚举可以进行自身合并。

interface自身合并

```ts
// 第一次声明
interface Person {
    name: string;
}

// 第二次声明（同名）
interface Person {
    age: number;
}

// 合并后等价于：
interface Person {
    name: string;
    age: number;
}

// 使用时必须同时满足两个声明的属性
const p: Person = { name: "Alice", age: 25 }; // ✅ 正确
// const p2: Person = { name: "Bob" };        // ❌ 缺少 age
```

命名空间自身合并

```ts
// 第一次声明
namespace Person {
    export const name = "Alice";
}

// 第二次声明（同名）
namespace Person {
    export const age = 25;
}

// 合并后等价于：
namespace Person {
    export const name = "Alice";
    export const age = 25;
}

// 使用时可以访问合并后的命名空间
console.log(Person.name); // 输出：Alice
console.log(Person.age);  // 输出：25
```

枚举自身合并

```ts
// 第一次声明
enum Color {
    Red,
    Green,
    Blue
}

// 第二次声明（同名）
enum Color {
    Yellow,
    Purple
}

// 合并后等价于：
enum Color {
    Red = 0,
    Green = 1,
    Blue = 2,
    Yellow = 3,
    Purple = 4
}

// 使用时可以访问合并后的枚举
console.log(Color.Red);    // 输出：0
console.log(Color.Green);  // 输出：1
console.log(Color.Yellow); // 输出：3
console.log(Color.Purple); // 输出：4
```

#### 交叉合并

函数、命名空间、类、枚举可以进行交叉合并。

function + namespace

```ts
// 声明一个函数
function greet(name: string): void {
    console.log("Hello, " + name);
}

// 同名 namespace 给函数添加属性
namespace greet {
    export let version = "1.0";
    export function admin() {
        console.log("Hello, admin");
    }
}

// 使用
greet("Alice");        // Hello, Alice
console.log(greet.version);  // 1.0
greet.admin();         // Hello, admin
```

class + namespace

```ts
// 声明一个类
class User {
    name: string;
    constructor(name: string) {
        this.name = name;
    }
}

// 给类添加静态属性和方法
namespace User {
    export let count = 0;
    export function create(name: string): User {
        User.count++;
        return new User(name);
    }
}

// 使用
const u = User.create("Alice");
console.log(User.count); // 1
```

enum + namespace

```ts
// 声明一个枚举
enum Color {
    Red,
    Green,
    Blue
}

// 给枚举添加静态方法
namespace Color {
    export function hex(c: Color): string {
        switch (c) {
            case Color.Red: return "#FF0000";
            case Color.Green: return "#00FF00";
            case Color.Blue: return "#0000FF";
        }
    }
}

console.log(Color.hex(Color.Red)); // #FF0000
```
