# 类型

TypeScript 的类型系统是在 JavaScript 基础上添加的静态类型检查机制，用来声明变量、函数、对象等应该是什么形状（即拥有哪些属性和方法）。

## Number详解（不推荐）

在TypeScript中，Number对象用于包装数值类型。类似于String对象，Number对象是引用类型，与基本的number类型有所不同。尽管Number对象提供了一些额外的属性和方法，但在TypeScript中更推荐使用基本的number类型，因为Number对象会带来性能开销和类型混淆。

语法：

`var num = new Number(value);`

如果参数值不能转换为数字将返回NaN（非数字值）

[Number对象属性/对象方法](https://www.runoob.com/typescript/ts-number.html)

### NaN实例

```ts
var month = 0 
if( month<=0 || month >12) { 
    month = Number.NaN 
    console.log("月份是："+ month) 
} else { 
    console.log("输入月份数值正确。") 
}
// 输出：月份是：NaN
```

### Prototype实例

Number 对象的 prototype 属性，可向构造函数添加属性和方法，所有实例均可访问。

```ts
function employee(id:number,name:string) { 
    this.id = id 
    this.name = name 
} 
 
var emp = new employee(123,"admin") 
employee.prototype.email = "admin@runoob.com" 
 
console.log("员工号: "+emp.id) 
console.log("员工姓名: "+emp.name) 
console.log("员工邮箱: "+emp.email)
// 输出：
// 员工号: 123
// 员工姓名: admin
// 员工邮箱: admin@runoob.com
```

### Number详解小结

`Number` 对象是 `number` 原始类型的包装类，**不推荐使用**：

- `new Number(value)` 创建的是对象而非原始数字，带来性能开销和类型混淆
- 始终使用原始 `number` 类型声明变量和参数
- `Number.NaN`、`Number.prototype` 等属性和方法了解即可，实际开发中很少直接使用

## String详解（不推荐）

在TypeScript中，String对象用于包装字符串类型。类似于Number对象，String对象是引用类型，与基本的string类型有所不同。尽管String对象提供了一些额外的属性和方法，但在TypeScript中更推荐使用基本的string类型，因为String对象会带来性能开销和类型混淆。

语法：

```ts
var str = new String(value);

//或
var str = "value";
```

任何值传入 `new String(value)` 都会被转换为字符串（如 `new String(undefined)` 返回 `String {"undefined"}`），不会返回 undefined。

[String对象属性/对象方法](https://www.runoob.com/typescript/ts-string.html)

String对象和字符串字面量在类型上是不同的，String对象是String类型，实际上是一个对象。

```ts
let strLiteral: string = "Hello";
let strObject: String = new String("Hello");

console.log(typeof strLiteral); // 输出："string"
console.log(typeof strObject);  // 输出："object"

console.log(strLiteral === strObject);  // 输出：false，内容相同，类型不同
console.log(strLiteral == strObject);   // 输出：true，内容相同
console.log(strLiteral === strObject.valueOf()); // 输出：true，将对象转为原始字符串后比较
```

String 对象是 `string` 原始类型的包装类，**不推荐使用**：

- `new String(value)` 创建的是对象（`typeof` 返回 `"object"`），而非原始字符串
- 带来性能开销和类型混淆（`===` 比较时因类型不同而返回 `false`），应始终使用原始 `string` 类型
- 任何值传入 `new String()` 都会被转为字符串（如 `new String(undefined)` → `String {"undefined"}`）

## Array详解

数组对象是使用单独的变量名来存储一系列的值，并且可以通过索引来访问每个值。

声明数组的语法格式如下：

```ts
var array_name: number[];        // 声明
array_name = [val1, val2, valn]; // 初始化

// 或直接声明时初始化
var array_name2: number[] = [val1, val2, valn];
```

如果数组声明时未设置类型，则会被认为是any类型，在初始化时根据第一个元素的类型来推断数组的类型。如果数组声明时设置了类型，则数组中的所有元素都必须是相同类型。

### Array对象

我们也可以使用Array对象创建数组。Array对象的构造函数接受以下两种值：

- 表示数组大小的数值。
- 数组元素值的列表。

```ts
var arr_names:number[] = new Array(4);
for (let i = 0; i < arr_names.length; i++) {
    arr_names[i] = i + 5;
    console.log(arr_names[i]);
}


var names:string[] = new Array("Google","Runoob","Taobao");
```

[Array方法](https://www.runoob.com/typescript/ts-array.html)

### 数组解构

数组元素可以赋值给变量。

```ts
var arr:number[] = [12,13] 
var[x,y] = arr // 将数组的两个元素赋值给变量 x 和 y
console.log(x) 
console.log(y)
// 输出：
// 12
// 13
```

### 数组迭代

可以使用for循环来遍历数组元素。

```ts
var nums:number[] = [1001,1002,1003,1004];
 
// 推荐：for...of 直接遍历值
for (let val of nums) {
    console.log(val);
}

// 或普通 for 循环
for (let i = 0; i < nums.length; i++) {
    console.log(nums[i]);
}
// 输出：
// 1001
// 1002
// 1003
// 1004
// （两种循环输出相同）
```

### 多维数组

一个数组的元素可以是另外一个数组，这样就构成了多维数组。

```ts
var multi:number[][] = [[1,2,3],[23,24,25]]  
console.log(multi[0][0]) 
console.log(multi[0][1]) 
console.log(multi[0][2]) 
console.log(multi[1][0]) 
console.log(multi[1][1]) 
console.log(multi[1][2])

// 输出：
// 1 
// 2
// 3
// 23
// 24
// 25
```

### 数组在函数中的使用

数组可作为参数传递给函数。

```ts
var sites:string[] = new Array("Google","Runoob","Taobao","Facebook") 
 
function disp(arr_sites:string[]) {
        for(var i = 0;i<arr_sites.length;i++) { 
                console.log(arr_sites[i]) 
        }  
}  
disp(sites);
// 输出：
// Google
// Runoob
// Taobao
// Facebook
```

也可以作为函数的返回值。

```ts
function disp():string[] {
    return new Array("Google","Runoob","Taobao","Facebook")
}

var sites:string[] = disp()
// 注意：for...in 遍历的是索引，推荐使用 for...of 遍历值
for(var i in sites) {
    console.log(sites[i])
}
// 输出：
// Google
// Runoob
// Taobao
// Facebook
```

### Array详解小结

数组是 TypeScript 中最常用的集合类型，核心要点：

- 声明语法：`type[]` 或 `Array<type>`，推荐前者
- 未声明类型时根据首元素推断，声明后所有元素必须同类型
- `for...of` 遍历值比 `for...in` 遍历索引更推荐
- 数组可作为函数参数和返回值，支持多维数组（`number[][]`）

## Map详解

Map对象保存键值对，并且能够记住键的原始插入顺序。任何值（对象或者原始值）都可以作为一个键或一个值。

### 创建Map

TypeScript使用Map类型和new关键字来创建Map。

```ts
var myMap = new Map();
```

初始化Map，可以以数组的格式来传入键值对。

```ts
var myMap = new Map([
    ["key1", "value1"],
    ["key2", "value2"]
]);
```

### 常用函数

[Map相关函数和属性](https://www.runoob.com/typescript/ts-map.html)

```ts
var myMap = new Map([
    ["key1", "value1"],
    ["key2", "value2"]
]);

myMap.set("key3", "value2"); // 添加键值对
myMap.set("key3", "value3"); // 修改键值对（覆盖前面的值）

const val1 = myMap.get("key1"); // 根据key1获取value1，不存在则返回undefined

const exists = myMap.has("key1"); // 判断是否存在key1，返回 true

const remove = myMap.delete("key3"); // 删除key3，成功则返回true，不存在则返回false

const size = myMap.size; // 获取Map的大小，值为 3

myMap.clear(); // 清空Map
// 执行后 myMap.size === 0
```

### 遍历Map

[迭代方法](https://www.runoob.com/typescript/ts-map.html)

```ts
var myMap = new Map([
    ["key1", "value1"],
    ["key2", "value2"]
]);

// 遍历键
for (let key of myMap.keys()) {
    console.log(key);
}
// 输出：
// key1
// key2

// 遍历值
for (let value of myMap.values()) {
    console.log(value);
}
// 输出：
// value1
// value2

// 遍历键值对
for (let [key, value] of myMap.entries()) {
    console.log(key, value);
}
// 输出：
// key1 value1
// key2 value2

// 对Map中的每个键值对执行一次回调函数。
myMap.forEach(function(value, key) {
    //Map的forEach回调参数顺序是(value,key)
    console.log(key, value);
});
// 输出：
// key1 value1
// key2 value2
```

### Map详解小结

Map 是有序键值对集合，键可以是任意类型：

- 创建：`new Map()` 或 `new Map([[k1,v1], [k2,v2]])`
- 核心操作：`set`（增/改）、`get`、`has`、`delete`、`clear`、`size`
- 遍历：`keys()`、`values()`、`entries()`、`forEach`，注意 `forEach` 回调参数顺序是 `(value, key)`
- 与普通对象相比：键类型不限、记住插入顺序、有专用迭代方法

## 枚举详解

枚举是TypeScript中非常有用的特性，它定义了一组命名常量。枚举允许开发者为一组数值赋予更加友好的名字，使代码更加易读和易维护。

### 数字枚举

默认情况下，枚举从0开始编号。

```ts
enum Direction {
    Up,    // 0
    Down,  // 1
    Left,  // 2
    Right  // 3
}

var dir: Direction = Direction.Up;
console.log("方向: " + dir);
console.log("方向名称: " + Direction[0]);
// 输出：
// 方向: 0
// 方向名称: Up
```

### 手动赋值

枚举成员可以手动赋值，后续成员的值会自动递增。

```ts
enum Status {
    Success = 1,
    Error = 2,
    Pending = 3
}

console.log("状态: " + Status.Success);
console.log("状态名称: " + Status[1]);
// 输出：
// 状态: 1
// 状态名称: Success
```

### 字符串枚举

字符串枚举的每个成员都必须用字符串字面量进行初始化。

```ts
enum Message {
    Success = "SUCCESS",
    Error = "ERROR",
    Warning = "WARNING"
}

console.log("消息: " + Message.Success);
// 输出：消息: SUCCESS
```

### 常量枚举

使用const修饰的枚举会在编译时内联，生成更优化的代码。

```ts
const enum Color {
    Red = "RED",
    Green = "GREEN",
    Blue = "BLUE"
}

var favoriteColor: Color = Color.Red;
console.log("喜欢的颜色: " + favoriteColor);
// 输出：喜欢的颜色: RED
```

### 异构枚举（不推荐）

异构枚举的成员值可以是数字或字符串。

```ts
enum BooleanLikeHeterogeneousEnum {
    No = 0,
    Yes = "YES"
}

console.log("值: " + BooleanLikeHeterogeneousEnum.No);
console.log("字符串值: " + BooleanLikeHeterogeneousEnum.Yes);
// 输出：
// 值: 0
// 字符串值: YES
```

### 枚举成员类型

当枚举成员都是字面量值时，成员类型可以作为类型使用。

```ts
enum ShapeKind {
    Circle = "circle",
    Square = "square"
}

interface Circle {
    kind: ShapeKind.Circle;
    radius: number;
}

interface Square {
    kind: ShapeKind.Square;
    sideLength: number;
}

var c: Circle = {
    kind: ShapeKind.Circle,
    radius: 10
};

console.log("圆形: " + JSON.stringify(c));
// 输出：圆形: {"kind":"circle","radius":10}
```

### 运行时常量枚举

```ts
enum FileAccess {
    Read = 1 << 1, // 2
    Write = 1 << 2, // 4
    ReadWrite = Read | Write // 6
}

console.log("文件访问: " + FileAccess.ReadWrite);
// 输出：文件访问: 6
```

### 枚举详解小结

枚举定义一组命名常量，提升代码可读性：

- **数字枚举**：默认从 0 递增，可手动赋值，支持反向映射（`Enum[value]` 获取名称）
- **字符串枚举**：每个成员必须初始化，无反向映射，但值更具可读性
- **常量枚举（`const enum`）**：编译时内联，不生成运行时代码，性能最优
- **异构枚举**：混合数字和字符串，不推荐使用
- **枚举成员类型**：字面量枚举成员可作为类型使用，用于区分形状

## 类型推断详解

类型推断是TypeScript最强大和便捷的特性之一，它允许编译器自动分析代码的上下文，并根据变量的值、函数的返回值等自动推断出变量的类型。

### 基础类型推断

TypeScript会根据变量的值自动推断出变量的类型。

```ts
let num = 10;
let str = "hello";
let bool = true;
```

### 返回类型推断

TypeScript会根据函数的返回值自动推断出函数的返回类型。

```ts
// TypeScript 推断返回类型为 number
// 因为返回的是 a + b 的结果
function add(a: number, b: number) {
    return a + b;
}

// TypeScript 推断返回类型为 string
// 因为返回的是字符串拼接结果
function greet(name: string) {
    return "Hello, " + name;
}

// 调用函数，返回值类型已被正确推断
var result = add(1, 2);
var message = greet("TypeScript");

console.log("加法结果: " + result); // 输出：加法结果: 3
console.log("问候语: " + message); // 输出：问候语: Hello, TypeScript
```

### 最佳通用类型推断

当有多个候选类型时，TypeScript会选择能够容纳所候选类型的“最佳通用类型”，其中也包括联合类型。

```ts
// 混合数组包含 number 和 string
// TypeScript 推断为 (number | string)[]（联合类型）
var mixed = [1, "two", 3, "four"];

// 访问不同类型的元素
console.log("混合数组: " + mixed);                                     // 输出：混合数组: 1,two,3,four
console.log("类型: " + typeof mixed[0] + ", " + typeof mixed[1]);     // 输出：类型: number, string
```

### 类型推断的限制

TypeScript的类型推断在某些情况下可能不够准确，例如在函数参数和返回值之间没有明确的类型声明时，TypeScript可能无法正确推断出类型。

```ts
// TypeScript 无法通过函数体内部反向推断参数是什么类型，只能退化为 any
function greet(name) {
    return "Hello, " + name;
    // 返回值类型推断为 string，因为字符串拼接的结果始终是 string
}

var message = greet("TypeScript");

console.log("问候语: " + message); // 输出：问候语: Hello, TypeScript
```

避免声明变量时不赋值，最好在声明时提供初始值，或显式声明类型，以充分利用类型检查功能。

```ts
// 声明变量时不赋值，也没有初始值
// TypeScript 无法推断类型，会推断为 any
var unknown;

// 可以赋任何类型的值
unknown = "hello";
unknown = 123;

// 这种行为容易导致类型错误
console.log("未指定类型: " + unknown);  // 输出：未指定类型: 123

// 建议：显式指定类型以获得更好的类型安全
var fixedNumber: number = 42;

console.log("指定类型: " + fixedNumber); // 输出：指定类型: 42
```

### 泛型函数推断

TypeScript会根据传入的参数类型推断出函数的泛型类型。

```ts
// 泛型函数：接受任意类型参数，返回同类型值
// T 会根据传入的参数自动推断
function identity<T>(arg: T): T {
    return arg;
}

// 传入字符串，T 推断为 string
var str = identity("hello");

// 传入数字，T 推断为 number
var num = identity(42);

// 传入对象，T 推断为对象类型
var obj = identity({ name: "TypeScript" });

console.log("字符串: " + str);                         // 输出：字符串: hello
console.log("数字: " + num);                           // 输出：数字: 42
console.log("对象: " + JSON.stringify(obj));            // 输出：对象: {"name":"TypeScript"}
```

### 类型推断详解小结

类型推断让 TypeScript 自动推导类型，减少显式注解：

- **基础推断**：根据初始值推断变量类型（`let x = 10` → `number`）
- **返回类型推断**：根据 `return` 语句推断函数返回值类型
- **最佳通用类型**：数组等集合会推断为能容纳所有元素的联合类型
- **泛型推断**：根据传入的实参自动推断泛型参数的类型
- **限制**：无初始值时退化为 `any`，建议始终声明类型或赋初值

## 类型断言详解

类型断言允许开发者显式地告诉编译器变量的类型，从而绕过类型检查。类似于其他语言的类型转换，但仅在编译阶段生效，不产生任何运行时代码。

### 语法形式

类型断言有两种写法，推荐统一使用as语法，尖括号写法在jsx文件中会与标签语法冲突。

```ts
// ① 尖括号语法（不推荐，在 .tsx 文件中无法使用）
const str1: any = "hello";
const len1: number = (<string>str1).length;

// ② as 语法（推荐）
const str2: any = "world";
const len2: number = (str2 as string).length;

console.log(len1);  // 5
console.log(len2);  // 5
```

### 常见使用场景

处理any/unknown类型：当从外部接口、JSON解析等场景获取的数据通常为any，通过断言可以恢复类型提示。

```ts
const response: any = { name: "Alice", age: 25 };

// 断言为具体类型，获得类型提示
const user = response as { name: string; age: number };

console.log(user.name);  // Alice
console.log(user.age);   // 25
```

收窄联合类型： 当已知联合类型中的具体分支，但编译器无法自动判断时，可使用断言明确类型。

```ts
type Shape =
    | { kind: "circle"; radius: number }
    | { kind: "rect"; width: number; height: number };

function getArea(shape: Shape): number {
    if (shape.kind === "circle") {
        return Math.PI * (shape as { kind: "circle"; radius: number }).radius ** 2;
    }
    const rect = shape as { kind: "rect"; width: number; height: number };
    return rect.width * rect.height;
}

console.log(getArea({ kind: "circle", radius: 5 }).toFixed(2));  // 78.54
console.log(getArea({ kind: "rect", width: 4, height: 6 }));     // 24
```

优先使用类型守卫（typeof/instanceof/判别属性）收窄类型，只有在守卫无法覆盖的场景才使用断言。

```ts
// 利用类型收窄，无需任何断言
function getArea(shape: Shape): number {
    if (shape.kind === "circle") {
        // TypeScript 自动收窄：shape 现在是 circle 类型
        return Math.PI * shape.radius ** 2;
    }
    // TypeScript 自动收窄：shape 现在是 rect 类型
    return shape.width * shape.height;
}
```

### 操作DOM元素

`document.querySelector`返回的是`Element | null`，断言为具体元素类型后才能访问其专属属性。

```ts
// querySelector 返回 Element | null
const input = document.querySelector("#username") as HTMLInputElement;

// 断言后可访问 HTMLInputElement 的专属属性
// console.log(input.value);

// 更安全的写法：先判空再断言
const btn = document.querySelector("#submit");
if (btn) {
    (btn as HTMLButtonElement).disabled = true;
}
```

### 非空断言

在值后加！，告诉编译器该值不为null或undefined，适用于你能确保非空但编译器无法推断的场景。

```ts
function printLength(str?: string) {
    // 用 ! 断言 str 一定有值
    console.log(str!.length);
}

// 运行正常
printLength("hello");  // 5

// 运行时报错：Cannot read properties of undefined
// printLength();
```

非空断言会绕过编译器的空值检查，若实际值为null/undefined，运行时仍会报错。在条件允许时优先使用可选链`?.`替代

```ts
function printLength(str?: string) {
    // 如果 str 是 undefined，运行时崩溃
    console.log(str!.length);
}

// 推荐：使用可选链
function printLengthSafe(str?: string) {
    console.log(str?.length);  // 如果 str 是 undefined，输出 undefined
}

// 更安全：结合空值合并提供默认值
function printLengthWithDefault(str?: string) {
    console.log(str?.length ?? 0);  // 如果 str 是 undefined，输出 0
}
```

### 常量断言

as const将字面量推断为最窄的只读字面量类型，常用于定义枚举值、配置常量和元组。

```ts
// 普通声明：推断为 string[]，元素可修改
const colors1 = ["red", "green", "blue"];
colors1.push("yellow");  // 允许

// as const：推断为 readonly ["red", "green", "blue"]，元素和长度均不可变
const colors2 = ["red", "green", "blue"] as const;
// colors2.push("yellow");  // 错误：只读数组不能 push

// 对象同理：所有属性变为 readonly 字面量类型
const config = {
    host: "localhost",
    port: 3000
} as const;
// config.port = 8080;  // 错误：只读属性不能修改

console.log(colors1);         // ['red', 'green', 'blue', 'yellow']
console.log(colors2);         // ['red', 'green', 'blue']
console.log(config.host);     // localhost
console.log(config.port);     // 3000
```

### 双重断言（不推荐）

当两个类型之间没有重叠关系，TypeScript会直接拒绝断言。此时可借助unknown作为中转，实现双重断言。

```ts
const num = 42;

// 直接断言：number 与 string 无重叠，编译器报错
// const str = num as string;

// 双重断言：先断言为 unknown，再断言为目标类型
const str = num as unknown as string;

console.log(str);          // 42
console.log(typeof str);   // number（运行时类型未变，断言只在编译期生效）
```

### 断言不是类型转换

类型断言只影响编译器的类型检查，编译后不生成任何转换代码。

```ts
const strNum: any = "42";

// 断言为 number，但运行时仍是 string
const wrongNum = strNum as number;
console.log(typeof wrongNum);    // string（不是 number！）
console.log(wrongNum + 1);       // 421（字符串拼接，不是数字相加）

// 真正的类型转换需要使用转换函数
const realNum = Number(strNum);
console.log(typeof realNum);     // number
console.log(realNum + 1);        // 43
```

### 类型断言详解小结

类型断言在编译期告诉编译器"我知道这是什么类型"，不影响运行时：

- **语法**：推荐 `值 as 类型`，尖括号写法在 `.tsx` 中不可用
- **典型场景**：处理 `any`/`unknown` 返回值、收窄联合类型、操作 DOM 元素
- **非空断言（`!`）**：告诉编译器值不为 `null`/`undefined`，有运行时风险
- **常量断言（`as const`）**：将值推断为最窄的只读字面量类型
- **关键认知**：断言不是类型转换，编译后不生成任何代码，运行时类型不变
- **原则**：优先使用类型守卫收窄，守卫无法覆盖时才用断言

## null和undefined详解

在TypeScript中，null和undefined有特殊的处理方式。启用strictNullChecks后，需要显式处理这些值。

### 联合类型处理null

启用strictNullChecks后，需要使用联合类型显式声明可能为null的值。

```ts
// 联合类型：可以是字符串或 null
// 显式声明值可能为空
var name: string | null = "Alice";
name = null;  // 正确：可以赋值为 null

// 访问可能为 null 的值需要先检查
// 函数参数可能是字符串或 null
function getLength(str: string | null): number {
    // 使用条件检查是否为 null
    if (str === null) {
        return 0;  // null 时返回 0
    }
    // TypeScript 会推断 str 不是 null
    return str.length;
}

console.log("长度: " + getLength("hello")); // 5
console.log("长度: " + getLength(null)); // 0
```

### 可选参数和属性

可选参数和可选属性自动包含undefined，不需要显式声明null。

```ts
// 可选参数：使用 ? 标记
// 参数可能是 string 或 undefined
function greet(name?: string): string {
    // 检查是否为 undefined
    if (name === undefined) {
        return "Hello, stranger!";
    }
    return "Hello, " + name;
}

console.log(greet("Alice")); // Hello, Alice
console.log(greet()); // Hello, stranger!

// 可选属性：使用 ? 标记
// age 属性是可选的，可能不存在
interface User {
    name: string;
    age?: number;  // 可选属性
}

var user: User = { name: "Bob" };
console.log("用户: " + JSON.stringify(user)); // 用户: {"name":"Bob"}

// 可选参数 name?: string 等同于 string | undefined
// 可选属性 age?: number 等同于 number | undefined
```

### 非空断言运算符（不建议）

使用`!`告诉编译器不为null或undefined。

```ts
function getLength(str: string | null): number {
    // 非空断言：告诉编译器 str 不为 null
    // 这是一个危险的写法，如果 str 真的为 null 会报错
    return str!.length;
}

console.log("长度: " + getLength("hello"));
```

### 空值合并运算符

使用`??`提供默认值，只有当值为null或undefined时才使用默认值。

```ts
// 初始值为 null
var name: string | null = null;

// ?? 运算符：左侧为 null/undefined 时使用右侧值
// 如果 name 为 null，使用 "Guest"
var displayName = name ?? "Guest";

console.log("显示名称: " + displayName); // 显示名称: Guest

// 对比 || 运算符（会把 0 和空字符串视为 falsy）
var num: number | null = 0;
var result1 = num ?? 100;  // 0（正确：0 不是 null/undefined）
var result2 = num || 100;  // 100（错误：0 被视为 falsy）

console.log("?? 结果: " + result1); // ?? 结果: 0
console.log("|| 结果: " + result2); // || 结果: 100
```

### 可选链

#### 可选链语法

使用 `?.` 访问可能为 null 或 undefined 的属性或方法，如果值为 null 或 undefined 则返回 undefined。

```ts
// 定义一个嵌套的用户对象
// 包含姓名和地址信息，地址中有城市
var user = {
    name: "RUNOOB",
    address: {
        city: "Beijing"
    }
};

// 传统方式：使用 && 逐层检查
// 这种方式代码冗长，容易遗漏
var city1 = user && user.address && user.address.city;

// 可选链方式：使用 ?. 运算符
// 如果任意一层为 null 或 undefined，直接返回 undefined
var city2 = user?.address?.city;

console.log("传统方式: " + city1);
console.log("可选链: " + city2);
```

#### 处理不存在的属性

当访问路径中的属性不存在时，可选链会安全地返回 undefined，而不会抛出错误。这对于处理来自 API 的数据或用户表单输入特别有用。

```ts
// 定义人员类型，地址是可选的
interface Person {
    name: string;
    address?: {
        city: string;
        age: number;
    };
}

// 创建人员对象，没有地址
var person: Person = { name: "Alice" };

// 可选链：安全访问可能不存在的属性
// 如果 address 不存在，返回 undefined 而不会报错
var city = person.address?.city;

// 结合空值合并运算符
console.log("城市: " + city); // 城市: undefined
console.log("城市: " + (person.address?.city ?? "未知")); // 城市: 未知
```

#### 可选链与数组结合

可选链可以与数组下标访问结合使用，安全地访问数组中的元素。使用 `?.[index]` 语法，可以在数组元素不存在时返回 undefined。

```ts
// 定义一个用户数组
var users = [
    { name: "Alice" },
    { name: "Bob" }
];

// 安全访问数组第一个元素的名字
// users?.[0] 存在，返回 "Alice"
var firstUser = users?.[0]?.name;

// 安全访问数组中不存在的元素
// users?.[9] 不存在（数组只有2个元素），返回 undefined
var tenthUser = users?.[9]?.name;

console.log("第一个用户: " + firstUser);
console.log("第十个用户: " + tenthUser);
```

#### 可选链与方法调用

可选链可以用于安全地调用可能不存在的方法。使用 `?.()` 语法，如果方法不存在则返回 undefined，而不会抛出错误。

```ts
// 定义一个用户对象，包含姓名和 greet 方法
var user = {
    name: "Alice",
    // 定义一个打招呼方法
    greet: function() {
        return "Hello, " + this.name;
    }
};

// 安全调用存在的方法
// user.greet 存在，正常调用并返回结果
var message1 = user.greet?.();

// 安全调用不存在的方法
// user.sayHello 不存在，返回 undefined 而不报错
var message2 = user.sayHello?.();

console.log("greet: " + message1);
console.log("sayHello: " + message2);
```

#### 空值合并与可选链

可选链经常与空值合并运算符 `??` 结合使用，可以在属性不存在时提供默认值，使代码更加健壮。

```ts
// 定义一个不完整的用户对象
var user = {
    name: "Alice"
    // address 不存在
};

// 可选链 + 空值合并：当 city 为 null 或 undefined 时使用默认值
var city = user?.address?.city ?? "未知城市";

console.log("城市: " + city);

// 对比：传统方式的复杂写法
var country = user && user.address && user.address.country
    ? user.address.country
    : "未知国家";

console.log("国家: " + country);
```

### null和undefined详解小结

`null` 和 `undefined` 的处理是 TypeScript 空值安全的核心：

- **可选参数/属性（`?`）**：自动包含 `undefined`，无需显式联合
- **空值合并（`??`）**：只在 `null`/`undefined` 时取默认值，语义比 `||` 更精确（`0` 和 `""` 不会被误判）
- **可选链（`?.`）**：安全访问深层属性，遇 `null`/`undefined` 返回 `undefined` 而不报错
- **非空断言（`!`）**：绕过检查，仅在确定非空时谨慎使用
- 最佳实践：`?.` + `??` 组合是处理可选嵌套属性的推荐模式

## 特殊类型详解

TypeScript有四个特殊类型：void、any、never和unknown。

### never

never表示永远不会返回的类型，通常用于抛出异常或无限循环的函数。

```ts
// 抛出异常的函数
// 函数永远不会正常返回，总会抛出错误
function throwError(message: string): never {
    throw new Error(message);
}

// 无限循环的函数
// 函数永远不会返回，程序会一直运行
function infiniteLoop(): never {
    while (true) {
        console.log("运行中...");
    }
}

// never 是所有类型的子类型
// 这意味着 never 可以赋值给任何类型
var neverValue: never;
var num: number = neverValue;  // 正确：never 是 number 的子类型
console.log("never 赋值给 number: " + num); // never 赋值给 number: undefined
```

### void

void表示没有返回值，通常用于声明没有return语句的函数。

```ts
// 无返回值函数
// 函数执行后没有返回值
function logMessage(message: string): void {
    console.log("日志: " + message);
    // 没有 return 语句，或 return undefined
}

logMessage("Hello");

// void 变量（很少使用）
// 只能赋值为 undefined
var empty: void = undefined;
console.log("void 变量: " + empty); // 输出：void 变量: undefined
```

### unknown

unknown表示未知类型，类似于any，但更安全。unknown类型的值不能直接赋值给其他类型，需要先进行类型检查。

```ts
// unknown 接受任意类型
// 这是"安全的" any
var value: unknown = true;
value = 42;
value = "hello";

// 未知类型不能直接赋值给其他类型
// 如果取消注释下一行，编译器会报错
// var str: string = value;

// 需要进行类型检查后，才能赋值
if (typeof value === "string") {
    // TypeScript 知道 value 是 string 类型
    var str: string = value;
    console.log("字符串长度: " + str.length); // 字符串长度: 5
}
```

### any

any完全绕过类型检查，可以赋值为任意类型，这是最不安全的类型。

```ts
// any 可以是任意类型
// 完全绕过类型检查
var anything: any = "hello";
anything = 42;
anything = true;

// any 可以赋值给任意类型
// 这会破坏类型安全
var str: string = anything;
var num: number = anything;

console.log("字符串: " + str); // 字符串: true
console.log("数字: " + num); // 数字: true

// any 类型的对象可以调用任意方法
// 编译器不会报错，但运行时可能出错
var obj: any = {};
obj.foo();  // 不会报错，但实际上 foo 方法不存在
obj.bar = "value";
```

### 实际应用：穷举检查

```ts
// 定义形状的联合类型
type Shape = { kind: "circle", radius: number }
           | { kind: "square", side: number };

// 计算形状面积
function area(shape: Shape): number {
    switch (shape.kind) {
        case "circle":
            return Math.PI * shape.radius **2;
        case "square":
            return shape.side** 2;
        default:
            // default 分支应该处理所有未预料到的情况
            // 如果添加新的 Shape 类型但忘记处理，编译器会报错
            var _exhaustive: never = shape;
            return _exhaustive;
    }
}

var circle = { kind: "circle" as const, radius: 5 };
var square = { kind: "square" as const, side: 4 };

console.log("圆形面积: " + area(circle).toFixed(2));
console.log("正方形面积: " + area(square));
```

### 特殊类型详解小结

四种特殊类型各有明确用途，不可混用：

- **`void`**：函数无返回值（或返回 `undefined`），是最常用的特殊类型
- **`any`**：完全绕过类型检查，应尽量避免，仅在渐进式迁移或紧急兜底时使用
- **`unknown`**："安全的 any"，接受任意值但不能直接使用，必须先做类型检查
- **`never`**：表示永远不会发生的类型（抛异常、死循环），可用于穷举检查的 `default` 分支
- 选择优先级：`void` > `unknown` > `never` > 尽量避免 `any`

## Set、WeakSet和WeakMap详解

### Set

Set是一种集合类型，用于存储唯一值，提供了自动去重、添加、删除等方法，Set可以是任意类型的值。

```ts
// 创建 Set，指定元素类型为 number
var numbers = new Set<number>();

// 添加元素
numbers.add(1);
numbers.add(2);
numbers.add(3);
numbers.add(1); // 重复值会被忽略，不会添加

// 检查大小和包含
console.log("Set 大小: " + numbers.size);
console.log("是否包含 2: " + numbers.has(2));

// 遍历 Set
numbers.forEach(function(value) {
    console.log("值: " + value);
});

// 转换为数组
var arr = Array.from(numbers);
console.log("转换为数组: " + arr);
```

### Set类型注解

可以显式指定Set中值的类型。

```ts
// 字符串 Set
// 只能添加字符串类型的值
var stringSet: Set<string> = new Set();
stringSet.add("a");
stringSet.add("b");

// 对象 Set
// 定义 Person 接口
interface Person {
    name: string;
}
// 创建存储 Person 对象的 Set
var personSet: Set<Person> = new Set();
personSet.add({ name: "Alice" });
personSet.add({ name: "Bob" });

console.log("字符串 Set: " + Array.from(stringSet));
console.log("对象 Set 大小: " + personSet.size);
```

泛型：使用`Set<T>`语法指定Set中元素的类型。

### WeakSet

WeakSet是一种弱集合类型，与Set类似，但只能存储对象，并且对象是弱引用，不会阻止垃圾回收。

```ts
// WeakSet 只能存储对象，不能存储原始值
var weakSet = new WeakSet();

// 创建对象
var obj1 = { name: "Alice" };
var obj2 = { name: "Bob" };

// 添加对象到 WeakSet
weakSet.add(obj1);
weakSet.add(obj2);

// 检查是否包含
console.log("是否包含 obj1: " + weakSet.has(obj1));

// 移除引用后，对象可能被垃圾回收
weakSet.delete(obj1);
console.log("删除后是否包含 obj1: " + weakSet.has(obj1));
```

### WeakMap

WeakMap是一种弱键值对集合类型，与Map类似，但键必须是对象，并且是弱引用，不会阻止垃圾回收。

```ts
// WeakMap 的键必须是对象
// 键类型为 object，值类型为 string
var weakMap = new WeakMap<object, string>();

// 创建对象作为键
var keyObj = { id: 1 };
// 设置键值对
weakMap.set(keyObj, "value1");

// 获取值
console.log("获取值: " + weakMap.get(keyObj));
console.log("是否包含: " + weakMap.has(keyObj));

// 删除键值对
weakMap.delete(keyObj);
console.log("删除后: " + weakMap.has(keyObj));
```

### Set、WeakSet和WeakMap详解小结

这三种集合类型用于特定场景的值存储：

- **`Set<T>`**：存储唯一值，自动去重，支持 `add`、`has`、`delete`、`forEach`
- **`WeakSet`**：只能存对象，弱引用（不阻止 GC），不可遍历，适用于标记/追踪对象
- **`WeakMap<K, V>`**：键必须是对象，弱引用，不可遍历，适用于关联额外数据而不影响 GC
- Weak 版本的核心价值：对象被回收后自动从集合中移除，避免内存泄漏

## 元组详解

一般来说数组中元素的类型是相同的（any[]类型除外），如果存储的元素数据类型不同，则需要使用元组。元组是固定长度的数组，每个元素都有指定的类型。

创建元组的语法格式如下：

`let 元组变量名: [类型1, 类型2, 类型3, ...]`

声明一个元组并初始化：

```ts
let mytuple: [number, string];
mytuple = [42,"Runoob"];
```

访问元组：

`元组变量名[索引]`

定义元组并访问：

```ts
let mytuple: [number, string, boolean] = [42, "Runoob", true];
 
// 访问元组中的元素
let num = mytuple[0]; // 访问第一个元素，值为 42，类型为 number
let str = mytuple[1]; // 访问第二个元素，值为 "Runoob"，类型为 string
let bool = mytuple[2]; // 访问第三个元素，值为 true，类型为 boolean
 
console.log(num);  // 输出: 42
console.log(str);  // 输出: Runoob
console.log(bool); // 输出: true
```

### 元组运算

可以使用以下两个函数向元组添加新元素或删除元素：

- `push()`：向元组末尾添加元素。
- `pop()`：删除元组末尾的元素。

`push()`在类型检查上比较宽松（允许添加联合类型的元素），但在索引访问时仍会检查长度，所以当索引越界后，不能通过索引访问。

```ts
var tuple: [number, string]= [42, "Hello"];
// 添加符合类型的元素
tuple.push("World"); 
// 合法，添加的元素是 (string | number) 类型
console.log(tuple); 
// 输出: [42, "Hello", "World"]
// 但不能通过tuple[2]访问，因为元组长度为2
```

`pop()`不仅会删除元组末尾的元素，也会返回删除的元素。

```ts
let tuple: [number, string, boolean] = [42, "Hello", true];

// 移除最后一个元素
let lastElement = tuple.pop();

console.log(lastElement); // 输出: true
console.log(tuple);       // 输出: [42, "Hello"]
```

### 更新元组

元组是可变的，可以通过索引来更新元组中的元素。

```ts
var mytuple = [42, "Runoob", "Taobao", "Google"]; // 创建一个元组
console.log("元组的第一个元素为：" + mytuple[0]) 
 
// 更新元组元素
mytuple[0] = 121     
console.log("元组中的第一个元素更新为："+ mytuple[0])
```

### 元组解构

元组也可以使用解构赋值语法来提取值。

```ts
let a: [number, string, boolean] = [42, "Hello", true];// 创建一个元组

var [b,c] = a
console.log( b )
console.log( c )
```

### 使用标签元组

TypeScript 还允许为元组中的每个元素添加标签，这使得元组的含义更加清晰：

```ts
let tuple: [id: number, name: string] = [1, "John"];
```

### 元组的类型推断

TypeScript 可以根据元组的初始值推断出元组的类型：

```ts
let tuple = [42, "Hello"] as const; // 元组类型：readonly [42, "Hello"]
```

通过as const断言，TypeScript会将该元组视为一个不可变的常量元组。

### 连接元组

元组可以使用数组的concat方法进行连接，但连接后的结果是一个普通的数组，而不是元组。

```ts
let tuple1: [number, string] = [42, "Hello"];
let tuple2: [boolean, number] = [true, 100];

let result = tuple1.concat(tuple2); // 结果是一个数组: [42, "Hello", true, 100]
console.log(result); // 输出: [42, "Hello", true, 100]
```

### 切片元组

可以使用数组的slice方法来获取元组的子集，但切片后的结果是一个普通的数组，而不是元组。

```ts
let tuple: [number, string, boolean] = [42, "Hello", true];

let sliced = tuple.slice(1); // 从索引 1 开始切片
console.log(sliced); // 输出: ["Hello", true]
```

### 遍历元组

可以使用for...of循环或forEach方法来遍历元组中的元素。

```ts
let tuple: [number, string, boolean] = [42, "Hello", true];

// 使用 for...of 循环遍历元组
for (let item of tuple) {
    console.log(item);
}

// 使用 forEach 方法遍历元组
tuple.forEach(item => console.log(item));
```

### 转换为普通数组

虽然元组是一个固定长度、固定类型的数组，但可以通过 Array.prototype 的方法将其转换为普通数组进行进一步处理。

```ts
let tuple: [number, string, boolean] = [42, "Hello", true];

// 转换为(string | number | boolean)[]数组并使用数组方法
let array = Array.from(tuple);
array.push("New Element");

console.log(array); // 输出: [42, "Hello", true, "New Element"]
```

### 拓展元组

使用剩余运算符可以轻松地将多个元组合并成一个新的元组。

```ts
let tuple1: [number, string] = [42, "Hello"];
let tuple2: [boolean] = [true];
let tuple3: [string, number] = ["World", 100];

let extendedTuple: [number, string, ...typeof tuple2] = [42, "Hello", ...tuple2, ...tuple3];
// 在元组类型的定义中，...展开运算符可以放在任意位置，这是TypeScript 4.0 引入的新特性。并非解构语法，而是类型定义语法。

console.log(extendedTuple); // 输出: [42, "Hello", true, "World", 100]
```

### 元组详解小结

元组是固定长度、每个位置有特定类型的数组：

- 声明：`[类型1, 类型2, ...]`，如 `[number, string]`
- `push`/`pop` 在类型检查上较宽松，但索引访问受长度限制
- 支持标签元组（`[id: number, name: string]`）提升可读性
- 支持解构、`as const` 常量断言、展开合并（TS 4.0+）
- 适合函数多返回值、固定结构的数据（如 CSV 行）

## 联合类型详解

联合类型表示一个值可以是几种类型之一，使用竖线（|）分隔每个类型。

创建联合类型的语法如下：

`let 变量名: 类型1 | 类型2 | 类型3`

声明一个联合类型的变量并初始化：

```ts
var val:string|number 
val = 12 
console.log("数字为 "+ val) 
val = "Runoob" 
console.log("字符串为 " + val)
```

也可以将联合类型作为函数参数使用：

```ts
function disp(name:string|string[]) { 
        if (typeof name === "string") { 
                console.log(name) 
        } else { 
                var i; 
                for(i = 0;i<name.length;i++) { 
                console.log(name[i])
                } 
        } 
} 
disp("Runoob") 
console.log("输出数组....") 
disp(["Runoob","Google","Taobao","Facebook"])
// 输出：
// Runoob
// 输出数组....
// Runoob
// Google
// Taobao
// Facebook
```

### 联合类型数组

也可以将数组声明为联合类型。

```ts
var arr:number[]|string[]; 
var i:number; 
arr = [1,2,4] 
console.log("**数字数组**")  
 
for(i = 0;i<arr.length;i++) { 
   console.log(arr[i]) 
}  
 
arr = ["Runoob","Google","Taobao"] 
console.log("**字符串数组**")  
 
for(i = 0;i<arr.length;i++) { 
   console.log(arr[i]) 
}
// 输出：
// **数字数组**
// 1
// 2
// 4
// **字符串数组**
// Runoob
// Google
// Taobao
```

### 联合类型详解小结

联合类型（`|`）表示"满足其一即可"，是 TypeScript 最常用的类型组合方式：

- 变量可以是多种类型之一，用 `typeof` 等类型守卫收窄后安全使用
- 可声明联合类型数组：`number[] | string[]`（整个数组是同一种类型）
- 与函数重载互补：重载返回类型更精确，联合类型写法更简洁
- 适合表示"多种可能"的场景：状态值、API 返回结果、配置选项等

## 泛型详解

泛型（Generics）是指在定义函数、接口或类时不具体指定类型，而是在使用时再指定类型的一种特性。泛型可以增加代码的复用性和灵活性。

### 泛型标识符

泛型标识符通常用`T`表示，也可以使用其他字母，如`U`、`V`等。

T：代表`Type`，是最常见的泛型类型参数

```ts
function identity<T>(arg: T): T {
    return arg;
}
```

K,V：代表`Key`和`Value`，表示键和值的泛型类型参数

```ts
interface KeyValuePair<K, V> {
    key: K;
    value: V;
}
```

E: 代表`Element`，代表数组元素的泛型类型参数

```ts
function printArray<E>(arr: E[]): void {
    arr.forEach(item => console.log(item));
}
```

R: 代表`Return`，表示函数返回值的泛型类型参数

```ts
function getResult<R>(value: R): R {
    return value;
}
```

U,V,W: 其他泛型标识符

```ts
function combine<U, V>(first: U, second: V): string {
    return `${first} ${second}`;
}
```

### 泛型函数

使用泛型来创建一个可以处理不同类型的函数。

```ts
function identity<T>(arg: T): T {
    // <T>表示使用泛型，arg: T表示参数类型为T
    // 返回值可以省略由TypeScript自动推断
    return arg;
}

// 使用泛型函数，通过尖括号明确指定泛型类型。
let result = identity<string>("Hello");
console.log(result); // 输出: Hello

let numberResult = identity<number>(42);
console.log(numberResult); // 输出: 42
```

### 泛型接口

可以使用泛型来定义接口，使接口的成员能够使用任意类型。

```ts
// 基本语法
interface Pair<T, U> {
    first: T;
    second: U;
}

// 使用泛型接口
let pair: Pair<string, number> = { first: "hello", second: 42 };
console.log(pair); // 输出: { first: 'hello', second: 42 }
```

### 泛型类

泛型类与泛型函数类似，可以在类中使用泛型来定义类的成员。

```ts
// 基本语法
class Box<T> {
    private value: T;

    constructor(value: T) {
        this.value = value;
    }

    getValue(): T {
        return this.value;
    }
}

// 使用泛型类
let stringBox = new Box<string>("TypeScript");
console.log(stringBox.getValue()); // 输出: TypeScript
```

### 泛型约束

有时我们希望泛型类型参数具有某些特定的属性或方法，这时可以使用泛型约束来限制泛型类型参数。

```ts
// 基本语法
interface Lengthwise {
    length: number;
    toUpperCase(): string;
}

function logLength<T extends Lengthwise>(arg: T): void {
    // T extends Lengthwise表示T必须满足Lengthwise接口的要求
    // 即T必须有length属性和toUpperCase方法
    console.log(arg.length);
    console.log(arg.toUpperCase());
}

// 正确
logLength("hello"); // 输出：5 HELLO

// 错误：数组有 length 属性，但没有 toUpperCase 方法
// logLength([1, 2, 3]); // 编译错误：类型“number[]”的参数不能赋给类型“Lengthwise”的参数
```

结构上匹配即可被认为满足约束，这也叫鸭子类型。

```ts
interface HasName {
    name: string;
}

function greet<T extends HasName>(obj: T): void {
    console.log(`Hello, ${obj.name}!`);
}

// 不需要 implements HasName
greet({ name: "Alice", age: 25 });  // ✅ 对象自带 name 属性
greet({ name: "Bob" });              // ✅
greet({ age: 25 });                  // ❌ 没有 name 属性
```

### 泛型的默认值

可以给泛型设置默认值，使得不指定类型参数时能够使用默认类型。

```ts
// 基本语法
function defaultValue<T = string>(arg: T): T {
    return arg;
}

// 使用带默认值的泛型函数
let result1 = defaultValue("hello"); // 推断为 string 类型
let result2 = defaultValue(42);      // 推断为 number 类型
```

### 泛型详解小结

泛型让代码"在定义时不指定类型，使用时再确定"，兼顾灵活与安全：

- **标识符约定**：`T`（Type）、`K,V`（Key/Value）、`E`（Element）、`R`（Return）
- **泛型约束（`extends`）**：限制泛型参数必须具有某些成员，如 `T extends { length: number }`
- **鸭子类型**：约束检查是结构化的，有对应成员就算满足，不要求显式 `implements`
- **应用范围**：泛型函数、泛型接口、泛型类均可定义，也可设默认类型参数

## Symbol详解

Symbol是ES6引入的一种基本数据类型，表示唯一的标识符。TypeScript中Symbol可以作为对象的属性键，确保属性的唯一性。
通过`Symbol()`函数创建的值是唯一的，即使两个Symbol的描述相同，它们也不相等。

### 创建Symbol

使用`Symbol()`函数创建唯一的Symbol值。

```ts
// 创建 Symbol，传入描述字符串（可选）
var sym1 = Symbol("description");
var sym2 = Symbol("description");

// 每次创建的 Symbol 都是唯一的，即使描述相同
console.log("sym1 === sym2: " + (sym1 === sym2)); // sym1 === sym2: false
console.log("sym1: " + sym1.toString()); // sym1: Symbol(description)
```

### Symbol作为对象属性

Symbol可以作为对象的属性键，创建唯一的属性名。

```ts
// 创建 Symbol 作为属性键
var sym = Symbol("key");

// 使用 Symbol 作为对象的属性键
var obj = {
    name: "Alice",  // 普通字符串属性
    [sym]: "secret value"  // Symbol 属性，计算属性名
};

// 访问普通属性
console.log("普通属性: " + obj.name);
// 访问 Symbol 属性
console.log("Symbol 属性: " + obj[sym]);
// Symbol 属性不会出现在 JSON 中
console.log("对象: " + JSON.stringify(obj));

// 输出：
// 普通属性: Alice
// Symbol 属性: secret value
// 对象: {"name":"Alice"}
```

Symbol属性不会出现在JSON序列化中，也不会被for...in遍历到，可以用来创建“私有”属性。

### 全局Symbol注册表

使用`Symbol.for(key)`创建或获取全局Symbol，确保相同描述的Symbol是同一个实例。

```ts
// 使用 Symbol.for 方法创建/获取全局 Symbol
// 如果 key 不存在，会创建新的；如果已存在，返回已有的
var globalSym1 = Symbol.for("global");
var globalSym2 = Symbol.for("global");

// 相同 key 的 Symbol 是相等的
console.log("全局 Symbol 相等: " + (globalSym1 === globalSym2)); // 全局 Symbol 相等: true

// 获取 Symbol 的 key
console.log("Symbol key: " + Symbol.keyFor(globalSym1)); // Symbol key: global
```

`Symbol()`每次创建新的值，`Symbol.for()`在全局注册表中查找或创建。

### 内置Symbol

JavaScript定义了一些内置Symbol值，用于自定义语言行为。

```ts
// Symbol.iterator 用于定义对象的默认迭代器
var arr = [1, 2, 3];
// 获取数组的迭代器
var iterator = arr[Symbol.iterator]();

// 使用迭代器遍历
console.log("第一个元素: " + iterator.next().value); // 第一个元素: 1
console.log("第二个元素: " + iterator.next().value); // 第二个元素: 2

// Symbol.toStringTag 自定义对象的 toString() 返回值
var obj = {
    [Symbol.toStringTag]: "MyObject"
};
console.log("对象类型: " + obj.toString()); // 对象类型: [object MyObject]
```

### Symbol类型注解

在TypeScript中，Symbol可以作为类型注解，用于表示唯一标识符。

```ts
// Symbol 类型注解
var sym: symbol = Symbol("key");

// 对象的键类型为 symbol，值为 string
var obj: { [key: symbol]: string } = {};

// 使用 Symbol 作为键
obj[sym] = "value";
console.log("Symbol 属性值: " + obj[sym]); // Symbol 属性值: value
```

### Symbol详解小结

Symbol 用于创建唯一标识符，避免属性名冲突：

- `Symbol()` 每次创建唯一值，即使描述相同也不相等
- `Symbol.for(key)` 在全局注册表中查找或创建，同 key 返回同一个 Symbol
- Symbol 属性不会被 `for...in`、`Object.keys()`、`JSON.stringify()` 遍历到
- 适用场景：定义"半私有"属性键、自定义 `Symbol.iterator` / `Symbol.toStringTag` 等内置行为

## 迭代器与生成器

迭代器与生成器是处理集合的重要模式，提供了一种统一的遍历数据的方式，让处理大数据流、无限序列等变得更加简单。

### 可迭代协议

实现了 `Symbol.iterator` 方法的对象是可迭代的。

```ts
// 数组默认可迭代
var arr = [1, 2, 3];
for (var item of arr) {
    console.log("数组元素: " + item);
}
// 输出：
// 数组元素: 1
// 数组元素: 2
// 数组元素: 3

// 字符串默认可迭代
var str = "hello";
for (var char of str) {
    console.log("字符: " + char);
}
// 输出：
// 字符: h
// 字符: e
// 字符: l
// 字符: l
// 字符: o
```

### 自定义可迭代对象

让普通对象实现Symbol.iterator接口，使其可迭代。

```ts
// 创建自定义可迭代对象：范围
var colorPalette = {
    colors: [
        { name: "红色", hex: "#FF0000" },
        { name: "绿色", hex: "#00FF00" },
        { name: "蓝色", hex: "#0000FF" }
    ],
    [Symbol.iterator]: function() {
        return {
            current: 0,
            colors: this.colors,
            //迭代器必须有一个 next() 方法，返回 { done: boolean, value: any } 格式的对象。
            next: function() {
                if (this.current < this.colors.length) {
                    // value 可以是元组（数组）
                    var color = this.colors[this.current++];
                    //执行返回后，this.current自增
                    return {
                        done: false,
                        value: [color.name, color.hex]  // ← 元组
                    };
                }
                return { done: true, value: undefined };
            }
        };
    }
};

// 遍历自定义可迭代对象
for (let _b of colorPalette) {
    let name = _b[0];
    let hex = _b[1];
    console.log(name + ": " + hex);
}
// 输出：
// 红色: #FF0000
// 绿色: #00FF00
// 蓝色: #0000FF

// 更优雅的写法（直接在 for...of 中解构元组）
for (let [name, hex] of colorPalette) {
    console.log(name + ": " + hex);
}
```

### 自定义迭代逻辑

```ts
var myArray = {
    data: [10, 20, 30],
    [Symbol.iterator]: function() {
        return {
            index: 0,
            data: this.data,
            next: function() {
                if (this.index < this.data.length) {
                    return {
                        done: false,
                        value: this.data[this.index++] * 2  
                        // 自定义：每个值翻倍
                    };
                }
                return { done: true, value: undefined };
            }
        };
    }
};

for (var val of myArray) {
    console.log(val);  // 输出：20, 40, 60
}
```

### 生成器函数

使用function*语法创建生成器，使用yield暂停执行并返回值。

```ts
// 生成器函数：使用 function* 语法
function* numberGenerator() {
    yield 1;  // 暂停并返回 1
    yield 2;  // 暂停并返回 2
    yield 3;  // 暂停并返回 3
}

// 创建生成器实例
var gen = numberGenerator();

// 每次调用 next() 都会执行到下一个 yield
console.log("第一个: " + gen.next().value); // 第一个: 1
console.log("第二个: " + gen.next().value); // 第二个: 2
console.log("第三个: " + gen.next().value); // 第三个: 3
console.log("完成: " + gen.next().done);    // 完成: true
```

生成器实际上作用和自定义迭代器相同，且更简洁。

```ts
// 自定义迭代器
var range = {
    from: 1, to: 5,
    [Symbol.iterator]: function() {
        var current = this.from;
        return {
            next: function() {
                if (current <= range.to) {
                    return { done: false, value: current++ };
                }
                return { done: true, value: undefined };
            }
        };
    }
};

// 等价生成器写法 — 简洁很多
function* rangeGenerator(from: number, to: number) {
    for (let i = from; i <= to; i++) {
        yield i;
    }
}
```

### 无限生成器

生成器可以产生无限序列，由于是惰性求值，可以节省内存。

```ts
// 无限数字生成器
// 每次调用只生成一个数字，不会一次性生成所有数字
function* infiniteNumbers() {
    var n = 1;
    while (true) {  // 无限循环
        yield n++;   // 暂停并返回当前值，然后递增
    }
}

var gen = infiniteNumbers();
console.log("第1个: " + gen.next().value);
console.log("第2个: " + gen.next().value);
console.log("第3个: " + gen.next().value);
// 输出：
// 第1个: 1
// 第2个: 2
// 第3个: 3

// 只获取前5个数字
var nums = [];
var iter = infiniteNumbers();
for (var i = 0; i < 5; i++) {
    nums.push(iter.next().value);
    //将值填入数组中
}
console.log("前5个: " + nums);
// 输出：
// 前5个: 1, 2, 3, 4, 5
```

### 委托生成器

使用yield*语法将生成器委托给另一个生成器。

```ts
// 第一个生成器
function* gen1() {
    yield 1;
    yield 2;
}

// 第二个生成器
function* gen2() {
    yield 3;
    yield 4;
}

// 组合生成器：使用 yield*委托
function* combined() {
    yield* gen1();  // 委托给 gen1
    yield* gen2();  // 委托给 gen2
}

// 遍历组合生成器（需将 tsconfig.json 的 target 设为 ES2015+）
for (var num of combined()) {
    console.log("值: " + num);
}
// 输出：
// 值: 1
// 值: 2
// 值: 3
// 值: 4
```

### 生成器类型

生成器的类型注解使用Generator类型。

```ts
// 生成器类型：Generator<yield类型, return类型, next参数类型>
function* idGenerator(): Generator<number, void, unknown> {
    var i = 1;
    while (i <= 3) {
        yield i++;  // yield number 类型
    }
    // return void
}

var gen = idGenerator();
console.log(Array.from(gen));  // 输出：[1, 2, 3]
```

### 迭代器与生成器小结

迭代器与生成器提供统一的遍历抽象，核心概念：

- **可迭代协议**：实现 `Symbol.iterator` 的对象可用 `for...of` 遍历
- **生成器（`function*`）**：用 `yield` 暂停执行，语法比手写迭代器简洁得多
- **惰性求值**：每次 `next()` 才计算下一个值，适合处理大数据流和无限序列
- **委托（`yield*`）**：可将生成器组合，复用已有逻辑
- **类型注解**：`Generator<yield类型, return类型, next参数类型>`

## 映射类型

映射类型允许我们从旧类型创建新类型，通过映射旧类型的属性来创建新类型的属性。

基本语法：
`type 新类型<T> = { [P in keyof T]: T[P] };`

### 内置映射类型

TypeScript内置了一些常用的映射类型：

- Partial：将类型中的某些属性设置为可选。
- Required：将类型中的某些属性设置为必选。
- Readonly：将类型中的所有属性设置为只读。
- Pick：从类型中挑选某些属性。
- Omit: 从类型中排除某些属性。
- Record：创建一个具有指定属性和值的对象类型。
- Exclude：从联合类型中排除某些类型。
- Extract：从联合类型中提取某些类型。
- NonNullable：从类型中排除null和undefined。
- ReturnType：获取函数类型的返回类型。
- Parameters：获取函数类型的参数类型。
- ConstructorParameters：获取构造函数类型的参数类型。
- InstanceType：获取构造函数类型的实例类型。

```ts
interface User { name: string; age: number; email: string; }

//type Partial<T> = { [P in keyof T]?: T[P] };
const user1: Partial<User> = { name: "Alice" }; // ✅ 只需部分属性

//type Required<T> = { [P in keyof T]-?: T[P] };
// 所有属性变为必选
const user2: Required<User> = { name: "Alice", age: 25, email: "a@b.com" }; // 必须提供所有属性

//type Readonly<T> = { readonly [P in keyof T]: T[P] };
const user3: Readonly<User> = { name: "Alice", age: 25, email: "a@b.com" };
// user.name = "Bob"; // ❌ 只读

//type Pick<T, K extends keyof T> = { [P in K]: T[P] };
type UserNameAndEmail = Pick<User, "name" | "email">;
// { name: string; email: string; }

//type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;
type UserWithoutEmail = Omit<User, "email">;
// { name: string; age: number; }

//type Record<K extends keyof any, V> = { [P in K]: V };
type Roles = "admin" | "user" | "guest";
type UserRoles = Record<Roles, User>;
// { admin: User; user: User; guest: User; }

//type Exclude<T, U> = T extends U ? never : T;
type T = Exclude<"a" | "b" | "c", "a" | "b">;  // "c"
type T2 = Exclude<string | number, string>;     // number

//type Extract<T, U> = T extends U ? T : never;
type T3 = Extract<"a" | "b" | "c", "a" | "b">;  // "a" | "b"
type T4 = Extract<string | number, string>;     // string

//type NonNullable<T> = T extends null | undefined ? never : T;
type T5 = NonNullable<string | null | undefined>; // string

//type ReturnType<T> = T extends (...args: any[]) => infer R ? R : any;
type F = () => string;
type R = ReturnType<F>; // string

//type Parameters<T> = T extends (...args: infer P) => any ? P : any;
type F2 = (a: number, b: string) => void;
type P = Parameters<F2>; // [number, string]

//type ConstructorParameters<T extends new (...args: any[]) => any> = T extends new (...args: infer P) => any ? P : never;
type C = new (a: string, b: number) => void;
type P2 = ConstructorParameters<C>; // [string, number]

//type InstanceType<T extends new (...args: any[]) => any> = T extends new (...args: any[]) => infer R ? R : any;
type I = InstanceType<C>; // void
```

### 基础映射类型

基础映射类型通过遍历原始类型的所有键来创建新类型。这是实现 Partial、Readonly 等工具类型的基础。

```ts
// 定义用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户邮箱
    email: string;
}

// 实现 Partial 工具类型
// 遍历 T 的所有键，添加可选修饰符 ?
type Partial<T> = {
    // P 遍历 keyof T 返回的所有键
    // T[P] 获取原类型中对应键的值类型
    [P in keyof T]?: T[P];
};

// 使用 Partial 类型
type PartialUser = Partial<User>;

// PartialUser 类型等同于：
// { id?: number; name?: string; email?: string }

// 可以只提供部分属性
var user: PartialUser = { name: "Alice" };

console.log("部分用户: " + JSON.stringify(user));
```

### 属性修饰符

映射类型支持多种属性修饰符来修改属性的特性。这些修饰符可以组合使用，实现不同的类型转换需求。

```ts
// 定义用户接口
interface User {
    // 用户名
    name: string;
    // 用户年龄
    age: number;
}

// 使用 readonly 映射：将所有属性变为只读
// 添加 readonly 修饰符
type Readonly<T> = {
    readonly [P in keyof T]: T[P];
};

// 使用可选映射：将所有属性变为可选（基础版）
type Optional<T> = {
    [P in keyof T]?: T[P];
};

// 使用 -? 映射：移除可选修饰符（变为必填）
// -? 会移除原有的 ? 修饰符
type Required<T> = {
    [P in keyof T]-?: T[P];
};

// 测试只读类型
var readonlyUser: Readonly<User> = { name: "Alice", age: 25 };
// readonlyUser.age = 30; // 错误：只读属性不能修改

// 测试可选类型
var optionalUser: Optional<User> = { name: "Bob" };

console.log("只读: " + JSON.stringify(readonlyUser));
console.log("可选: " + JSON.stringify(optionalUser));
```

### 键名映射

映射类型还可以使用 as 关键字来重映射键名。这在需要统一修改键名格式时非常有用。

```ts
// 定义用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户年龄
    age: number;
}

// 使用 as 关键字重映射键名
// 为所有键添加前缀
type WithPrefix<T, Prefix extends string> = {
    // 使用模板字面量类型重命名键
    // Capitalize 将首字母大写
    [P in keyof T as `${Prefix}${Capitalize<string & P>}`]: T[P];
};

// 使用 WithPrefix 添加 "user" 前缀
type PrefixedUser = WithPrefix<User, "user">;

// 转换后的类型：
// { userId: number; userName: string; userAge: number }

// 使用带前缀的类型
var user: PrefixedUser = { userId: 1, userName: "Alice", userAge: 25 };

console.log("带前缀: " + JSON.stringify(user));
```

### 键过滤

通过条件类型和映射类型的组合，可以实现键的过滤。这在实现 Omit 等工具类型时非常有用。

```ts
// 定义用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户密码
    password: string;
    // 用户邮箱
    email: string;
}

// 实现 Omit：排除指定键
// 使用条件类型过滤键
type Omit<T, K extends keyof T> = {
    // P 遍历 T 的所有键
    // 如果 P 可以赋值给 K（即在排除列表中），返回 never（不包含）
    // 否则返回 P（保留该键）
    [P in keyof T as P extends K ? never : P]: T[P];
};

// 使用 Omit 排除 password 键
type UserWithoutPassword = Omit<User, "password">;

// 转换后的类型：
// { id: number; name: string; email: string }

// 使用排除 password 后的类型
var user: UserWithoutPassword = { id: 1, name: "Alice", email: "a@b.com" };

console.log("无密码: " + JSON.stringify(user));
```

### 条件映射

映射类型可以与条件类型结合，根据属性类型的不同应用不同的转换。这使得类型转换更加灵活和智能。

```ts
// 定义 API 响应接口
interface APIResponse {
    // 响应数据
    data: string;
    // 错误信息
    error: string;
    // 是否加载中
    isLoading: boolean;
    // 时间戳
    timestamp: number;
}

// 将函数类型转换为 () => void
// 遍历所有属性，根据属性类型进行条件转换
type FunctionToVoid<T> = {
    // 如果 T[P] 是函数类型，转换为 () => void
    // 否则保持原类型不变
    [P in keyof T]: T[P] extends (...args: any[]) => any
        ? () => void
        : T[P];
};

// 使用条件映射
var response: FunctionToVoid<APIResponse> = {
    data: "hello",
    error: "",
    isLoading: false,
    timestamp: Date.now()
};

console.log("响应: " + JSON.stringify(response));
```

### 自定义映射类型

除了使用内置的映射类型，我们还可以自定义映射类型。

```ts
// 自定义：将所有属性值改为 string 类型
type Stringify<T> = { [P in keyof T]: string };

// 自定义：为每个属性添加 null 联合
type Nullable<T> = { [P in keyof T]: T[P] | null };

// 自定义：移除 readonly
type Mutable<T> = { -readonly [P in keyof T]: T[P] };
```

### 映射类型小结

映射类型基于旧类型批量生成新类型，语法为 `{ [P in keyof T]: ... }`：

- **内置常用**：`Partial`（变可选）、`Required`（变必选）、`Readonly`、`Pick`、`Omit`、`Record`
- **类型操作**：`Exclude`/`Extract` 操作联合类型，`ReturnType`/`Parameters` 提取函数类型信息
- **自定义**：可定义 `Nullable<T>`（加 `null`）、`Stringify<T>`（全改 `string`）等
- 是 `interface` 无法实现的能力，属于 TypeScript 独有的高级类型系统

## 类型别名

当代码中多次使用同一个复杂类型时，每次都需要重复书写这个类型，这会导致代码冗余。类型别名可以解决这个问题，它允许我们给一个复杂类型起一个简短的名字。

```ts
// 类型别名：为联合类型定义别名
// ID 可以是字符串或数字
type ID = string | number;

// 类型别名：为对象类型定义别名
// Point 表示一个坐标点
type Point = { x: number; y: number };

// 使用类型别名
var userId: ID = "123";
var productId: ID = 456;

// 使用 Point 类型别名
var point: Point = { x: 10, y: 20 };

console.log("用户ID: " + userId); // 输出: 用户ID: 123
console.log("产品ID: " + productId); // 输出: 产品ID: 456
console.log("坐标: " + JSON.stringify(point)); // 输出: 坐标: {"x":10,"y":20}
```

### 接口和类型别名

type可以定义任何类型（联合类型、元组、函数类型等），而接口主要用于定义对象类型。接口支持声明合并，type不支持。

```ts
// 使用 type 别名定义对象类型
// type 可以定义任何类型，不仅仅是对象
type PersonType = {
    name: string;
    age: number;
};

// 使用接口定义对象类型
// 接口可以声明合并，可以被类实现
interface PersonInterface {
    name: string;
    age: number;
}

// 两者都可以用来声明变量类型
var person1: PersonType = { name: "Alice", age: 25 };
var person2: PersonInterface = { name: "Bob", age: 30 };

console.log("PersonType: " + JSON.stringify(person1)); // 输出: PersonType: {"name":"Alice","age":25}
console.log("PersonInterface: " + JSON.stringify(person2)); // 输出: PersonInterface: {"name":"Bob","age":30}
```

### 类型别名和联合类型

类型别名非常适合定义联合类型，因为它们可以包含多个类型，并且可以给这些类型起一个简短的名字。

```ts
// 联合类型别名：定义状态的可能值
// Status 只能是这三个字符串字面量之一
type Status = "pending" | "success" | "error";

// 联合类型别名：定义多种可能的返回类型
// Result 可以是字符串、数字或布尔值
type Result = string | number | boolean;

// 使用联合类型别名
function getStatus(status: Status): void {
    console.log("状态: " + status);
}

getStatus("success");
getStatus("error");

// 使用 Result 类型
var result: Result = "hello";
result = 42;
console.log("结果: " + result);
```

### 类型别名和元组

类型别名可以用来定义元组类型，元组类型是一种特殊的数组类型，它具有固定的长度和每个位置上的类型。

```ts
// 元组类型别名：定义坐标
// Coordinate 是一个包含两个数字的元组
type Coordinate = [number, number];

// 元组类型别名：定义姓名和年龄
// NameAge 是一个字符串和数字的元组
type NameAge = [string, number];

// 使用元组类型别名
var coord: Coordinate = [10, 20];
var person: NameAge = ["Alice", 25];

console.log("坐标: " + coord);
console.log("信息: " + person[0] + ", " + person[1]);
```

### 类型别名与函数

类型别名可以简化函数类型的声明。

```ts
// 函数类型别名：定义回调函数类型
// Callback 接受一个字符串参数，返回 void
type Callback = (result: string) => void;

// 函数类型别名：定义数学运算函数类型
// MathOperation 接受两个数字参数，返回数字
type MathOperation = (a: number, b: number) => number;

// 使用函数类型别名
var add: MathOperation = (a, b) => a + b;
 // 等价于直接定义箭头函数类型
// var add: (a: number, b: number) => number = (a, b) => { return a + b; };
var multiply: MathOperation = function(a, b) { return a * b; };

console.log("加法: " + add(2, 3));
console.log("乘法: " + multiply(4, 5));
```

### 类型别名与泛型

```ts
// 泛型类型别名：定义结果类型
// Result<T> 是一个泛型类型，T 是成功时的数据类型
type Result<T> = { success: boolean; data?: T; error?: string };

// 泛型类型别名：定义键值对类型
// Pair<K, V> 有两个类型参数
type Pair<K, V> = { key: K; value: V };

// 使用泛型类型别名
var result: Result<string> = { success: true, data: "Hello" };
var pair: Pair<string, number> = { key: "age", value: 25 };

console.log("结果: " + JSON.stringify(result)); // 输出: 结果: {"success":true,"data":"Hello"}
console.log("键值对: " + JSON.stringify(pair)); // 输出: 键值对: {"key":"age","value":25}
```

### 类型别名和映射类型

```ts
// 映射类型：将所有属性变为只读
// Readonly<T> 遍历 T 的所有属性并添加 readonly
type Readonly<T> = { readonly [P in keyof T]: T[P] };

// 映射类型：将所有属性变为可选
// Partial<T> 遍历 T 的所有属性并添加 ?
type Partial<T> = { [P in keyof T]?: T[P] };

// 定义用户接口
interface User {
    name: string;
    age: number;
}

// 使用映射类型别名
var readonlyUser: Readonly<User> = { name: "Alice", age: 25 };
// readonlyUser.name = "Bob"; // 错误：只读属性不能修改

var partialUser: Partial<User> = { name: "Bob" };

console.log("只读用户: " + JSON.stringify(readonlyUser));
console.log("部分用户: " + JSON.stringify(partialUser));
```

### 类型别名小结

类型别名（`type`）给复杂类型起名，消除重复书写：

- 可定义任何类型：联合类型、元组、函数类型、对象类型、泛型类型
- 与 `interface` 的核心区别：`type` 不能声明合并，但可描述联合/元组/映射类型
- `type` 更适合：联合类型、元组、函数签名、组合已有类型
- `interface` 更适合：需要被类实现、需要声明合并、纯对象类型

## 字面量类型详解

字面量类型是TypeScript中一种表示特定值而不是通用类型的类型。字面量类型是基础类型的子类型，如 `"up"` 是 `string` 的子类型，只能赋值为特定的字符串。

### 字符串字面量类型

字符串字面量类型将变量的取值限制为特定的字符串。

```ts
// 定义字符串字面量类型
// direction 只能是四个特定值之一
var direction: "up" | "down" | "left" | "right";

// 正确：赋值为字面量类型中的值
direction = "up";

// 错误：赋值不在列表中的值
// direction = "upup"; // 编译错误！

console.log("方向: " + direction);

// 使用类型别名创建可复用的字面量类型
// 定义状态类型：只能是三个固定值之一
type Status = "pending" | "active" | "completed";

// 使用类型别名
var currentStatus: Status = "active";
console.log("状态: " + currentStatus);
```

### 数字字面量类型

数字字面量类型将变量的取值限制为特定的数字。

```ts
// 定义数字字面量类型
// code 只能是三个特定值之一
var code: 200 | 404 | 500;

// 正确：赋值 200
code = 200;

// 错误：赋值不在列表中的值
// code = 301; // 编译错误！

console.log("状态码: " + code);

// 使用数字字面量模拟枚举
// 定义一周的天数
type Weekday = 1 | 2 | 3 | 4 | 5 | 6 | 7;

// 只能赋值 1-7 之间的数字
var today: Weekday = 1;
console.log("今天是星期: " + today);
```

### 布尔字面量类型

布尔字面量类型将变量的取值限制为特定的布尔值。实际上，TypeScript 中的 boolean 类型就是 true | false 的别名。

```ts
// 定义布尔字面量类型
// isActive 只能是 true 或 false
var isActive: true | false;

// 赋值 true
isActive = true;

// 实际上 boolean 就是 true | false 的联合类型
// 所以这两种写法是等价的
var flag: boolean = true;

console.log("是否激活: " + isActive);
```

### 对象字面量类型

对象字面量类型可以定义对象的结构，并可以使用 readonly 修饰符将属性设为只读。这在需要创建不可变对象时非常有用。

```ts
// 定义对象字面量类型
// 指定对象的结构和属性类型
type Point = {
    // 点的 x 坐标
    x: number;
    // 点的 y 坐标
    y: number;
};

// 使用对象字面量类型
var p: Point = { x: 10, y: 20 };
console.log("点: " + JSON.stringify(p));

// 定义只读对象类型
// 使用 readonly 修饰符将属性设为只读
type ReadonlyPoint = {
    // 只读的 x 坐标
    readonly x: number;
    // 只读的 y 坐标
    readonly y: number;
};

// 使用只读对象类型
var rp: ReadonlyPoint = { x: 1, y: 2 };

// 尝试修改只读属性会报错
// rp.x = 3; // 编译错误：只读属性不能修改

console.log("只读点: " + JSON.stringify(rp));
```

### 字面量类型与类型推断

TypeScript 会根据变量的声明方式自动推断为字面量类型。使用 const 声明的变量会被推断为具体的字面量类型，而不是宽泛的类型。

```ts
// 使用 var 声明并赋值
// TypeScript 会推断为宽泛类型
var name = "Alice";    // 类型: string（因为 var 可以重新赋值）
var age = 25;          // 类型: number
var enabled = true;    // 类型: boolean

console.log("名字: " + name + ", 年龄: " + age + ", 启用: " + enabled);

// 使用 as const 创建深度只读字面量
// 数组变为只读元组，值为字面量类型
var colors = ["red", "green", "blue"] as const;

// colors 的类型变为：
// readonly ["red", "green", "blue"]

// 访问数组元素
console.log("颜色: " + colors[0]);

// 尝试修改会报错
// colors[0] = "yellow"; // 编译错误：只读
```

### 模版字面量类型

模板字面量类型使用 JavaScript 模板字符串的语法来创建新的类型。它允许我们动态地生成字符串类型。

```ts
// 使用模板字面量类型定义事件名称
// `on${string}` 表示以 "on" 开头的任意字符串
type EventName = `on${string}`;

// 使用模板字面量类型定义处理器名称
// Capitalize 将字符串首字母大写
type Handler = `handle${Capitalize<string>}`;

// 使用模板字面量类型
var event: EventName = "onClick";
var handler: Handler = "handleSubmit";

console.log("事件: " + event);
console.log("处理器: " + handler);

// 错误：不符合模板格式
// var e: EventName = "click"; // 编译错误：不是以 "on" 开头
```

### 字面量类型详解小结

字面量类型将变量限制为**具体的值**而非宽泛的类型：

- **字符串/数字/布尔字面量**：联合使用可模拟枚举，如 `"up" | "down"`
- **对象字面量类型**：搭配 `readonly` 可创建不可变对象结构
- **`as const`**：将值推断为最窄的只读字面量类型，是定义常量的利器
- **模板字面量类型**：用 `` `on${string}` `` 语法动态生成字符串类型，实现类型级模式匹配
- 字面量类型是 TypeScript 从"类型约束"走向"值约束"的关键能力

## 形状约束

形状约束（Shape Constraint）是 TypeScript 结构化类型系统的核心体现 —— 不关心"你是谁"，只关心"你有什么属性和方法"。它通过内联对象类型字面量来描述一个对象应该具备的结构。

### 形状约束语法

形状约束的语法就是直接在类型位置写一个对象结构：

```ts
// 基本形式：{ 属性名: 类型; 方法名(参数): 返回值 }
{ name: string; age: number }          // 要求有 name 和 age
{ serialize(): string }                // 要求有 serialize() 方法
{ x: number; y: number }              // 要求有 x 和 y 坐标
```

### 函数参数约束

最常见的用法是约束函数参数必须符合某种形状：

```ts
// 约束：传入的对象必须长得像有 name 和 age
function greet(person: { name: string; age: number }) {
    console.log(person.name + " 今年 " + person.age + " 岁");
}

// 正确：传入的对象完全匹配形状
greet({ name: "Alice", age: 25 });
// 输出：Alice 今年 25 岁

// 错误：缺少 age 属性
// greet({ name: "Bob" });

// 错误：age 类型不对
// greet({ name: "Bob", age: "二十" });
```

注意：TypeScript 使用**结构化类型**检查，只要对象具备所需的属性和方法即可，不需要显式声明实现某个接口：

```ts
function logPoint(p: { x: number; y: number }) {
    console.log("(" + p.x + ", " + p.y + ")");
}

// 不要求 implement 任何接口，有 x 和 y 就行
logPoint({ x: 10, y: 20 });                // ✅ 输出：(10, 20)
var point3D = { x: 5, y: 5, z: 5 };
logPoint(point3D);                          // ✅ 输出：(5, 5)，多出的 z 会被忽略
```

### 泛型约束（重要）

形状约束在泛型中非常重要，用于限制泛型参数必须具备某些成员：

```ts
// 约束 T 必须具有 .length 属性（number 类型）
function getLength<T extends { length: number }>(item: T): number {
    return item.length;
}

getLength("hello");           // ✅ string 有 length
getLength([1, 2, 3]);        // ✅ 数组有 length
getLength({ length: 10 });   // ✅ 对象有 length
// getLength(123);            // ❌ number 没有 length
```

这在 Mixin 模式中尤为重要，外层用于约束类型，内层用于约束实例形状：

```ts
// Constructor 类型：描述"可以用 new 实例化"的类
type Constructor<T = {}> = new (...args: any[]) => T;

// 泛型参数 TBase 受 Constructor 约束
// 翻译：Base 必须是一个类，且其可以用 new 实例化
function Serializable<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        serialize(): string {
            return JSON.stringify(this);   // 将当前实例的所有属性序列化为 JSON
        }
    };
}

// 泛型参数 TBase 受 Constructor<{ serialize(): string }> 约束
// 翻译：Base 必须是一个类，且其实例必须有 serialize() 方法
function Loggable<TBase extends Constructor<{ serialize(): string }>>(Base: TBase) {
    return class extends Base {
        log(): void {
            console.log("[LOG]", this.serialize());
        }
    };
}
```

这里的 `Constructor<{ serialize(): string }>` 是"容器 + 内容"的双重约束：

- `Constructor` 保证参数是类（能用 `new`）。
- `{ serialize(): string }` 保证实例上有 `serialize()` 方法。

### 函数返回值约束

```ts
// 约束函数必须返回符合形状的对象
function createPoint(): { x: number; y: number } {
    return { x: 10, y: 20 };

// 约束返回的对象包含方法
function createHandler(): { onClick(): void; onHover(): void } {
```

### 可选属性

使用 `?` 标记可选属性：

```ts
// age 是可选的，可以不传
function register(user: { name: string; age?: number }) {

```


### 形状约束 vs 接口（interface）

| 对比维度 | 形状约束（内联） | 接口（interface） |
| :--- | :--- | :--- |
| 写法 | `{ a: number; b: string }` | `interface Foo { a: number; b: string }` |
| 可复用性 | 匿名，无法复用 | 有名字，可导出、可继承 |
| 适合场景 | 一次性约束、泛型约束 | 多处使用、需要扩展 |
| 泛型中使用 | 可直接内联，简洁 | 也可以，但更啰嗦 |

```ts
// 内联形状约束：适合"只用一次"
function f(obj: { x: number; y: number }) {}

// 接口：适合"多处复用"
interface Point {
    x: number;
    y: number;
}
function f1(obj: Point) {}
function f2(obj: Point) {}
class Circle implements Point {
    x = 0;
    y = 0;
}
```

### 形状约束小结

形状约束是 TypeScript 中最灵活的类型描述方式之一：

- **函数参数**：临时描述参数结构，无需定义命名接口
- **泛型约束**：精确控制泛型参数必须具备的成员，如 `T extends { length: number }`
- **"容器 + 内容"模式**：在 `Constructor<{ serialize(): string }>` 中，外层约束类型，内层约束实例形状
- **与接口互补**：接口适合复用和继承，形状约束适合一次性、泛型中的简洁描述

## 类型守卫

类型守卫（Type Guards）是 TypeScript 中非常重要的一种类型缩小机制。它允许开发者在运行时通过特定的条件检查，让 TypeScript 编译器能够准确推断出变量的具体类型。通过类型守卫，我们可以安全地访问联合类型变量中特定类型的属性和方法。

### typeof类型守卫

typeof 是最常用的类型守卫，用于检查原始类型（string、number、boolean 等）。它返回一个字符串，表示值的类型。

```ts
// 定义一个接收联合类型的函数
// 参数 value 可能是字符串或数字
function printValue(value: string | number): void {
    // 使用 typeof 检查类型
    // 在 if 条件为 true 时，TypeScript 会自动将 value 缩小为 string 类型
    if (typeof value === "string") {
        // 此时 TypeScript 知道 value 是 string
        // 可以安全地访问字符串的 length 属性
        console.log("字符串长度: " + value.length);
    } else {
        // 进入 else 分支时，TypeScript 知道 value 不是 string
        // 只能是 number 类型
        // 可以安全地进行数学运算
        console.log("数字翻倍: " + (value * 2));
    }
}

// 测试调用
printValue("hello");  // 输出：字符串长度: 5
printValue(42);       // 输出：数字翻倍: 84
```

typeof 支持的类型：

- `"string"` - 字符串类型
- `"number"` - 数字类型
- `"bigint"` - 大整数类型
- `"boolean"` - 布尔类型
- `"symbol"` - 符号类型
- `"undefined"` - 未定义类型
- `"object"` - 对象类型（包括 null）
- `"function"` - 函数类型

typeof 对于数组和 null 都会返回 "object"，所以需要使用其他类型守卫来区分它们。

### instanceof类型守卫

instanceof 用于检查对象是否是某个类的实例。它通过检查对象的原型链来判断类型。

```ts
// 定义 Dog 类
class Dog {
    // 狗的叫声方法
    bark(): void {
        console.log("汪汪汪!");
    }
}

// 定义 Cat 类
class Cat {
    // 猫的叫声方法
    meow(): void {
        console.log("喵喵喵!");
    }
}

// 接收联合类型的函数
function makeSound(animal: Dog | Cat): void {
    // 使用 instanceof 检查 animal 是 Dog 还是 Cat
    // 在 if 条件为 true 时，TypeScript 将 animal 缩小为 Dog 类型
    if (animal instanceof Dog) {
        // 此时可以调用 Dog 特有的方法
        animal.bark();
    } else {
        // else 分支中，TypeScript 将 animal 缩小为 Cat 类型
        animal.meow();
    }
}

// 测试调用
makeSound(new Dog());  // 输出：汪汪汪!
makeSound(new Cat());  // 输出：喵喵喵!
```

instanceof 检查的是对象的原型链，因此它只能用于类实例，不能用于接口和类型别名。

### 自定义类型守卫

当内置的 typeof 和 instanceof 无法满足需求时，可以创建自定义类型函数守卫。自定义类型守卫使用 `value is Type` 的返回类型语法。

```ts
// 检查是否为字符串类型
// 返回类型使用 "value is Type" 格式
// 当函数返回 true 时，参数类型就是 string
// 当函数返回 false 时，参数类型就不是 string
function isString(value: any): value is string {
    // 使用 typeof 检查是否为字符串
    return typeof value === "string";
}

// 检查是否为数字类型
function isNumber(value: any): value is number {
    return typeof value === "number";
}

// 检查是否为数组类型
function isArray(value: any): value is any[] {
    return Array.isArray(value);
}

// 处理值的函数
function processValue(value: string | number | any[]): void {
    // 使用自定义守卫进行类型检查
    if (isString(value)) {
        // TypeScript 知道 value 是 string 类型
        // 可以调用 toUpperCase() 方法
        console.log("字符串转大写: " + value.toUpperCase());
    } else if (isNumber(value)) {
        // TypeScript 知道 value 是 number 类型
        // 可以调用 toFixed() 方法
        console.log("数字格式化: " + value.toFixed(2));
    } else if (isArray(value)) {
        // TypeScript 知道 value 是数组类型
        console.log("数组长度: " + value.length);
    }
}

// 测试调用
processValue("hello");             // 输出：字符串转大写: HELLO
processValue(3.14159);             // 输出：数字格式化: 3.14
processValue([1, 2, 3, 4, 5]);    // 输出：数组长度: 5
```

### in操作符类型守卫

in 操作符可以检查对象是否包含某个属性。在条件判断中使用 in，TypeScript 会自动缩小对象的类型范围。

```ts
// 定义两个接口，它们有不同的属性
interface A {
    a: string;  // 只有属性 a
}

interface B {
    b: number;  // 只有属性 b
}

// 接收联合类型的函数
function process(obj: A | B): void {
    // 使用 in 检查对象是否包含属性 "a"
    if ("a" in obj) {
        // 在 if 分支中，TypeScript 知道 obj 包含属性 a
        // 因此 obj 的类型被缩小为 A
        console.log("A 的属性 a: " + obj.a);
    } else {
        // else 分支中，obj 不包含属性 a
        // TypeScript 知道 obj 只能是 B 类型
        // 因此可以安全访问属性 b
        console.log("B 的属性 b: " + obj.b);
    }
}

// 测试调用
process({ a: "hello" });  // 输出：A 的属性 a: hello
process({ b: 42 });       // 输出：B 的属性 b: 42
```

### 可辨识联合与类型守卫

可辨识联合是一种强大的模式，它通过一个公共的"标识"属性来区分联合类型成员。结合 switch 语句或 if 判断，可以实现完整的类型守卫。

```ts
// 定义圆形接口，使用 kind 属性作为标识
interface Circle {
    kind: "circle";       // 标识字段：值为 "circle"
    radius: number;       // 半径
}

// 定义矩形接口
interface Rectangle {
    kind: "rectangle";    // 标识字段：值为 "rectangle"
    width: number;        // 宽度
    height: number;      // 高度
}

// 定义三角形接口
interface Triangle {
    kind: "triangle";    // 标识字段：值为 "triangle"
    base: number;        // 底边
    height: number;      // 高度
}

// 定义联合类型
type Shape = Circle | Rectangle | Triangle;

// 计算面积的函数
function getArea(shape: Shape): number {
    // 使用 switch 语句进行类型守卫
    // 根据 kind 属性的值，TypeScript 会自动缩小类型
    switch (shape.kind) {
        case "circle":
            // shape 被缩小为 Circle 类型
            // 可以访问 radius 属性
            return Math.PI * shape.radius ** 2;

        case "rectangle":
            // shape 被缩小为 Rectangle 类型
            // 可以访问 width 和 height 属性
            return shape.width * shape.height;

        case "triangle":
            // shape 被缩小为 Triangle 类型
            return 0.5 * shape.base * shape.height;
    }
}

// 测试调用
var circle = { kind: "circle" as const, radius: 5 };
var rectangle = { kind: "rectangle" as const, width: 4, height: 6 };
var triangle = { kind: "triangle" as const, base: 3, height: 4 };

console.log("圆形面积: " + getArea(circle).toFixed(2));    // 圆形面积: 78.54
console.log("矩形面积: " + getArea(rectangle));            // 矩形面积: 24
console.log("三角形面积: " + getArea(triangle));           // 三角形面积: 6
```

可辨识联合是 TypeScript 中最推荐使用的模式之一。它通过一个公共的字面量属性（通常是 kind 或 type）来区分不同的类型成员，使代码既类型安全又易于维护。

### null和undefined检查

处理可能为 null 或 undefined 的值时，直接的相等性检查也是有效的类型守卫。

```ts
// 定义一个可能为 null 的函数参数
function getLength(str: string | null): number {
    // 直接检查 str 不等于 null
    // 在条件为 true 时，TypeScript 知道 str 不是 null
    // 此时可以安全地访问 str 的属性
    if (str !== null) {
        return str.length;
    }

    // null 情况的处理
    return 0;
}

// 调用测试
console.log(getLength("hello"));  // 正常字符串
console.log(getLength(null));      // 传入 null
```

### 真值缩小

除了显式的类型检查，TypeScript 还会通过真值断言（Truthiness）来缩小类型范围。

```ts
// 可能为 undefined 的字符串
function greet(name?: string): string {
    // 使用短路运算符：如果 name 为 undefined 或空字符串，使用默认值
    // 在 && 后的代码块中，TypeScript 知道 name 一定有值
    return name && "Hello, " + name;
}

// 测试
console.log(greet("RUNOOB")); // Hello, RUNOOB
console.log(greet());         // undefined（短路返回）
```

### 类型守卫小结

类型守卫是 TypeScript **类型缩小**（Type Narrowing）的核心机制，通过运行时检查让编译器在条件分支内自动推断出更精确的类型：

- **`typeof`**：用于原始类型检查（`"string"`、`"number"` 等），注意数组和 `null` 都返回 `"object"`
- **`instanceof`**：用于类实例检查，通过原型链判断，不能用于接口和类型别名
- **`in` 操作符**：检查对象是否包含某个属性，适合区分不同形状的接口
- **自定义守卫**：使用 `value is Type` 返回类型语法，封装可复用的类型检查逻辑
- **可辨识联合**：通过公共字面量属性（如 `kind`）配合 `switch` 实现穷尽性检查，是最推荐的模式
- **真值缩小**：利用 `&&`、`||`、`if (value)` 等真值判断自动排除 `null`/`undefined`/`""`/`0` 等假值
- **选择原则**：优先使用 `typeof`/`instanceof` 等内置守卫，复杂逻辑用自定义守卫封装，多成员联合类型用可辨识联合

## 工具类型

### Partial - 可选属性

Partial 将类型 T 的所有属性设置为可选，常用于创建部分更新对象或处理表单数据。

```ts
// 定义用户接口，包含必填属性
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户邮箱
    email: string;
}

// Partial：将所有属性变为可选
// 转换后的类型所有属性都是可选的
type PartialUser = Partial<User>;

// 使用 PartialUser 类型
// 可以只提供部分属性，不需要全部提供
var user: PartialUser = { name: "Alice" };

console.log("部分用户: " + JSON.stringify(user)); // 输出: 部分用户: {"name":"Alice"}
```

### Required - 必填属性

Required 与 Partial 相反，将所有可选属性设置为必填。当需要确保对象包含所有属性时使用。

```ts
// 定义配置接口，属性都是可选的
interface Config {
    // 服务器地址
    host?: string;
    // 端口号
    port?: number;
}

// Required：将所有可选属性变为必填
// 转换后的类型所有属性都是必填的
type RequiredConfig = Required<Config>;

// 使用 RequiredConfig 类型
// 必须提供所有属性
var config: RequiredConfig = { host: "localhost", port: 8080 };

console.log("配置: " + JSON.stringify(config)); // 输出: 配置: {"host":"localhost","port":8080}
```

### Readonly - 只读属性

Readonly 将所有属性设置为只读，适用于创建后不可修改的对象。

```ts
// 定义用户接口
interface User {
    // 用户名
    name: string;
    // 用户年龄
    age: number;
}

// Readonly：将所有属性变为只读
// 转换后的类型所有属性都不能修改
type ReadonlyUser = Readonly<User>;

// 创建只读用户对象
var user: ReadonlyUser = { name: "Alice", age: 25 };

// 尝试修改只读属性会报错
// user.name = "Bob"; // 错误：只读属性不能修改

console.log("只读用户: " + JSON.stringify(user)); // 输出: 只读用户: {"name":"Alice","age":25}
```

### Pick - 选择属性

Pick 从类型 T 中选择指定的属性 K 组成新类型。当你只需要某个类型的部分属性时使用。

```ts
// 定义完整的用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户邮箱
    email: string;
    // 用户密码
    password: string;
}

// Pick：选择指定的属性组成新类型
// 从 User 中选择 id 和 name 属性
type UserBasicInfo = Pick<User, "id" | "name">;

// 使用选择后的类型
var user: UserBasicInfo = { id: 1, name: "Alice" };

console.log("用户基本信息: " + JSON.stringify(user)); // 输出: 用户基本信息: {"id":1,"name":"Alice"}
```

### Omit - 排除属性

Omit 从类型 T 中排除指定的属性 K，返回剩余属性组成的新类型。与 Pick 相反，用于删除不需要的属性。

```ts
// 定义完整的用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户邮箱
    email: string;
    // 用户密码
    password: string;
}

// Omit：排除指定的属性
// 从 User 中排除 password 属性
type UserWithoutPassword = Omit<User, "password">;

// 使用排除后的类型
var user: UserWithoutPassword = { id: 1, name: "Alice", email: "a@b.com" };

console.log("无密码用户: " + JSON.stringify(user)); // 输出: 无密码用户: {"id":1,"name":"Alice","email":"a@b.com"}
```

### Record - 构建映射类型

Record 构造一个对象类型，键的类型为 K，值的类型为 T。常用于创建键值对映射、字典类型等。

```ts
// 定义角色类型
type Role = "admin" | "user" | "guest";

// Record：构造对象类型
// 键为 Role 类型，值为字符串数组类型
type RolePermissions = Record<Role, string[]>;

// 使用 Record 创建权限映射
var permissions: RolePermissions = {
    // 管理员拥有所有权限
    admin: ["read", "write", "delete"],
    // 普通用户拥有读写权限
    user: ["read", "write"],
    // 访客只有读权限
    guest: ["read"]
};

console.log("管理员权限: " + permissions.admin); // 管理员权限: read,write,delete
console.log("访客权限: " + permissions.guest);   // 访客权限: read
```

### Exclude - 排除类型

Exclude 从类型 T 中排除可以赋值给类型 U 的类型。主要用于联合类型，排除特定的类型成员。

```ts
// 定义联合类型，包含 a、b、c、d
type T = "a" | "b" | "c" | "d";

// Exclude：从 T 中排除指定类型
// 排除 "a"、"b"、"c"，只保留 "d"
type NonABC = Exclude<T, "a" | "b" | "c">;

// 使用排除后的类型，只能赋值 "d"
var value: NonABC = "d";

console.log("值: " + value); // 输出: 值: d
```

### Extract - 提取类型

Extract 与 Exclude 相反，从类型 T 中提取可以赋值给类型 U 的类型。用于筛选联合类型中的特定成员。

```ts
// 定义混合联合类型，包含字符串和数字
type T = "a" | "b" | "c" | 1 | 2 | 3;

// Extract：从 T 中提取指定类型
// 提取所有字符串类型："a"、"b"、"c"
type Letters = Extract<T, string>;

// 使用提取后的类型
var letter: Letters = "a";

console.log("字母: " + letter); // 输出: 字母: a
```

### NonNullable - 排除 null 和 undefined

NonNullable 从类型 T 中排除 null 和 undefined。确保类型不包含空值时使用。

```ts
// 定义混合类型，包含字符串、null、undefined、数字
type T = string | null | undefined | number;

// NonNullable：排除 null 和 undefined
// 转换后只保留 string 和 number
type NotNull = NonNullable<T>;

// 使用非空类型
var value: NotNull = "hello";
value = 42;

// 尝试赋值 null 会报错
// value = null; // 错误：不能赋值 null

console.log("值: " + value); // 输出: 值: 42
```

### ReturnType - 获取返回类型

ReturnType 获取函数类型 T 的返回类型。常用于从已存在的函数中提取返回类型。

```ts
// 定义获取用户的函数
function getUser() {
    return { name: "Alice", age: 25 };
}

// 定义获取配置的函数
function getConfig() {
    return { host: "localhost", port: 8080 };
}

// ReturnType：获取函数的返回类型
// 提取 getUser 函数的返回类型
type UserType = ReturnType<typeof getUser>;
// 提取 getConfig 函数的返回类型
type ConfigType = ReturnType<typeof getConfig>;

// 使用提取的返回类型创建对象
var user: UserType = { name: "Bob", age: 30 };
var config: ConfigType = { host: "example.com", port: 3000 };

console.log("用户: " + JSON.stringify(user));   // 输出: 用户: {"name":"Bob","age":30}
console.log("配置: " + JSON.stringify(config)); // 输出: 配置: {"host":"example.com","port":3000}
```

### 工具类型小结

TypeScript 内置工具类型基于**映射类型**和**条件类型**实现，用于快速变换已有类型：

**对象属性操作：**

- `Partial<T>` — 所有属性变可选
- `Required<T>` — 所有属性变必填
- `Readonly<T>` — 所有属性变只读
- `Pick<T, K>` — 从 T 中选出 K 指定的属性
- `Omit<T, K>` — 从 T 中排除 K 指定的属性
- `Record<K, V>` — 构造键为 K、值为 V 的对象类型

**联合类型操作：**

- `Exclude<T, U>` — 从联合类型 T 中排除 U
- `Extract<T, U>` — 从联合类型 T 中提取 U
- `NonNullable<T>` — 排除 null 和 undefined

**函数类型操作：**

- `ReturnType<T>` — 提取函数返回值类型
- `Parameters<T>` — 提取函数参数类型（元组）

**选择原则**：`Pick`/`Omit` 操作对象属性，`Exclude`/`Extract` 操作联合类型，`ReturnType`/`Parameters` 从函数中提取类型信息。

## 条件类型

### 基本语法

条件类型使用三元表达式的语法，在类型层面进行条件判断。这使得我们可以根据输入类型动态地计算返回类型。

```ts
// 条件类型语法：T extends U ? X : Y
// 如果 T 是字符串类型，返回 true，否则返回 false
type IsString<T> = T extends string ? true : false;

// 使用条件类型
// string extends string 为 true，所以 A 类型是 true
type A = IsString<string>;
// number extends string 为 false，所以 B 类型是 false
type B = IsString<number>;

// 实际用法：根据条件类型决定函数返回类型
// 如果是 string 就返回大写，否则返回原值
function process<T extends string | number>(value: T): IsString<T> extends true ? string : number {
    if (typeof value === "string") {
        return value.toUpperCase() as any;  // 字符串 → 大写
    }
    return value * 2 as any;  // 数字 → 翻倍
}

var result1 = process("hello");  // "HELLO"（类型推断为 string）
var result2 = process(5);        // 10（类型推断为 number）

console.log("处理字符串: " + result1); // 处理字符串: HELLO
console.log("处理数字: " + result2);   // 处理数字: 10
```

### 类型过滤

条件类型最常见的应用之一是过滤类型。例如，可以创建一个类型来排除 null 和 undefined。

```ts
// 使用条件类型实现 NonNullable
// 如果 T 是 null 或 undefined，返回 never（空类型），否则返回 T 本身
type NonNullable<T> = T extends null | undefined ? never : T;

// 使用 NonNullable 类型
// string 不是 null/undefined，所以类型是 string
type A = NonNullable<string>;
// null 是 null/undefined，所以类型是 never
type B = NonNullable<null>;
// undefined 是 null/undefined，所以类型是 never
type C = NonNullable<undefined>;

// 验证类型
var a: A = "hello";
console.log("非空: " + a);
```

### infer关键字

`infer` 关键字只能在**条件类型**的 `extends` 子句中使用，用于在类型层面声明一个待推断的类型变量。当条件类型匹配成功时，TypeScript 会自动从输入类型中提取出对应位置的类型并赋值给 `infer` 声明的变量。

基本语法：

```ts
type SomeType<T> = T extends SomePattern<infer R> ? R : never;
//                          └─ infer 出现在匹配模式的某个位置 ─┘
```

`infer R` 放在 TypeScript **能够推断出类型的位置**，常见的位置包括：

- **函数返回值**位置：`(...args: any[]) => infer R`
- **数组元素**位置：`(infer U)[]`
- **泛型参数**位置：`Promise<infer T>`、`Array<infer E>`
- **构造函数参数**位置：`new (...args: infer P) => any`

```ts
// 使用 infer 推导函数的返回类型
// 如果 T 是函数类型，返回推断出的返回类型 R，否则返回 never
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// 定义一个返回用户对象的函数
function getUser() {
    return { name: "Alice" };
}

// 定义一个返回数字的函数
function getNumber() {
    return 42;
}

// 使用 ReturnType 获取函数返回类型
// 推导为 { name: string }
type R1 = ReturnType<typeof getUser>;
// 推导为 number
type R2 = ReturnType<typeof getNumber>;

// 使用推导出的类型
var r1: R1 = { name: "Bob" };
var r2: R2 = 100;

console.log("用户: " + JSON.stringify(r1));
console.log("数字: " + r2);
```

### 分布条件类型

当条件类型的泛型参数是联合类型时，会自动进行"分布"处理。也就是说，条件会对联合类型中的每个成员分别执行，然后合并结果。

```ts
// ToArray 会将类型 T 转换为数组类型
// 当 T 是联合类型时，会自动分布处理每个类型
type ToArray<T> = T extends any ? T[] : never;

// 联合类型会自动分布
// string | number 会分布为：ToArray<string> | ToArray<number>
// 即：string[] | number[]
type StrOrNum = ToArray<string | number>;
// 结果类型：string[] | number[]

// 可以赋值 string[]
var arr1: StrOrNum = ["hello", "world"];
// 也可以赋值 number[]
var arr2: StrOrNum = [1, 2, 3];

console.log("字符串数组: " + arr1);  // 输出：字符串数组: hello,world
console.log("数字数组: " + arr2);    // 输出：数字数组: 1,2,3
```

### 条件类型与映射类型结合

条件类型可以与映射类型结合，创建强大的类型转换工具。这种组合是实现 TypeScript 内置工具类型的基础。

```ts
// 定义用户接口
interface User {
    // 用户 ID
    id: number;
    // 用户名
    name: string;
    // 用户邮箱
    email: string;
}

// 使用映射类型和条件类型实现 Partial
// 遍历 T 的所有属性，添加可选修饰符 ?
type Partial<T> = {
    [P in keyof T]?: T[P];
};

// 使用映射类型和条件类型实现 Required
// 遍历 T 的所有属性，移除可选修饰符 -?
type Required<T> = {
    [P in keyof T]-?: T[P];
};

// 使用 Partial：所有属性变为可选
var partial: Partial<User> = { name: "Alice" };

// 使用 Required：将 Partial<User> 的属性变为必填
// 需要先有 Partial<User> 类型
type RequiredUser = Required<Partial<User>>;
var required: RequiredUser = { name: "Bob", id: 1 };

console.log("可选: " + JSON.stringify(partial));
console.log("必填: " + JSON.stringify(required));
```

### 高级示例：类型检查

条件类型还可以用于实现更复杂的类型检查。

```ts
// 检查类型是否为 any
// any 与任何类型交叉都会得到 any，0 extends any 为 true
type IsAny<T> = 0 extends (1 & T) ? true : false;

// 测试 IsAny
// any 是特殊类型，与任何类型交叉都返回 any
type A = IsAny<any>;       // true
// string 不是 any
type B = IsAny<string>;    // false

console.log("any 是 any?: " + A);
console.log("string 是 any?: " + B);

// 检查类型是否可以赋值
// 如果 T 可以赋值给 U，返回 true，否则返回 false
type IsAssignableTo<T, U> = T extends U ? true : false;

// 测试类型可赋值性
// string 可以赋值给 any，所以返回 true
type CanAssign = IsAssignableTo<string, any>;
console.log("string 赋值给 any?: " + CanAssign);
```

### 条件类型小结

条件类型基于 `T extends U ? X : Y` 语法在类型层面实现"if/else"逻辑，是 TypeScript 类型编程的基石：

- **基本语法**：`T extends U ? X : Y`，根据条件返回不同类型
- **`infer` 关键字**：在条件类型中声明待推断的类型变量，用于提取函数返回值（`ReturnType`）、数组元素等
- **分布特性**：当 T 是联合类型时，条件类型自动分布到每个成员（`T extends any ? T[] : never` 对 `string | number` 会得到 `string[] | number[]`）
- **类型过滤**：利用 `T extends null | undefined ? never : T` 模式排除特定类型
- **与映射类型结合**：条件类型 + 映射类型是实现 `Partial`、`Required`、`Omit` 等工具类型的基础
- **选择原则**：`infer` 用于"提取"，分布特性用于"遍历联合类型"，过滤模式用于"排除/筛选"

## 交叉类型

在开发中，一个类型往往需要拥有多个类型的特性。例如，一个员工既是一个人（Person），也是一个工作者（Worker），需要同时具有两者的属性。交叉类型让我们可以将多个类型合并成一个，满足这种需求。

### 交叉类型基本语法

交叉类型使用 `&` 符号表示，将多个类型合并为一个类型。

```ts
// 定义人员类型
// 包含姓名和年龄
interface Person {
    name: string;
    age: number;
}

// 定义工作者类型
// 包含公司名称和薪资
interface Worker {
    company: string;
    salary: number;
}

// 使用交叉类型合并两个接口
// Employee 类型同时具有 Person 和 Worker 的所有属性
type Employee = Person & Worker;

// 创建同时具有两个类型特性的对象
var employee: Employee = {
    name: "Alice",
    age: 25,
    company: "Google",
    salary: 100000
};

console.log("员工: " + JSON.stringify(employee));  
// 输出：员工: {"name":"Alice","age":25,"company":"Google","salary":100000}
```

### 交叉类型与接口继承

交叉类型可以替代接口的多重继承。

```ts
// 定义类型 A
interface A {
    a: string;
}

// 定义类型 B
interface B {
    b: number;
}

// 使用接口继承多个接口
// 需要使用 extends 继承多个接口
interface AB extends A, B {
    c: boolean;
}

// 使用交叉类型（更简洁）
// 直接使用 & 符号组合类型
type ABType = A & B & { c: boolean };

// 两种方式都能创建包含所有属性的类型
var obj: ABType = { a: "hello", b: 42, c: true };
console.log("对象: " + JSON.stringify(obj));
// 输出：对象: {"a":"hello","b":42,"c":true}
```

### 交叉类型与联合类型

交叉类型和联合类型结合时需要特别注意优先级。不兼容的类型进行交叉会得到 never 类型。例如 string & number 是无效的。

```ts
// 联合类型：可以是字符串或数字
type StringOrNumber = string | number;

// 交叉类型：不兼容类型的交叉
// string & number = never（没有类型同时是字符串和数字）
type Both = string & number;

// 定义三个类型
type A = { a: string };
type B = { b: number };
type C = { c: boolean };

// 联合类型与交叉类型结合
// (A | B) & C 会将联合类型的每个分支都与 C 交叉
type Combined = (A | B) & C;

// 实际结果是：{ a: string; c: boolean } | { b: number; c: boolean }
// 即要么是 A + C，要么是 B + C
var obj: Combined = { a: "hello", c: true };
console.log("组合: " + JSON.stringify(obj)); // 输出：组合: {"a":"hello","c":true}
```

- 原始类型交叉 → 找"既是 A 又是 B"的值 → 不可能 → never
- 对象类型交叉 → 找"同时有 A 的属性又有 B 的属性"的值 → 合并属性 → 新对象类型

### 交叉类型小结

交叉类型（`&`）将多个类型合并为一个，结果**同时拥有所有类型的成员**——"既要 A 又要 B"：

- **基本语法**：`type C = A & B`，C 拥有 A 和 B 的全部属性和方法
- **vs 接口继承**：`A & B & { c }` 比 `interface AB extends A, B { c }` 更简洁，且 `&` 不限于接口，可组合任意类型
- **对象合并规则**：同名属性类型必须兼容（如两个 `name: string` 无冲突），不兼容则结果为 `never`
- **分配律**：`(A | B) & C` 等价于 `(A & C) | (B & C)`，交叉对联合有分配性质
- **vs 联合类型**：`|` 是"或"（满足其一即可），`&` 是"且"（全部满足才行）；原始类型交叉（如 `string & number`）因不存在交集而得到 `never`
