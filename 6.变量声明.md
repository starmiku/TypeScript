# 变量

let和const是JavaScript里相对较新的声明变量方式，let在很多方面与var是相似的，但可以避免一些常见问题。const是对let的一个增强，能阻止对一个变量再次赋值。

## var声明

JavaScript变量一直以来都是通过var关键字定义.

```ts
var a = 10;
```

可以在函数内部定义变量。

```ts
function f() {
    var message = "Hello, world!";

    return message;
}
```

内部函数可以捕获并访问外部函数中的变量（闭包）。

```ts
function f() {
    var a = 10;
    return function g() {
        var b = a + 1;
        return b;
    }
}

var g = f();
g(); // 返回 11
```

### 作用域规则

var声明可以在包含它的函数、模块、命名空间或全局作用域内部任何位置被访问，这些作用域规则可能会引发一些错误，如多次声明同一个变量并不会报错。

```ts
function sumMatrix(matrix: number[][]) {
    var sum = 0;
    for (var i = 0; i < matrix.length; i++) {
        var currentRow = matrix[i];
        // 内层 for 循环结束后 i=2，覆盖了外层的 i
        for (var i = 0; i < currentRow.length; i++) {
            sum += currentRow[i];
            // 内层循环结束，i=2
        }
    }
    return sum; 
    // 内层循环结束后 i=2，外层 i++ 后 i=3，不满足 i<2
    // 因此只处理了第一行：matrix[0][0] + matrix[0][1] = 3
}

const matrix = [
  [1, 2],
  [3, 4]
];
sumMatrix(matrix); // 返回 3
```

### 捕获变量

JavaScript的回调函数捕获的是变量的引用，而不是变量的值。

```ts
for (var i = 0; i < 10; i++) {
    // for 循环同步执行，每次把 setTimeout 的回调放入任务队列
    // 循环结束时 i 已经变成 10，之后回调依次执行，全部输出 10
    setTimeout(function() { console.log(i); }, 100 * i);
}
// 最终输出 10 个 10
```

### 立即执行函数表达式

通常的解决办法是使用立即执行的函数表达式（IIFE）来捕获每次迭代时i的值

```ts
for (var i = 0; i < 10; i++) {
    // 定义一个匿名函数 function(i) { ... }
    // 用 () 包裹，表示这是一个函数表达式
    // 最后面的 (i) 表示立即调用这个函数，并把当前的 i 作为参数传入
    (function(i) {
        setTimeout(function() { console.log(i); }, 100 * i);
    })(i);  // ← 这里的 (i) 是调用，传入当前循环的 i 值
}
```

### var小结

var 是 JavaScript 最早的变量声明方式，它的核心问题是**函数作用域**和**捕获引用而非值**：

- 同一函数内重复声明不报错，容易意外覆盖变量
- for 循环中用 var 声明的变量在循环结束后仍然存在，且所有回调共享同一个变量引用
- 解决循环捕获问题的经典方案是 IIFE（立即执行函数表达式），但写法繁琐
- 如今 var 仅用于兼容极旧环境，新代码应使用 let 或 const

## let声明

为了解决var作用域问题，JavaScript引入了let声明。let声明的变量使用块作用域，只在包含它的`{}`块（或for循环）内有效；而var声明的变量使用函数作用域，在包含它的整个函数内都可访问。

### 块作用域

```ts
function f(input: boolean) {
    let a = 100;

    if (input) {
        // a=100
        let b = a + 1;
        return b;
    }

    // Error: b不存在
    return b;
}
```

使用let替代上文IIFE的例子：

```ts
for (let i = 0; i < 10; i++) {
    // let在for循环的每次迭代中都会创建一个新的绑定
    setTimeout(function() { console.log(i); }, 100 * i);
}
```

块作用域变量的另一个特点是，它们不能在被声明之前读或写

```ts
a++; // 错误：在声明前不能使用 'a'（暂时性死区）
let a;
```

但仍可以在一个拥有块作用域变量被声明前获取它，只是不能在变量声明前调用。

```ts
function foo() {
    // okay to capture 'a'
    return a;
}

// 不能在'a'被声明前调用'foo'
let a;
foo();
```

### 重定义及屏蔽

使用var声明时，不管声明多少次都只会获得一个，因为所有声明都引用同一个变量，但let声明不允许。

```ts
let x = 10;
let x = 20; // 错误，不能在1个作用域里多次声明`x`
```

不仅在同一块级作用域内重复声明会报错，函数参数与 let 声明重名、同一函数内 var 与 let 声明重名也会报错。

```ts
function f(x) {
    let x = 100; // error: interferes with parameter declaration
}

function g() {
    let x = 100;
    var x = 100; // error: can't have both declarations of 'x'
}
```

在一个嵌套作用域里引入一个新名字的行为称作屏蔽。并不是说块级作用域变量不能与外部变量同名，而是需要在明显不同的块中声明，并有意识地使用这一特性。

```ts
function f(condition, x) {   
    // 这里的x是函数参数，作用域是整个函数
    if (condition) {
        let x = 100;// 这里的x是if块内的局部变量，作用域仅限于这个块
        return x; // 返回100（块内的x屏蔽了外部的参数x）
    }

    return x; // 返回参数 x（不受块内 let x 影响）
}

f(false, 0); // returns 0
f(true, 0);  // returns 100
```

通常来讲应该避免使用屏蔽，因为需要写出清晰的代码，同时也有些场景适合利用它。

```ts
function sumMatrix(matrix: number[][]) {
    let sum = 0;
    for (let i = 0; i < matrix.length; i++) {
        // 内层 i 不会影响外层 i，因为 let 是块级作用域
        var currentRow = matrix[i];
        for (let i = 0; i < currentRow.length; i++) {
            sum += currentRow[i];
        }
    }

    return sum;
}
```

### let小结

let 解决了 var 的两大痛点，是声明**可变变量**的首选：

- **块级作用域**：变量只在 `{}` 内有效，for 循环每次迭代创建新绑定，天然解决循环回调问题
- **暂时性死区**：声明前不能使用变量，避免变量提升带来的混乱
- **禁止重复声明**：同一作用域内不可重复声明，防止意外覆盖
- **屏蔽**：内层块可以用同名 let 覆盖外层变量，但应有意识地使用，避免造成混淆

## const声明

const声明是声明变量的另一种方式，与let类似，但它们被赋值后不能再改变（不能重新绑定）。但需要注意的是，const 只保证变量引用不变，如果值是对象或数组，其内部属性或元素仍然可以修改。

```ts
const numLivesForCat = 9;
const kitty = {
    name: "Aurora",
    numLives: numLivesForCat,
}

// 不能重新赋值
numLivesForCat = 10; // 错误
kitty = {
    name: "Danielle",
    numLives: numLivesForCat
}; // 错误

// 可以修改kitty的内部属性
kitty.name = "Rory";
kitty.name = "Kitty";
kitty.name = "Cat";
kitty.numLives--;
```

const 是声明**不可变绑定**的方式，应作为默认选择：

- 与 let 一样具有块级作用域、暂时性死区和禁止重复声明的特性
- 声明后**不可重新赋值**（不可重新绑定），但对象内部的属性和数组元素仍可修改
- 适用原则：**默认使用 const**，只有当确定变量需要重新赋值时才改用 let

## 解构

解构本质上是从对象或数组中提取数据，并将其赋值给不同的变量。

### 解构数组

最简单的解构是数组的解构赋值。

```ts
let input = [1, 2];
let [first, second] = input;
console.log(first); // 输出 1
console.log(second); // 输出 2
```

相当于使用了索引。

```ts
first = input[0];
second = input[1];
```

解构也可以对已声明的变量重新赋值。

```ts
// 交换变量的值
[first, second] = [second, first];
```

作用于函数参数。

```ts
function f([first, second]: [number, number]) {
    console.log(first);
    console.log(second);
}

let input = [1, 2];
f(input);
```

### 剩余变量

解构数组也可以使用...语法创建剩余变量。

```ts
let [first, ...rest] = [1, 2, 3, 4];
console.log(first); // 输出 1
console.log(rest); // 输出 [2, 3, 4]
```

可以忽略不关心的尾随元素。

```ts
let [first] = [1, 2, 3, 4];
console.log(first); // 输出 1
```

或其他元素。

```ts
let [, second, , fourth] = [1, 2, 3, 4];
```

### 对象解构

对象也同样可以解构，同时，类似数组解构，也可以使用没有声明的赋值。

```ts
let o = {
    a: "foo",
    b: 12,
    c: "bar"
};

let { a, b } = o;
/*
等价于
let a = o.a;
let b = o.b;
*/

({ a, b } = { a: "baz", b: 101 });
//重新赋值a和b
//()是必须的，否则语法解析器会认为是一个代码块。
```

也可以使用...语法创建剩余变量。

```ts
let { a, ...passthrough } = o;
let total = passthrough.b + passthrough.c.length;
```

### 属性重命名

对象解构也可以用于重命名属性。

```ts
let { a: newName1, b: newName2 } = o;
/*
等价于
let newName1 = o.a;
let newName2 = o.b;
*/
```

解构的左边 {} 中写的是要从对象中读取的属性名，不是随意命名的变量名。想要用不同的变量名，必须用 属性名: 变量名 的语法。如果属性名和变量名相同，可以省略属性名。

如果需要指定类型，仍需要在后面写上完整的模式。

```ts
let { a: newName1, b: newName2 }: { a: string, b: number } = o;
```

### 默认值

默认值可以让你在属性为undefined时使用缺省值。

```ts
function keepWholeObject(wholeObject: { a: string, b?: number }) {
    let { a, b = 1001 } = wholeObject;
    //a=wholeObject.a
    //b=wholeObject.b，当wholeObject.b未定义时，b=1001
}
```

### 函数声明

解构也能用于函数声明。可用于以下经典场景：

避免重复书写`参数.属性`

```ts
// ❌ 没解构，每次都要写 obj.
function printUserOld(obj: { name: string; age: number }) {
  console.log(obj.name + " 今年 " + obj.age + " 岁");
}

// ✅ 解构后直接使用变量名
function printUser({ name, age }: { name: string; age: number }) {
  console.log(name + " 今年 " + age + " 岁");
}
```

结合默认值处理可选参数

```ts
function createUser({ name, age = 18, email = "未设置" }: { name: string; age?: number; email?: string }) {
  // age 和 email 没传就用默认值
  console.log(`${name}, ${age}岁, ${email}`);
}

createUser({ name: "Alice" });         // Alice, 18岁, 未设置
createUser({ name: "Bob", age: 25 }); // Bob, 25岁, 未设置
```

只获取需要的属性并忽略剩余部分

```ts
// API 返回大量字段
const response = { id: 1, name: "Alice", age: 25, phone: "123456", address: "..." };

// 只解构需要的那几个
function showName({ name, age }: { name: string; age: number }) {
  console.log(name, age);
}

showName(response); // ✅ 多余的字段被忽略，不会报错
```

Vue/React组件props使用

```ts
// Vue 组件中
defineProps<{
  title: string;
  count?: number;
}>()

// 使用解构
function MyComponent({ title, count = 0 }: { title: string; count?: number }) {
  return `${title} (${count})`;
}
```

结合剩余参数提取需要的属性并保留剩余部分

```ts
function saveUser({ id, ...rest }: { id: number; name: string; age: number; email: string }) {
  // id 单独处理
  console.log("更新用户 ID:", id);
  // rest 包含剩下的字段，可用于批量更新
  console.log("更新字段:", rest);
}

saveUser({ id: 1, name: "Alice", age: 25, email: "a@b.com" });
// 更新用户 ID: 1
// 更新字段: { name: "Alice", age: 25, email: "a@b.com" }
```

### 展开

展开操作符与解构操作符相反。它允许你将一个数组展开为另一个数组，或将一个对象展开为另一个对象。

```ts
let first = [1, 2];
let second = [3, 4];
let bothPlus = [0, ...first, ...second, 5]; // [0, 1, 2, 3, 4, 5]
```

展开操作符还可以用于对象。常用于合并配置，先展开默认配置，再使用具体传入的值进行覆盖。

```ts
let defaults = { food: "spicy", price: "$$", ambiance: "noisy" };
let search = { ...defaults, food: "rich" }; // { food: "rich", price: "$$", ambiance: "noisy" }
let search2 = { food: "rich", ...defaults }; // { food: "spicy", price: "$$", ambiance: "noisy" }
```

对象的展开是从左到右进行处理的，意味着后面的属性会覆盖前面的属性。

### 解构小结

解构和展开是处理数组与对象的利器，能显著提升代码简洁度：

- **数组解构**：按位置提取，可用逗号跳过元素，用 `...` 收集剩余元素
- **对象解构**：按属性名提取，可重命名（`a: newName`），可设默认值，可收集剩余属性
- **函数参数解构**：消除 `obj.xxx` 的重复，配合默认值处理可选参数，只取所需字段忽略其余
- **展开（`...`）**：将数组/对象"打散"合并，后出现的属性覆盖先出现的，常用于合并配置、创建浅拷贝
