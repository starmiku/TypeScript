# TypeScript简介

TypeScript是由Microsoft开发的开源编程语言，是JavaScript的超集，意味着任何有效JavaScript代码也是有效的TypeScript代码。

## 主要优势

TypeScript通过添加可选的静态类型、接口、枚举等特性，大大增强JavaScript的开发体验。

* 静态类型检查：在编译时发现类型错误，减少运行时错误
* 强大的IDE支持：智能代码补全、导航和重构
* 更好的代码可读性：类型本身就是最好的文档
* 现代JavaScript特性：支持ES6+语法，如箭头函数、模块、类等
* 渐进式迁移：可以将现有JavaScript项目迁移到TypeScript

## 工作原理

TypeScript不能直接在浏览器中运行，它需要先编译为JavaScript，这个编译过程会检查类型错误，并将TypeScript特有的语法转换为纯JavaScript

## 语言特性

核心增强能力：

* 类型批注&编译时检查：开发阶段发现类型错误
* 类型推断：无需显式声明类型，编译器可自动推断
* 类型擦除：编译后移除类型信息，输出纯JavaScript
* 接口：用于定义对象结构，增强代码约束
* 枚举：提供命名常量集合，提升可读性
* 泛型：支持类型服用，增强函数和类的灵活性
* Mixin：实现多重继承的组合模式
* 命名空间：组织代码结构
* 元组：固定长度、不同类型的数组
* 异步：提供更直观的异步流程控制

来自ES2015的语法支持：

* 类：面相对象编程支持
* 模块：支持模块化开发（import/export）
* 箭头函数：更简洁的函数写法，绑定词法作用域
* 可选参数：函数参数可以忽略
* 默认参数：为参数提供默认值

关键特性：

* 接口和类型定义：TypeScript提供了`interface`和`type`关键字，允许你定义复杂的数据结构。这对于项目中不同部分的代码写作和数据交互来说非常重要。
* 类和模块支持：TypeScript支持面向对象编程中的类（class）概念，增加过了构造函数、继承、访问控制修饰符（如public、private、protected）

## 新增主要功能

静态类型：TypeScript最大特性就是增加了静态类型系统，可以显式地声明变量、参数、返回值的类型，常见类型包括：number、string、boolean、array、tuple、enum等。

```ts
let name: string = "Alice";
let age: number = 25;
```

类型推断：TypeScript可以自动推断变量类型，即使不显式声明类型，也会根据变量的赋值内容来推断类型

```ts
let name = "alice";//推断为string
```

接口interface：TypeScript提供了接口，允许定义复杂的对象结构。接口可以定义属性和方法，还可以通过implements关键字实现接口，或者通过extends进行扩展，便于定义复杂的数据类型。

```ts
interface Person {
  name: string;//一个字符串类型的name属性
  age: number;//一个数字类型的age属性
  greet(): void;//一个名为greet，没有返回值的方法
}//定义一个名为person的接口

class Student implements Person {
  constructor(public name: string, public age: number) {}
  //语法糖，自动创建并初始化同命的实例属性

  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }//实现定义的greet方法
}//一个遵循person接口的名为student的类，必须包含person中的属性
```

类型别名：类型别名可以为复杂的类型定义简短的别名，便于代码复用

```ts
type StringOrNumber = string | number;
let value: StringOrNumber = 42;
```

枚举enum：TypeScript引入了enum类型，用于定义一组命名的常量，提高可读性。

```ts
enum Direction {
  Up,
  Down,
  Left,
  Right,
}
let dir: Direction = Direction.Up;
```

元组：元组允许定义具有固定数量和类型的数组，适用于需要固定数据结构的场景，比如坐标或RGB颜色值

```ts
let point: [number, number] = [10, 20];
```

访问控制修饰符:TypeScript 在类中提供了 public、private 和 protected 修饰符，允许控制属性或方法的可见性，支持更好的封装。

```ts
class Person {
  private name: string;//仅限类person访问
  protected age: number;//仅限类person及子类访问
  public constructor(name: string, age: number) {
    this.name = name;//将参数name的值赋给实例的name属性，不可访问
    this.age = age;//将参数age的值赋给实例的age属性，可访问
  }//一个公开的构造函数，用于接收参数
}
```

抽象类abstract classes：TypeScript支持抽象类，抽象类不能直接实例化，需要由子类实现，抽象类适用于定义通用行为和抽象方法的类层次结构。

```ts
abstract class Animal {
  abstract makeSound(): void;//声明一个抽象方法，必须由子类实现，否则报错
}//声明一个抽象类animal

class Dog extends Animal {
  makeSound() {
    console.log("Woof!");//实现父类的抽象方法
  }//继承抽象类animal的子类dog
}
```

泛型：TypeScript支持泛型，允许在类、接口和函数中使用参数化类型，使得代码可以适应不同的类型需求，同时保持类型安全。

```ts
function identity<T>(value: T): T {
  return value;//value类型为T，返回类型也为T
}

let num = identity<number>(42);//num类型为数字
```

模块和命名空间：TypeScript 提供了基于 ES6 的模块系统，使用 import 和 export 导入和导出模块。此外，TypeScript 还支持命名空间（Namespace），用于组织代码和避免命名冲突。

```ts
// math.ts
export function add(a: number, b: number): number {
  return a + b;
}//导出一个函数add(a,b)，返回a+b的值，其中a、b、返回值类型均为数字

// main.ts
import { add } from "./math";//导入函数add
console.log(add(2, 3));//输出函数add(2,3)的返回值
```

类型守卫：TypeScript提供了类型守卫，可以在代码中检查变量类型，这对于联合类型尤为重要。

```ts
function printId(id: string | number) {
  if (typeof id === "string") {
    console.log(id.toUpperCase());
  } else {
    console.log(id.toFixed(2));
  }
} 
```

可选链和空值合并运算符：TypeScript 增加了 JavaScript 的可选链 (?.) 和空值合并运算符(??)，简化了代码中对可能为null或undefined值的处理，组合则可以安全取值并提供默认值。

```ts
let user = { name: "Alice", address: { city: "Wonderland" } };
console.log(user?.address?.city); //如果user及address均存在则输出city，否则返回undefined，

let value = null;
console.log(value ?? "default"); //如果问号左边的值为null或undefined，则返回右边的值
```

类型兼容性和工具类型：TypeScript提供了一些工具类型，如Partial、Pick、Readonly、Record等，这些类型可以帮助生成新的类型，简化类型定义。

```ts
interface Todo {
  title: string;
  description: string;
}

let partialTodo: Partial<Todo> = { title: "Learn TypeScript" }; // 可选属性
```
