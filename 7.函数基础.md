# 函数

## 函数定义

函数就是包裹在花括号中的代码块，语法格式如下：

```ts
function 函数名(参数1: 类型, 参数2: 类型): 返回值类型 {
    // 函数体
}
```

函数定义是 TypeScript 最基本的函数声明语法：

- 语法：`function 函数名(参数: 类型): 返回值类型 { ... }`
- 参数类型和返回值类型可选，但建议写明以获得类型检查
- 这是具名函数的基础形式，拥有声明提升特性

## 调用函数

函数只有通过调用才能执行函数中的代码。

```ts
function sayHello(name: string): void {
    console.log(`Hello, ${name}`);
}

sayHello("York");
```

函数只有被调用时才会执行，调用方式为 `函数名(实参)`：

- 调用时传入的实参与定义时的形参一一对应
- 不调用则函数体内的代码永远不会运行

## 函数返回值

函数通过return将执行的结果返回到调用的地方，返回值类型在函数名后声明。

```ts
// 函数定义
function greet():string { // 返回一个字符串
    return "Hello World" 
} 
 
function caller() { 
    var msg = greet() // 调用 greet() 函数 
    console.log(msg) 
} 
 
// 调用函数
caller()
```

函数通过 `return` 将执行结果返回给调用方：

- 返回值类型写在函数名后的冒号之后，如 `function greet(): string`
- 函数执行到 `return` 语句即结束，后面的代码不会执行
- 如果不需要返回值，声明为 `void` 类型

## 带参数函数

调用函数时，可以向其传递值，这些值被称为参数，语法格式如下：

```ts
function 函数名(参数1: 类型, 参数2: 类型): 返回值类型 {
    // 函数体
}
```

其中参数类型和返回值类型都是可选的。

```ts
function add(x, y) {
    return x + y;
}
console.log(add(1, 2));
```

### 可选参数

在 TypeScript 中，函数的参数是可选的，即可以不传值。可选参数必须在必选参数之后，否则会报错。可选参数使用问号标识。

```ts
function add(x: number, y?: number): number {
    if (y) {
        return x + y;
    } else {
        return x;
    }
}
console.log(add(1)); // 输出：1
console.log(add(1, 2)); // 输出：3
console.log(add(1, 2, 3)); // 报错：Expected 2 arguments, but got 3.
```

### 默认参数

在 TypeScript 中，函数的参数可以有默认值，当调用函数时没有传递该参数时，会使用默认值。

```ts
function add(x: number, y: number = 0): number {
    return x + y;
}
console.log(add(1)); // 输出：1
console.log(add(1, 2)); // 输出：3
```

### 带参数函数小结

参数让函数变得灵活，TypeScript 提供了三种参数形式：

- **必选参数**：调用时必须传入，放在参数列表最前面
- **可选参数（`?`）**：可传可不传，必须放在必选参数之后
- **默认参数（`= 值`）**：未传时使用默认值，语法最简洁，推荐优先使用

## 剩余参数

在 TypeScript 中，函数的参数可以使用剩余参数，即可以传递多个参数，这些参数会被存储在一个数组中。函数的最后一个命名参数以...为前缀，将成为一个由剩余参数组成的数组，索引值从0到参数.length-1。

```ts
function add(...args: number[]): number {
    let sum = 0;
    for (let i = 0; i < args.length; i++) {
        sum += args[i];
    }
    return sum;
}
console.log(add(1, 2, 3, 4, 5)); // 输出：15
```

可用于处理不确定数量的参数，例如：

```ts
function log(level: string, ...messages: string[]): void {
    const timestamp = new Date().toLocaleString();
    const prefix = `[${timestamp}] [${level.toUpperCase()}]`;
    // messages 数组可以传多个日志内容
    console.log(prefix, ...messages);
}

log("info", "服务启动", "端口: 3000");
log("error", "数据库连接失败", "重试中...", "错误代码: ECONNREFUSED");
log("warn", "磁盘空间不足");
// 输出示例:
// [2026/6/4 10:30:00] [INFO] 服务启动 端口: 3000
// [2026/6/4 10:30:00] [ERROR] 数据库连接失败 重试中... 错误代码: ECONNREFUSED
```

剩余参数（`...args`）用于处理**不确定数量**的参数：

- 语法：在最后一个参数前加 `...`，所有多余实参被收集为一个数组
- 典型场景：日志函数、求和函数等参数个数不固定的场合
- 与 `arguments` 对象不同，剩余参数是真正的数组，可以直接使用数组方法

## 匿名函数

匿名函数是一个没有函数名的函数，在程序运行时动态声明，可以将匿名函数赋值给变量，这种表达式就成为函数表达式。

### 不带参匿名函数

```ts
const msg = function() { //由于函数没有重新赋值，所以使用const声明
    return "hello world";  
} 
console.log(msg())
```

### 带参匿名函数

```ts
const res = function(a:number,b:number) { 
    return a*b;  
}; 
console.log(res(12,2))
```

### 自调用匿名函数

匿名函数自调用在函数后使用()即可。

```ts
(function() {
    console.log("hello world");
})();
```

### 匿名函数小结

匿名函数是没有名称的函数，通常作为表达式使用：

- **函数表达式**：将匿名函数赋值给变量，通过变量名调用
- **自调用（IIFE）**：定义后立即执行，格式为 `(function(){}())`
- 适合作为回调、事件处理器等一次性使用的场景

## 构造函数（不推荐）

Typescript也支持构造函数Function()来定义函数。语法格式如下：

```ts
let 函数名 = new Function(参数1, 参数2, ..., 参数N, 函数体)
```

函数体是一个包含函数定义的JS字符串

```ts
let add = new Function("x", "y", "return x + y");
console.log(add(1, 2)); // 输出：3
```

`new Function()` 可以用字符串动态创建函数，但**不推荐使用**：

- 语法：`new Function("参数1", "参数2", ..., "函数体字符串")`
- 存在安全风险（类似 `eval`），且无法利用 TypeScript 的类型检查
- 几乎所有场景都有更好的替代方案（箭头函数、普通函数等）

## 递归函数

递归函数在函数内调用函数本身。

```ts
function factorial(number) {
    if (number <= 0) {         // 停止执行
        return 1;
    } else {
        return (number * factorial(number - 1));     // 调用自身
    }
};
console.log(factorial(6));      // 输出 720
```

递归函数在函数体内调用自身，是解决分治问题的经典模式：

- **必须包含终止条件**（基准情形），否则会导致无限递归和栈溢出
- 经典例子：阶乘、斐波那契数列、树遍历
- 递归需要函数有名称才能调用自身，因此必须使用具名函数

## 箭头函数

箭头函数表达式的语法比函数表达式更短。箭头函数是匿名函数，它本身没有this，它会捕获其所在上下文的this值，作为自己的this值。语法格式如下：

```ts
(参数1: 类型, 参数2: 类型, ..., 参数N: 类型): 返回值类型 => {};
```

### 箭头函数基础

箭头函数提供更简洁的函数定义语法。

```ts
const add = (x: number, y: number): number => {
    return x + y;
}
console.log(add(1, 2)); // 输出：3
```

箭头函数的参数和函数体之间使用箭头符号 =>，箭头函数的函数体如果只有一行代码，可以省略大括号和return关键字，即可以改写为：

```ts
const add = (x: number, y: number) => x + y;
console.log(add(1, 2)); // 输出：3
```

如果不指定函数的参数类型，则参数会退化为any类型，此时需要通过函数内来推断参数类型。

```ts
const func = (x): void => {
    if (typeof x === "number") {
        console.log(`${x} 是一个数字`);
    } else if (typeof x === "string") {
        console.log(`${x} 是一个字符串`);
    }
};
```

单个参数可以省略括号。

```ts
const func = x => x * x;
console.log(func(3)); // 输出：9
```

箭头函数可以用于回调函数，例如：

```ts
let arr = [1, 2, 3, 4, 5];
arr.forEach(value => {
    console.log(value);
});
```

### 箭头函数与this

箭头函数最核心的特性是不绑定自己的this，而是捕获定义时所在外层作用域的this，并保持不变

```ts
// 使用普通函数
function Person1() {
    this.name = "Alice";
    const self = this; // 保存外层 this

    // 普通函数会创建自己的 this
    // 在 setTimeout 回调中，this 指向 window（浏览器）或 undefined（严格模式）
    setTimeout(function() {
        console.log("普通函数: " + self.name);  // this.name 为 undefined
    }, 100);
    
    /*
    也可以使用：
    const name=this.name;
    setTimeout(function() {
        console.log("普通函数: " + name);
    }, 100);
    将name作为变量输入也可以得到正确结果
    */
}

// 使用箭头函数
function Person2() {
    this.name = "Bob";

    // 箭头函数不创建自己的 this
    // 它捕获外层的 this，所以能正确访问到 name
    setTimeout(() => {
        console.log("箭头函数: " + this.name);  // this.name 为 "Bob"
    }, 100);
}

// 测试
new Person1(); // 输出：普通函数: Alice
new Person2(); // 输出：箭头函数: Bob
```

### 类中的箭头函数

在typescript中，可以使用箭头函数作为类的方法或属性，这样可以确保方法被传递或作为回调使用时，this仍指向类的实例。

```ts
// 定义计数器类
class Counter {
    // 计数器的当前值
    count: number = 0;

    // 使用箭头函数作为类属性
    // 每次创建实例时，都会创建一个新的函数
    // this 指向实例
    increment = () => {
        this.count++;
        console.log("当前计数: " + this.count);
    };

    // 普通方法
    decrement() {
        this.count--;
        console.log("当前计数: " + this.count);
    }
}

// 创建计数器实例
var counter = new Counter();

// 调用箭头函数方法
counter.increment(); // 输出：当前计数: 1
counter.increment(); // 输出：当前计数: 2

// 调用普通方法
counter.decrement(); // 输出：当前计数: 1
```

### 箭头函数的类型

TypeScript中箭头函数的类型注解使用不同的语法，使用=>而不是冒号来定义函数类型。

基础语法如下：

```ts
// 直接定义箭头函数类型
// (a: number, b: number) => number 表示接受两个 number 参数，返回 number
const add: (a: number, b: number) => number = (a, b) => a + b;

//更常见的写法，在实现时声明函数类型。
const add2 = (a: number, b: number): number => a + b;
```

通常这种写法适合于希望明确声明函数类型，再写具体实现时有用，特别是：

- 函数类型复杂，想在变量声明处明确约束
- 实现部分参数较多，不想重复写类型
- 配合接口或类型别名复用函数类型

```ts
// 使用接口定义箭头函数类型
// 这种方式更适合在接口或类型别名中复用
interface MathOperation {
    // 定义函数签名
    (a: number, b: number): number;
}

// 使用接口类型
var add: MathOperation = (a, b) => a + b;
var multiply: MathOperation = (a, b) => a * b;

console.log("加法: " + add(2, 3));
console.log("乘法: " + multiply(4, 5));
```

### 箭头函数小结

箭头函数是 ES6 引入的简洁函数语法，核心特性有三点：

- **语法简洁**：单行可省略 `{}` 和 `return`，单参数可省略 `()`
- **不绑定自己的 `this`**：捕获定义时外层作用域的 `this`，是解决回调中 `this` 丢失的首选方案
- **类型注解**：用 `(a: number, b: number): number => a + b` 或接口定义函数类型
- 适用于回调、类属性方法、简短映射函数；不适合需要动态 `this` 的场景（如事件处理器的 `event.target`）

## 具名函数

具名函数是具有名称的函数，语法格式如下：

```ts
function 函数名(参数1: 类型, 参数2: 类型, ..., 参数N: 类型): 返回值类型 {
    // 函数体
}
```

### 关键规则

具名函数具有函数声明提升的特性，会被提升到作用域顶部，可以在定义之前调用。

```ts
// 可以在定义之前调用
sayHello("World");  // 正常执行

function sayHello(name: string) {
    console.log("Hello, " + name);
}
```

具名函数必须有函数名，不能是匿名函数。

```ts
function myFunction() {}  // 具名
function() {}              // 语法错误，匿名函数必须作为表达式使用
```

具名函数的name属性就是函数名，其本身也可以赋值给变量。

```ts
function login() {}
console.log(login.name);  // 输出: "login"

const myFunc = login;
console.log(myFunc.name);  // 输出: "login"
```

### 主要作用

具名函数可以使代码结构更清晰,更易于调试和维护。

```ts
// 主逻辑
function main() {
    validateInput(data);
    processData(data);
    saveResult(data);
}

// 辅助函数放在后面
function validateInput(data: any) { ... }
function processData(data: any) { ... }
function saveResult(data: any) { ... }

main();  // 调用主函数
```

作为递归函数是另一个典型应用场景，递归函数需要调用自身，而匿名函数无法直接调用自身。

```ts
// 可以在定义前间接调用自身
function factorial(n: number): number {
    return n <= 1 ? 1 : n * factorial(n - 1);
    //当n <= 1时，递归结束，返回1，否则继续递归
}
```

当两个函数互相调用时，提升可以省去繁琐的前置声明。

```ts
function isEven(n: number): boolean {
    if (n === 0) return true;
    return isOdd(n - 1);  // isOdd 在后面定义，但可以被调用
}

function isOdd(n: number): boolean {
    if (n === 0) return false;
    return isEven(n - 1);  // isEven 在前面定义
}
```

### 具名函数小结

具名函数即用 `function 函数名(){}` 声明的函数，核心优势是**声明提升**：

- **提升（Hoisting）**：可以在定义之前调用，适合"主逻辑在前、辅助函数在后"的代码组织方式
- **必须有名称**：匿名函数只能作为表达式，不能独立声明
- **典型场景**：递归（需要函数名调用自身）、相互递归（两个函数互相调用）、代码结构组织
- 函数名可通过 `.name` 属性获取，赋值给变量后 `.name` 保持不变

## 函数重载

重载是方法名字相同，但参数不同，返回类型可以相同也可以不同。每个重载的方法（或者构造函数）都必须有一个独一无二的参数类型列表。编译器会根据传入的参数类型选择正确的实现。

参数类型不同

```ts
function disp(string):void; 
function disp(number):void;
```

参数数量不同

```ts
function disp(n1:number):void; 
function disp(n1:number,n2:number):void;
```

参数类型顺序不同

```ts
function disp(n1:number,s1:string):void;
function disp(s1:string,n1:number):void;
```

### 基本语法

先声明多个函数签名，然后实现一个统一函数。

```ts
// 函数重载签名
function add(a: number, b: number): number;
function add(a: string, b: string): string;
function add(a: any, b: any): any {
    return a + b;
}

console.log("数字相加: " + add(1, 2));
console.log("字符串相加: " + add("Hello, ", "World"));
```

### 多参数重载

如果参数类型不同，则参数类型应设置为any，参数数量不同可以将不同的参数设置为可选。

```ts
function disp(s1:string):void;
function disp(n1:number,s1:string):void;

function disp(x:any,y?:any):void {
    console.log(x);
    console.log(y);
}
disp("abc")
disp(1,"xyz");
```

### 方法重载

类中的方法也可以使用重载。

```ts
class Calculator {
    // 重载签名
    add(a: number, b: number): number;
    add(a: string, b: string): string;
    add(a: number, b: string): string;
    add(a: any, b: any): any {
        return a + b;
    }
}

var calc = new Calculator();
console.log("数字: " + calc.add(1, 2));
console.log("字符串: " + calc.add("Hello", "World"));
console.log("混合: " + calc.add(5, " apples"));
```

### 构造函数重载

构造函数同样可以重载，当不同参数组合需要完全不同的初始化逻辑时，才需要使用构造函数重载。

```ts
class User {
    name: string;
    age: number;
    email: string;

    // 有重载时，不同参数走不同初始化逻辑
    constructor(name: string);
    constructor(name: string, age?: number, email?: string) {
        this.name = name;
        if (age !== undefined && email !== undefined) {
            this.age = age;
            this.email = email;
        } else if(age !== undefined && email === undefined){
            this.age = age;
            this.email = "未设置";
        } else {
            this.age = 0;
            this.email = "未设置";
        }
    }
}

console.log(new User("Alice",25)); // User { name: 'Alice', age: 25 , email: '未设置' }
```

### 重载与联合类型

使用重载而不是联合类型可以获得更精确的类型判断。

```ts
// 推荐：使用重载
function process(value: number): number;
function process(value: string): string;
function process(value: any): any {
    if (typeof value === "number") {
        return value * 2;
    }
    return value.toUpperCase();
}

// TypeScript 知道返回类型
var numResult: number = process(10);  // number
var strResult: string = process("hello");  // string

console.log("数字结果: " + numResult);
console.log("字符串结果: " + strResult);
```

当入参和返回值类型一一对应时，重载更精确；如果不在乎返回值推断精度，联合类型更简洁。

```ts
// 使用联合类型
function process(value: number | string): number | string {
    if (typeof value === "number") {
        return value * 2;
    }
    return value.toUpperCase();
}

// 重载：TypeScript 知道返回的具体类型
var numResult: number = process(10);   // 可以直接赋值给 number
var strResult: string = process("hello"); // 可以直接赋值给 string

// 联合类型：TypeScript 只知的是 number | string
var numResult = process(10);   // 类型: number | string
var strResult = process("hello"); // 类型: number | string

// 需要用类型守卫才能确定具体类型
if (typeof numResult === "number") {
    console.log(numResult.toFixed(2)); // 这里才确定是 number
}
```

### 注意事项

- 重载签名必须放在实现签名之前
- 实现签名必须兼容所有重载签名
- 重载签名只是类型声明，不生成实际代码

### 函数重载小结

函数重载让同一个函数名支持**多种参数组合**，并获得精确的返回值类型：

- **写法规则**：先写多个重载签名（只有类型声明），最后写一个实现签名（兼容所有重载）
- **核心价值**：当入参与返回值类型一一对应时，重载比联合类型更精确——调用方无需再做类型收窄
- **适用场景**：类方法、构造函数、API 函数等参数类型不同导致返回值类型也不同的情况
- **注意事项**：重载签名必须在实现签名之前，实现签名参数类型通常用 `any` 兼容，重载签名不生成运行时代码
