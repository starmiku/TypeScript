# 接口与类

接口与类是 TypeScript 中非常重要的概念，它们为对象提供了结构和行为。

## 接口

接口是一系列抽象方法的声明，是一些方法特征的集合，这些方法都应该是抽象的，需要由具体的类去实现。

```ts
interface IPerson { 
    firstName:string, 
    lastName:string, 
    sayHi: ()=>string 
} 
 
var customer:IPerson = { 
    firstName:"Tom",
    lastName:"Hanks", 
    sayHi: ():string =>{return "Hi there"} 
} 
 
console.log("Customer 对象")
console.log(customer.firstName)
console.log(customer.lastName)
console.log(customer.sayHi())

var employee: IPerson = {
    firstName: "Jim",
    lastName: "Blakes",
    sayHi: (): string => { return "Hello!!!"; }
};

console.log("Employee 对象")
console.log(employee.firstName) 
console.log(employee.lastName)
```

### 联合类型接口

接口属性可使用联合类型，表示该属性可接受多种类型之一的值。

```ts
interface RunOptions { 
    program:string; 
    commandline:string[]|string|(()=>string); 
} 
 
// commandline 是字符串
var options:RunOptions = {program:"test1",commandline:"Hello"}; 
console.log(options.commandline)  
 
// commandline 是字符串数组
options = {program:"test1",commandline:["Hello","World"]}; 
console.log(options.commandline[0]); 
console.log(options.commandline[1]);  
 
// commandline 是一个函数表达式
options = {program:"test1",commandline:()=>{return "**Hello World**";}}; 
 
var fn: any = options.commandline;
console.log(fn());
```

联合类型接口允许接口属性为多种类型的组合（`string | string[] | (() => string)`），使同一个属性可接受不同形态的值，适配灵活的输入场景。

### 接口和数组

接口中可以将数组的索引值和元素设置为不同类型，索引值可以是数字或字符串。

```ts
interface NameList {
    [index: number]: string;
}

// 类型一致，正确
var list2: NameList = ["Google", "Runoob", "Taobao"];
// 错误：元素 1 不是 string 类型
// var list2: NameList = ["Runoob", 1, "Taobao"];
```

接口可用索引签名约束数组元素类型，如 `[index: number]: string` 表示所有元素必须为 string。

### 接口继承

接口可以通过 `extends` 关键字继承其他接口，也可以继承类。

接口继承语法格式：

`interface 子接口 extends 父接口1, 父接口2, ...`

接口继承实例：

```ts
interface IParent1 { 
    v1:number 
} 
 
interface IParent2 { 
    v2:number 
} 
 
interface Child extends IParent1, IParent2 { 
    v3: string
} 

var Iobj:Child = { v1:12, v2:23, v3:"This is child"} 
console.log("value 1: "+Iobj.v1+" value 2: "+Iobj.v2+" value 3: "+Iobj.v3)
```

接口通过 `extends` 支持多继承（`Child extends Parent1, Parent2`），子接口合并所有父接口的成员，实现接口层面的能力组合。

## 类

类是面向对象编程（OOP）的基础，它提供了一种结构化的方式来定义对象和它们的行为。

### 类的定义

一个类可以包含以下成员：

* 字段：类中声明的变量，表示对象的属性。
* 构造函数：类实例化时调用的特殊方法，用于初始化对象。
* 方法：类中定义的函数，表示对象的行为。

类定义的语法格式如下：

```ts
class 类名 {
    字段1: 类型;
    字段2: 类型;
    ...
    构造函数(参数1: 类型, 参数2: 类型, ...): void {
        // 初始化代码
    }
    方法1(参数1: 类型, 参数2: 类型, ...): 返回类型 {
        // 方法实现
    }
    ...
}
```

类的定义包含三大成员：字段（属性）、构造函数（初始化）、方法（行为）。构造函数在 `new` 时自动调用，用于初始化字段。

### 类的字段和构造函数

字段是存储对象数据的地方，构造函数在对象创建时自动调用，用于初始化字段。

```ts
// 定义 Car 类
class Car {
    // 字段：描述汽车的属性
    engine: string;

    // 构造函数：用于接受外来数据并初始化
    // 当所有字段都有默认值或不需要初始化的情况下不需要构造函数
    constructor(engine: string) {
        this.engine = engine;
    }

    // 方法：显示发动机信息
    disp(): void {
        console.log("发动机型号: " + this.engine);
    }
}

// 创建类的实例
var car = new Car("V8 发动机");

// 访问字段
console.log("读取发动机: " + car.engine);

// 调用方法
car.disp();
```

字段存储对象数据，构造函数在实例化时自动调用并初始化字段。若所有字段都有默认值，构造函数可省略。

### 访问控制修饰符

TypeScript 提供了三种访问修饰符来控制类成员的可访问性：

* public：公共成员，可以在任何地方访问。
* private：私有成员，只能在类内部访问。
* protected：受保护成员，只能在类内部和子类中访问。

#### public

public 是默认的访问修饰符，表示成员可以在任何地方被访问。如果不写访问修饰符，TypeScript 会默认使用 `public`，所以 `public` 可以省略。

```ts
class Person {
    public name: string;    // 公有属性
    public age: number;     // 公有属性

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    public introduce(): void {
        console.log("我是 " + this.name + "，今年 " + this.age + " 岁");
    }
}

var person = new Person("Alice", 25);
console.log("姓名: " + person.name);   // 可以访问
person.introduce();                    // 可以访问
```

#### private

private 表示私有成员，只能在定义它的类内部访问。子类和类的外部都不能访问 private 成员。这常用于隐藏类的内部实现细节，保护数据不被意外修改。

```ts
class Person {
    public name: string;
    private secret: string;  // 私有属性，外部无法直接访问

    constructor(name: string, secret: string) {
        this.name = name;
        this.secret = secret;
    }

    // 公有方法可以访问私有属性
    public revealSecret(): void {
        console.log("秘密: " + this.secret);
    }
}

var person = new Person("Alice", "我喜欢编程");

console.log("姓名: " + person.name);   // 可以访问
// console.log(person.secret);        // 错误：'secret' 是私有属性

person.revealSecret();                 // 通过公有方法访问私有属性
```

#### protected

protected 表示受保护成员，可以在类内部和子类中访问。类的外部不能直接访问 protected 成员。这在需要让子类继承父类的某些功能，同时隐藏实现细节时非常有用。

```ts
class Person {
    protected name: string;  // 受保护属性

    constructor(name: string) {
        this.name = name;
    }

    protected sayHello(): void {
        console.log("你好，我是 " + this.name);
    }
}

class Student extends Person {
    private grade: string;

    constructor(name: string, grade: string) {
        super(name);
        this.grade = grade;
    }

    public introduce(): void {
        // 子类可以访问受保护的属性和方法
        console.log("我是 " + this.name + "，年级: " + this.grade);
        this.sayHello();
    }
}

var student = new Student("Bob", "高三");
student.introduce();    // 通过公有方法访问

// console.log(student.name);  // 错误：'name' 是受保护属性
```

#### readonly修饰符

readonly 用于将属性设置为只读。只能在声明时或构造函数中赋值，之后不能修改。这在定义常量或标识符时非常有用。

```ts
// 定义用户类
class User {
    // 使用 readonly 修饰的属性只能在初始化时赋值
    // 用户 ID
    readonly id: number;
    // 用户名
    readonly name: string;

    constructor(id: number, name: string) {
        this.id = id;
        this.name = name;
    }
}

// 创建用户实例
var user = new User(1, "Alice");

console.log("用户: " + user.id + ", " + user.name);

// 错误：不能修改 readonly 属性
// user.id = 2; // 编译错误！
// user.name = "Bob"; // 编译错误！
```

#### 参数属性

参数属性是一种简写方式，可以在构造函数中声明并初始化字段。使用`public`、`private`或`protected`关键字来定义参数属性。

```ts
// 定义点类
class Point {
    // 在构造函数参数上直接使用修饰符
    // 相当于同时声明属性并赋值
    constructor(
        // public 修饰：创建公开属性 x
        public x: number,
        // public 修饰：创建公开属性 y
        public y: number,
        // private 修饰：创建私有属性 z
        private z: number
    ) {
        // 构造函数体可以为空，属性已自动创建
    }

    // 计算三维度总和
    public sum(): number {
        // 可以在类内部访问所有属性
        return this.x + this.y + this.z;
    }
}

// 创建点实例
var point = new Point(1, 2, 3);

// 公开属性可以从外部访问
console.log("x: " + point.x);
console.log("y: " + point.y);

// 调用方法
console.log("总和: " + point.sum());

// 错误：私有属性不能从外部访问
// console.log(point.z); // 编译错误！
```

#### 访问控制修饰符小结

三种修饰符 + 两种特殊修饰符控制类成员的可访问性：

* **`public`**（默认）：任意位置可访问，可省略不写
* **`private`**：仅类内部可访问，子类和外部均不可见
* **`protected`**：类内部 + 子类可访问，外部不可见
* **`readonly`**：只在声明或构造函数中赋值，之后不可改
* **参数属性**：构造函数参数加修饰符（`public x: number`），等同于声明属性并赋值

### 类的继承

继承允许创建一个类（子类），作为另一个类（父类）的扩展。子类可以复用父类的代码，还可以扩展或重写父类的行为。

基本语法如下：

`class 子类 extends 父类 { ... }`

#### 单继承

```ts
// 父类：形状
class Shape {
    area: number;

    constructor(a: number) {
        this.area = a;
    }
}

// 子类：圆，继承自 Shape
class Circle extends Shape {
    disp(): void {
        console.log("圆的面积: " + this.area);
    }
}

var circle = new Circle(223);
circle.disp();
```

#### 多层继承

TypeScript 不支持多继承（一个类继承多个类），但支持多层继承（A → B → C）。

```ts
// 根类
class Root {
    str: string;
}

// 子类：继承 Root
class Child extends Root {
}

// 叶子类：继承 Child（多层继承）
class Leaf extends Child {
}

var leaf = new Leaf();
leaf.str = "hello";
console.log("str 值: " + leaf.str);
```

#### 类的继承小结

TypeScript 只支持单继承（一个类只能继承一个父类），但支持多层继承（A→B→C）：

* 子类用 `extends` 继承父类所有 public/protected 成员
* 构造函数中必须先用 `super()` 调用父类构造，再使用 `this`
* 多层继承形成继承链，可逐层复用和扩展

### 方法重写

子类可以重写（override）父类的方法，即在子类中定义与父类同名的方法，实现自己的行为。使用`super`关键字可以调用父类的方法。

```ts
// 父类
class PrinterClass {
    doPrint(): void {
        console.log("父类的 doPrint() 方法");
    }
}

// 子类：重写父类方法
class StringPrinter extends PrinterClass {
    doPrint(): void {
        // 调用父类的方法
        super.doPrint();

        // 子类自己的逻辑
        console.log("子类的 doPrint() 方法");
    }
}

var obj = new StringPrinter();
obj.doPrint();
```

子类定义与父类同名方法即可重写（override），用 `super.方法名()` 调用父类版本，实现"增强而非替换"。

### 静态成员

使用`static`关键字定义的成员属于类本身，而不是类的实例。可以直接通过类名访问，不需要创建实例。

```ts
class StaticMem {
    // 静态属性
    static num: number;

    // 静态方法
    static disp(): void {
        console.log("num 值为 " + StaticMem.num);
    }
}

// 直接通过类名访问静态成员
StaticMem.num = 12;
StaticMem.disp();
```

`static` 成员属于类本身而非实例，直接通过 `类名.成员` 访问，无需 `new`。常用于工具函数和全局常量。

### instanceof

`instanceof`运算符用于检查一个对象是否是某个类的实例。

```ts
class Person {
}

var obj = new Person();
var isPerson = obj instanceof Person;
// instanceof 运算符返回一个布尔值

console.log("obj 是 Person 类的实例吗？ " + isPerson);
```

`instanceof` 检查对象是否为某个类的实例（含子类实例），返回布尔值。用于运行时类型判断。

### 类实现接口

类可以使用`implements`关键字实现接口，确保类提供接口中声明的所有属性和方法。

```ts
// 定义接口
interface ILoan {
    interest: number;  // 利率
}

// 类实现接口
class AgriLoan implements ILoan {
    interest: number;
    rebate: number;    // 回扣

    constructor(interest: number, rebate: number) {
        this.interest = interest;
        this.rebate = rebate;
    }
}

var loan = new AgriLoan(10, 1);
console.log("利率: " + loan.interest + "%，回扣: " + loan.rebate);
```

`implements` 强制类提供接口声明的所有成员，一个类可实现多个接口，确保"这个类一定能做某事"。

### 鸭子类型

鸭子类型（Duck typing）源自动态类型语言，其核心思想是：不关注对象所属的类，只关注对象所具有的方法和属性。在 TypeScript 的结构化类型系统中，相较于 `implements` 的显式声明，鸭子类型是隐式、自动匹配的。

```ts
interface Duck {
    quack(): void;
    fly(): void;
}

// ─── 用 implements（显式声明） ───
class Mallard implements Duck {     // 显式声明实现了Duck
    //需要在此处实现接口的方法
    quack() { console.log("嘎"); }
    fly()   { console.log("飞"); }
}

// ─── 用鸭子类型（隐式匹配） ───
class Dog {                          // 隐式匹配了Duck
    quack() { console.log("嘎"); }
    fly()   { console.log("飞"); }
}

// 参数类型为 Duck，可以传入 Mallard 或 Dog
function makeDuck(d: Duck) {
    d.quack();
}

makeDuck(new Mallard()); // implements Duck
makeDuck(new Dog());     // Dog 有 quack 和 fly，自动匹配
```

鸭子类型的关键：不要求显式 `implements` 接口，只要对象具备接口中定义的方法和属性，就认为符合该接口。

```ts
interface Duck {
    quack(): void;
    fly(): void;
}

// implements — 定义类时就报错
class Cat implements Duck {
    // 编译错误，缺少 quack() 和 fly()
}

// 鸭子类型 — 传入时才报错
class Fish {
    swim() { console.log("游"); }
}

function makeDuck(d: Duck) { /* ... */ }

makeDuck(new Fish());
// 编译错误，Fish 缺少 quack 和 fly
```

```ts
interface IPoint { 
    x:number 
    y:number 
} 
function addPoints(p1:IPoint,p2:IPoint):IPoint { 
    var x = p1.x + p2.x 
    var y = p1.y + p2.y 
    return {x:x,y:y} 
} 
 
// 正确
var newPoint = addPoints({x:3,y:4},{x:5,y:1})  
 
// 错误，p1缺少y属性
var newPoint2 = addPoints({x:1},{x:4,y:3})
```

鸭子类型是 TypeScript **结构化类型系统**的核心体现 —— 不关注"你是谁"，只关注"你有什么"：

* 只要对象具备接口要求的属性和方法，TypeScript 就认为它符合该接口，**无需显式 `implements`**
* `implements` 在**定义类时**就检查是否满足接口，不满足则立即报错
* 鸭子类型在**使用（传参）时**才检查，更具灵活性和隐式性
* 多余属性不影响匹配（如 `{ x, y, z }` 可传给 `{ x, y }` 类型的参数），缺少必需属性则报错
* 这是 TypeScript 与 Java/C# 等名义类型系统的关键区别之一

### 抽象类

抽象类不能被实例化，只能作为基类供子类继承。抽象类可以包含抽象方法（没有实现的方法）和具象方法（已有实现的方法），子类必须实现这些抽象方法。

```ts
// 使用 abstract 关键字声明抽象类
// 抽象类不能直接实例化，只能作为基类
abstract class Animal {
    // 动物的名字属性
    name: string;

    // 构造函数
    constructor(name: string) {
        this.name = name;
    }

    // 抽象方法：使用 abstract 修饰，没有方法体
    // 子类必须实现这个方法
    abstract speak(): void;

    // 具象方法：有具体实现的方法
    // 子类可以直接继承使用，不需要重写
    move(): void {
        console.log(this.name + " 在移动");
    }
}

// 尝试实例化抽象类会报错
// var animal = new Animal("动物"); // 错误：不能实例化抽象类

// 定义 Dog 类继承 Animal
class Dog extends Animal {
    // 子类必须实现父类的抽象方法 speak()
    speak(): void {
        console.log(this.name + " 汪汪汪!");
    }
}

// 创建 Dog 实例
var dog = new Dog("旺财");
dog.speak();  // 调用子类实现的方法
dog.move();   // 继承父类的具象方法
```

#### 抽象类作为类型

抽象类可以作为参数类型使用。这意味着函数可以接受任何抽象类的子类实例。

```ts
// 定义抽象类 Animal
abstract class Animal {
    // 抽象方法：所有动物都会发出声音
    abstract speak(): void;
}

// 定义 Cat 类继承 Animal
class Cat extends Animal {
    // 实现抽象方法
    speak(): void {
        console.log("喵喵喵!");
    }
}

// 定义 Dog 类继承 Animal
class Dog extends Animal {
    // 实现抽象方法
    speak(): void {
        console.log("汪汪汪!");
    }
}

// 定义函数参数类型为抽象类 Animal
// 这个函数可以接受任何 Animal 的子类实例
function makeSpeak(animal: Animal): void {
    animal.speak();
}

// 传入不同的子类实例，实现不同的行为（多态）
makeSpeak(new Cat());
makeSpeak(new Dog());

// 抽象类类型数组：可以存储不同子类实例
var animals: Animal[] = [new Cat(), new Dog()];
```

#### 抽象类小结

`abstract class` 介于接口和具体类之间：

* **不能实例化**，只能被继承
* **抽象方法**：子类必须实现；**具象方法**：子类直接继承使用
* 可作为参数类型，实现多态（接受任意子类实例）
* 与接口的区别：抽象类可以有实现代码，接口只有声明

### 综合实例

实例1

```ts
// 接口定义
interface IPrintable {
    print(): void;
}

// 抽象类
abstract class Item {
    protected name: string;
    protected price: number;

    constructor(name: string, price: number) {
        this.name = name;
        this.price = price;
    }

    abstract getDetails(): string;
}

// 具体类
class Product extends Item implements IPrintable {
    // 继承抽象类Item并实现接口IPrintable
    private category: string;

    constructor(name: string, price: number, category: string) {
        super(name, price);
        // 调用父类构造函数传入name和price
        this.category = category;
    }

    // 实现抽象方法
    getDetails(): string {
        return `产品: ${this.name}, 价格: ¥${this.price}, 类别: ${this.category}`;
    }

    // 实现接口方法
    print(): void {
        console.log(this.getDetails());
    }

    // 静态方法
    static create(name: string, price: number): Product {
        return new Product(name, price, "默认类别");
    }
}

// 使用
var product = Product.create("笔记本电脑", 5999);
product.print();

// 折扣方法
class DiscountedProduct extends Product {
    private discount: number;

    constructor(name: string, price: number, category: string, discount: number) {
        super(name, price, category);
        // 调用父类构造函数传入name、price和category
        this.discount = discount;
    }

    getDetails(): string {
        //重写getDetails方法
        var discountedPrice = this.price * (1 - this.discount / 100);
        return `产品: ${this.name}, 原价: ¥${this.price}, 折扣: ${this.discount}%, 现价: ¥${discountedPrice.toFixed(2)}`;
        //toFixed(2)保留两位小数
    }
}

var discountedProduct = new DiscountedProduct("手机", 2999, "电子产品", 20);
discountedProduct.print();
//调用父类print方法，输出重写后getDetails方法的结果
```

## 混入

Mixin 的核心是一个接收基类、返回扩展类的泛型函数。Constructor 类型约束用于描述"任意可被继承的类"，防止传入普通函数或对象。

### 基本概念

```ts
// Constructor 类型别名
// 描述"任意可以被 new 实例化的类"。
// new  (...args: any[]) => T 表示：
//   - 可以用 new 调用
//   - 接受任意数量和类型的参数（...args: any[]）
//   - 返回类型为 T 的实例
// T = {} 是默认泛型参数，表示"不关心实例具体长什么样"。
type Constructor<T = {}> = new (...args: any[]) => T;

// Mixin 函数：Timestamped
// 泛型参数 <TBase extends Constructor> 确保：
//   传入的 Base 必须是一个"可以被 extends 的类"，防止误传普通函数或对象。
// 返回值是一个匿名类，extends Base 继承了传入类的所有属性和方法，
// 同时新增一个 createdAt 属性，记录对象创建的时间戳。
function Timestamped<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        // 新增属性：每个实例创建时自动获取当前时间
        createdAt = new Date();
    };
}

// 普通的基类
class User {
    // 参数属性简写：public name 既是构造函数参数，又是实例属性
    constructor(public name: string) {}
}

// 使用 Mixin
// Timestamped(User) 将 User 类"喂"给 Mixin 函数，
// 返回一个融合了 User + Timestamped 功能的全新类。
// 注意：TimestampedUser 是一个类，不是实例。
const TimestampedUser = Timestamped(User);

// 用新类创建实例，传入的 "Alice" 最终会传给 User 的构造函数
const user = new TimestampedUser("Alice");

// 访问基类 User 自带的属性
console.log(user.name);                        // Alice

// 访问 Mixin 新增的属性
console.log(user.createdAt instanceof Date);   // true
```

简单例子：

```ts
// Mixin：给任何类添加 "加个尾巴" 的能力
function addTail(baseClass) {
    return class extends baseClass {
        tail = true;  // 新增尾巴属性
    };
}

// 原始类
class Dog {
    bark() { console.log("汪汪"); }
}

// 混入后得到新类
const DogWithTail = addTail(Dog);
const dog = new DogWithTail();

dog.bark();   // 汪汪（来自原类）
console.log(dog.tail); // true（来自 mixin）
```

### 组合多个Mixin

```ts
// Constructor 类型：描述任意"可以用 new 实例化"的类
//  - T 是泛型，默认为 {}，表示不限制实例的具体形状
//  - 用于约束 Mixin 的参数必须是类（不能是普通函数或对象）
type Constructor<T = {}> = new (...args: any[]) => T;

// ────── Mixin 1：给类加入"时间戳"能力 ──────
// 功能：为每个实例自动添加 createdAt 属性，记录创建时间
// 约束：只要求 Base 是一个类，不依赖任何特定方法
function Timestamped<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        createdAt = new Date();       // 实例创建时自动获取当前时间
    };
}

// ────── Mixin 2：给类加入"序列化"能力 ──────
// 功能：为实例添加 serialize() 方法，将自身转为 JSON 字符串
// 约束：只要求 Base 是一个类，不依赖任何特定方法
function Serializable<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        serialize(): string {
            return JSON.stringify(this);   // 将当前实例的所有属性序列化为 JSON
        }
    };
}

// ────── Mixin 3：给类加入"日志输出"能力 ──────
// 功能：让实例拥有 log() 方法，以 "[LOG]" 前缀输出序列化结果
// 依赖：要求 Base 必须已有 serialize() 方法（由 Mixin 2 或手动提供）
//       泛型约束 Constructor<{ serialize(): string }> 保证了类型安全
function Loggable<TBase extends Constructor<{ serialize(): string }>>(Base: TBase) {
    return class extends Base {
        log(): void {
            console.log("[LOG]", this.serialize());  // 调用基类的 serialize() 输出日志
        }
    };
}

// ────── 业务基类：产品 ──────
class Product {
    constructor(public name: string, public price: number) {}
}

// ─── 组合 A：仅时间戳 ───
// 场景：只需记录创建时间，不需要序列化和日志
const TimestampedProduct = Timestamped(Product);
const p1 = new TimestampedProduct("Book", 49);
console.log(p1.createdAt instanceof Date);  // true
// p1.serialize();  // ❌ 没有此方法
// p1.log();        // ❌ 没有此方法

// ─── 组合 B：时间戳 + 序列化 ───
// 场景：需要时间戳和 JSON 序列化（如缓存、存储），但不需要日志
const SaveableProduct = Serializable(Timestamped(Product));
const p2 = new SaveableProduct("Mouse", 199);
console.log(p2.createdAt instanceof Date);   // true
console.log(p2.serialize());                 // {"name":"Mouse","price":199,"createdAt":"..."}
// p2.log();  // ❌ 没有此方法

// ─── 组合 C：仅序列化 ───
// 场景：只需 JSON 序列化（如 API 请求体），不关心时间
const JsonProduct = Serializable(Product);
const p3 = new JsonProduct("Keyboard", 399);
console.log(p3.serialize());                 // {"name":"Keyboard","price":399}
// p3.log();          // ❌ 没有此方法（Loggable 依赖 serialize，但需主动叠加）
// console.log(p3.createdAt);  // ❌ 没有此属性

// ─── 组合 D：序列化 + 日志（无时间戳） ───
// 场景：需要调试日志和序列化输出，但不需要创建时间
const LoggedProduct = Loggable(Serializable(Product));
const p4 = new LoggedProduct("Monitor", 1599);
p4.log();                                    // "[LOG]" {"name":"Monitor","price":1599}
// console.log(p4.createdAt);  // ❌ 没有此属性（未叠加 Timestamped）

// ─── 组合 E：全部三种 ───
// 场景：完整功能——时间戳 + 序列化 + 日志
const FullProduct = Loggable(Serializable(Timestamped(Product)));
const p5 = new FullProduct("Laptop", 6999);
console.log(p5.createdAt instanceof Date);   // true
console.log(p5.serialize());                 // 含 createdAt
p5.log();                                    // "[LOG]" 含全部字段


// 五种组合共用同一个 `Product` 基类，每种组合都是独立的类，互不影响。
// 这就是 Mixin 相比继承的最大优势——不需要为每种能力组合创建单独的子类。
```

### 混入与接口结合

Mixin 返回的类可以声明实现某个接口，使消费方通过接口类型操作混入后的对象，而不依赖具体实现。

```ts
type Constructor<T = {}> = new (...args: any[]) => T;

// 定义能力接口
interface ISerializable {
    serialize(): string;
}

interface ICloneable<T> {
    clone(): T;
}

// Mixin 实现接口
function Serializable<TBase extends Constructor>(Base: TBase) {
    return class extends Base implements ISerializable {
        serialize(): string {
            return JSON.stringify(this);
        }
    };
}

function Cloneable<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        // 鸭子类型：实现了 clone() 方法，符合 ICloneable 接口
        clone() {
            return Object.assign(
                Object.create(Object.getPrototypeOf(this)),
                this
            );
        }
    };
}

class Article {
    constructor(public title: string, public content: string) {}
}

const RichArticle = Cloneable(Serializable(Article));

const a1 = new RichArticle("TypeScript 入门", "正文内容...");
const a2 = a1.clone();
a2.title = "TypeScript 进阶";

console.log(a1.serialize());
// {"title":"TypeScript 入门","content":"正文内容..."}

console.log(a2.serialize());
// {"title":"TypeScript 进阶","content":"正文内容..."}

console.log(a1.title === a2.title);  // false（克隆后独立修改）
```

### 带约束的混入

通过泛型约束，可以限定 Mixin 只能混入满足特定条件的类，避免运行时缺少必要属性。

```ts
type Constructor<T = {}> = new (...args: any[]) => T;

// 约束：基类必须有 id 和 name 属性
type WithIdAndName = Constructor<{ id: number; name: string }>;
// 相当于：type WithIdAndName = new (...args: any[]) => { id: number; name: string };

function Printable<TBase extends WithIdAndName>(Base: TBase) {
    return class extends Base {
        print(): void {
            console.log(`[${this.id}] ${this.name}`);
        }
    };
}

class Item {
    constructor(public id: number, public name: string) {}
}

// 正确：Item 满足约束
const PrintableItem = Printable(Item);
const item = new PrintableItem(42, "Keyboard");
item.print();  // [42] Keyboard

// 错误示例（编译器会阻止）：
// class NoId { constructor(public name: string) {} }
// const Bad = Printable(NoId);  // 错误：NoId 缺少 id 属性
```

### Mixin 与继承的对比

Mixin 和继承是两种不同的代码复用方式，各有适用场景：

```ts
// ── 继承方式：纵向关系 ──
// 问题：要组合 N 种能力，需要 2^N 个子类
class Animal { eat() {} }
class FlyableAnimal extends Animal { fly() {} }
class SwimmableAnimal extends Animal { swim() {} }
// class FlyableSwimmableAnimal extends ???   // 无法同时继承两个！

// ── Mixin 方式：横向组合 ──
// 每种能力独立定义，按需组合
type Constructor<T = {}> = new (...args: any[]) => T;

function Flyable<TBase extends Constructor>(Base: TBase) {
    return class extends Base { fly() { console.log("飞"); } };
}
function Swimmable<TBase extends Constructor>(Base: TBase) {
    return class extends Base { swim() { console.log("游"); } };
}

class Bird extends Animal {}
const Duck = Flyable(Swimmable(Bird));  // 鸟 + 飞 + 游
const FlyingFish = Flyable(Swimmable(Animal)); // 动物 + 飞 + 游
```

| 对比维度 | 继承（extends） | Mixin |
| :--- | :--- | :--- |
| 关系 | "is-a"（垂直父子） | "has-a" / "can-do"（水平能力） |
| 组合性 | 单一父类，无法同时继承多个 | 可叠加任意多个，自由组合 |
| 耦合度 | 子类与父类强耦合 | 各 Mixin 独立，松耦合 |
| 灵活性 | 修改父类影响所有子类 | 增删 Mixin 不影响其他组合 |
| 适用场景 | 明确的层级关系（Animal→Bird） | 跨领域的能力混入（日志、序列化、缓存等） |

**选择原则**：当不同类需要共享**不相关**的能力时用 Mixin，当存在**天然层级**时用继承。两者可配合使用——继承表达"是什么"，Mixin 表达"能做什么"。

### 混入小结

Mixin 通过泛型函数实现"组合优于继承"，突破单继承限制：

* **核心模式**：函数接收基类，返回扩展后的匿名类（`extends Base`）
* **`Constructor<T>` 类型**：约束参数必须是"能 new 的类"，防止误传，`<T>` 泛型约束返回值类型，表示必须包含特定属性或方法。
* **组合叠加**：从内到外依次嵌套，后加的 Mixin 可依赖先加的（如 `Loggable` 依赖 `Serializable` 提供的 `serialize()`）
* **与继承的区别**：Mixin 是横向能力组合，继承是纵向层级关系，二者可配合使用
* **适用场景**：需要灵活组合多种能力（时间戳、序列化、日志等），避免深层继承树

## 装饰器

装饰器（Decorators）是 TypeScript 的一项实验性功能。它允许开发者在不修改原类的情况下，为类、方法、属性或参数添加额外的功能。

### 装饰器配置

在使用装饰器之前，需要在 TypeScript 配置文件 tsconfig.json 中启用相关编译选项。

```json
{
    "compilerOptions": {
        // 启用装饰器语法
        "experimentalDecorators": true,

        // 启用装饰器元数据（用于依赖注入框架）
        "emitDecoratorMetadata": true
    }
}
```

* experimentalDecorators：启用装饰器语法支持，这是使用装饰器的前提条件。
* emitDecoratorMetadata：在编译后的 JavaScript 中生成装饰器的元数据，供依赖注入框架使用。

### 类装饰器

类装饰器应用于类的构造函数，可以修改类的定义或添加额外的处理逻辑。类装饰器接收一个参数，即目标类的构造函数。

```ts
// 定义一个类装饰器函数 —— sealed（密封）
// 类装饰器本质上是一个函数，它接收一个参数：
//   target —— 被装饰的类（也就是类本身，即构造函数）
// 装饰器会在类定义完成后、类被使用之前自动执行。
function sealed(target: Function) {
    // 输出日志，确认装饰器已被触发，并显示被装饰的类名
    console.log("装饰器 applied to: " + target.name);
    // --------------------------------------------------------
    // Object.seal() 的作用：
    //   - 禁止向对象添加新属性
    //   - 禁止删除已有属性
    //   - 但允许修改已有属性的值（与 Object.freeze 不同）
    // --------------------------------------------------------
    // 密封构造函数本身（静态成员无法新增/删除）
    Object.seal(target);
    // 密封原型对象（实例成员无法新增/删除）
    // 注意：new Person() 创建的是 target.prototype 的实例，
    //       所以必须同时密封 target 和 target.prototype
    Object.seal(target.prototype);
}


// 使用 @sealed 语法将装饰器"安装"到 Person 类上
// 等价于：class Person { ... } 之后执行 sealed(Person)
// 装饰器在类定义阶段就会执行，而不是在 new 的时候执行。
@sealed
class Person {
    name: string;

    constructor(name: string) {
        this.name = name;
    }
}

// 测试：创建实例
// 因为装饰器在类定义时就已经执行了，
// 所以在创建第一个实例之前，"装饰器 applied to: Person" 已经被打印。
var person = new Person("TEST");
console.log("创建: " + person.name);   // 创建: TEST


// 验证密封效果：尝试添加新属性
// 由于原型已被 Object.seal，无法向实例动态添加新属性：
//   person.age = 25;
// 在非严格模式下，赋值操作会静默失败（不报错但也不生效）；
// 在严格模式（'use strict'）下，会抛出 TypeError。
// 取消下面这行注释即可观察效果：
// person.age = 25; // 静默失败（非严格模式）或抛出 TypeError（严格模式）
```

类装饰器就是「在类定义完后自动执行一个函数」的语法糖，把类本身作为参数传进去， 对类做一些额外处理。本质上能对JS类做的任何操作都能放进装饰器里，`Object.seal`只是个例子。

### 方法装饰器

方法装饰器应用于类的方法，可以修改方法的定义或添加额外的处理逻辑。方法装饰器接收三个参数，由 TypeScript 在类定义完成后自动传入：

1. **target**：对于静态方法，是类的构造函数；对于实例方法，是类的原型对象（如 `Greeter.prototype`）。
2. **propertyKey**：方法名称的字符串（如 `"greet"`）。
3. **descriptor**：方法的属性描述符，一个包含 `value`（方法本身）、`writable`（可修改性）、`enumerable`（可遍历性）和 `configurable`（可重配性）的对象。

方法装饰器可以通过修改 `descriptor` 来改变方法的行为特性。如果需要传参，则需要用装饰器工厂（外层函数接收参数，内层函数接收上述三个参数）。

```ts
// ────── 方法装饰器工厂 ──────
// 外层函数 enumerable 是"装饰器工厂"——用来接收调用者传入的参数（true/false）。
// 写法 @enumerable(true) 时，先调外层传入 true，返回内层真正的装饰器函数。
// 如果不需要传参，可以省略外层工厂，直接写装饰器函数。
function enumerable(value: boolean) {
    // ─── 内层函数才是真正的装饰器 ───
    // 以下三个参数由 TypeScript 在类定义完成后自动传入：
    return function (
        target: any,           // 参数1：被装饰方法所在的原型对象
                               //   → Greeter.prototype（因为 greet 是实例方法）
                               //   → 如果装饰的是静态方法，则 target 是构造函数本身
        propertyKey: string,    // 参数2：被装饰的方法名
                               //   → "greet"
        descriptor: PropertyDescriptor // 参数3：方法的属性描述符
                               //   → { value: greet函数, writable: true,
                               //       enumerable: false, configurable: true }
    ) {
        // ─── 核心操作 ───
        // descriptor.enumerable 控制该方法能否被 for...in 遍历到：
        //   - false（默认）：方法不可遍历（如普通的 greet）
        //   - true：方法可被 for...in 遍历到
        // 这里将传入的 value（true/false）赋值给 enumerable，
        // 相当于修改了 Greeter.prototype 上 greet 方法的"遍历开关"。
        descriptor.enumerable = value;
    };
}

class Greeter {
    greeting: string;          // ← 实例属性，定义时声明

    constructor(message: string) {
        this.greeting = message; // ← greeting 赋值到实例 g 自身上（可枚举）
    }

    // ↓ 应用装饰器工厂，传入 true
    //    效果等同于在类定义完成后自动执行：
    //      enumerable(true)(Greeter.prototype, "greet", descriptor)
    //    将 greet 的 enumerable 从默认的 false 改为 true
    @enumerable(true)
    greet() {                    // ← 原型方法，默认 enumerable = false（不可遍历）
        return "Hello, " + this.greeting;
    }
}

var g = new Greeter("World");

// 检查 greet 方法是否可枚举
// 因为 @enumerable(true) 把 enumerable 改成了 true，
// 所以这里返回 true
console.log("方法可枚举: " + g.propertyIsEnumerable("greet"));

// 遍历对象 g 的所有可枚举属性
// 由于 @enumerable(true) 让 greet 变为可枚举，
// 所以 for...in 会遍历到：
//   1. greeting（实例属性，默认可枚举）
//   2. greet（原型方法，被装饰器改为可枚举）
// 由于TypeScript不允许对构造函数应用装饰器，
// 所以构造函数不会出现在遍历结果中
for (var key in g) {
    console.log("属性: " + key);
}
// 输出：
//   方法可枚举: true
//   属性: greeting
//   属性: greet
```

### 访问器装饰器

访问器装饰器应用于类的 `getter` 和 `setter` 方法。与方法装饰器类似，访问器装饰器也接收三个参数，由 TypeScript 在类定义完成后自动传入：

1. **target**：对于静态访问器，是类的构造函数；对于实例访问器，是类的原型对象（如 `Point.prototype`）。
2. **propertyKey**：访问器的属性名称（如 `"x"` 或 `"y"`）。
3. **descriptor**：访问器的属性描述符，一个包含 `get`（getter 函数）、`set`（setter 函数）、`enumerable`（可遍历性）和 `configurable`（可重配性）的对象。与方法装饰器不同，访问器描述符包含的是 `get`/`set` 而非 `value`。

访问器装饰器可以通过修改 `descriptor` 来改变访问器的行为特性。如果需要传参，则需要用装饰器工厂（外层函数接收参数，内层函数接收上述三个参数）。

**注意**：访问器装饰器不能同时应用于同一个属性的 `getter` 和 `setter`，只能选择其中一个应用。因为 TypeScript 会将 getter 和 setter 视为同一个属性描述符，装饰器会合并应用到两者上。

```ts
// ────── 访问器装饰器工厂 ──────
// 外层函数 configurable 是"装饰器工厂"——用来接收调用者传入的参数（true/false）。
// 写法 @configurable(false) 时，先调外层传入 false，返回内层真正的装饰器函数。
function configurable(value: boolean) {
    // ─── 内层函数才是真正的装饰器 ───
    // 以下三个参数由 TypeScript 在类定义完成后自动传入：
    return function (
        target: any,           // 参数1：所属类的原型对象
                               //   → Point.prototype（因为 get x 是实例访问器）
                               //   → 如果是静态访问器，则 target 是构造函数本身
        propertyKey: string,    // 参数2：访问器的属性名
                               //   → "x" 或 "y"
        descriptor: PropertyDescriptor // 参数3：访问器的属性描述符
                               //   → { get: getter函数, set: setter函数,
                               //       enumerable: true, configurable: true }
    ) {
        // ─── 核心操作 ───
        // descriptor.configurable 控制该访问器属性描述符能否被修改或删除：
        //   - true（默认）：可重新配置（Object.defineProperty）或删除（delete）
        //   - false：锁定访问器本身，后续无法修改其描述符，也无法删除
        // 这里将传入的 value（true/false）赋值给 configurable，
        // 相当于锁定或解锁 Point.prototype 上 x/y 访问器的"可重配开关"。
        descriptor.configurable = value;
    };
}

class Point {
    private _x: number = 0;   // ← 私有字段，存储 x 的实际值
    private _y: number = 0;   // ← 私有字段，存储 y 的实际值

    // ↓ 应用装饰器工厂，传入 false
    //    效果等同于在类定义完成后自动执行：
    //      configurable(false)(Point.prototype, "x", descriptor)
    //    将 x 访问器的 configurable 从默认的 true 改为 false
    @configurable(false)
    get x() {                    // ← getter：读取 _x 的值
        return this._x;
    }

    @configurable(false)
    get y() {                    // ← getter：读取 _y 的值
        return this._y;
    }

    // setter 没有加装饰器，但 getter 加了 @configurable(false) 后，
    // 整个 x/y 属性的 configurable 都会被锁定（getter 和 setter 共享同一个描述符）
    set x(value: number) {       // ← setter：修改 _x 的值
        this._x = value;
    }

    set y(value: number) {       // ← setter：修改 _y 的值
        this._y = value;
    }
}

var point = new Point();
point.x = 10;   // 调用 set x(10)，将 _x 设为 10
point.y = 20;   // 调用 set y(20)，将 _y 设为 20
console.log("坐标: (" + point.x + ", " + point.y + ")");
// 输出：坐标: (10, 20)

// 注意：configurable 只影响"访问器本身"（getter/setter 函数的替换、
// 描述符的修改、属性的删除），完全不影响"值的读写"。
// point.x 读取 _x     → 正常
// point.x = 100 写入  → 正常
// delete point.x      → 被阻止
// Object.defineProperty 重新配置 → 被阻止
```

### 属性装饰器

属性装饰器应用于类的属性定义。与方法装饰器和访问器装饰器不同，属性装饰器**只接收两个参数**，没有属性描述符（descriptor），因为实例属性在定义时还没有被初始化，无法获取描述符。

1. **target**：对于静态属性，是类的构造函数；对于实例属性，是类的原型对象。
2. **propertyKey**：属性名称。

属性装饰器不能直接修改属性的值或行为，常用于**存储元数据**（在原型上记录额外的信息，供后续使用）。

```ts
// ────── 属性装饰器工厂 ──────
// 外层函数 format 是"装饰器工厂"——用来接收调用者传入的格式字符串。
// 写法 @format("YYYY-MM-DD") 时，先调外层传入格式字符串，返回内层真正的装饰器函数。
function format(formatString: string) {
    // ─── 内层函数才是真正的装饰器 ───
    // 以下两个参数由 TypeScript 在类定义完成后自动传入：
    return function (
        target: any,           // 参数1：所属类的原型对象
                               //   → User.prototype（因为 birthDate 是实例属性）
                               //   → 如果是静态属性，则 target 是构造函数本身
        propertyKey: string     // 参数2：属性名称
                               //   → "birthDate"
    ) {
        // ─── 核心操作 ───
        // 在原型对象上添加一个新属性 "birthDate_format"，
        // 用来存储这个属性的"格式元数据"。
        // 由于属性装饰器没有 descriptor，无法直接修改属性本身的行为，
        // 所以常见的做法是在原型上"附加"元数据信息，供其他地方使用。
        // 使用 propertyKey + "_format" 作为新属性名，避免与原有属性冲突。
        Object.defineProperty(target, propertyKey + "_format", {
            value: formatString,  // 存储格式字符串，如 "YYYY-MM-DD"
            writable: false,      // 不可修改
            enumerable: false,    // 不可遍历（隐藏元数据）
            configurable: true    // 可删除或重配
        });
    };
}

class User {
    // ↓ 应用属性装饰器工厂，传入 "YYYY-MM-DD"
    //    效果等同于在类定义完成后自动执行：
    //      format("YYYY-MM-DD")(User.prototype, "birthDate")
    //    在 User.prototype 上添加一个隐藏属性 birthDate_format = "YYYY-MM-DD"
    @format("YYYY-MM-DD")
    birthDate: string;         // ← 实例属性，在 new 时由构造函数赋值

    constructor(birthDate: string) {
        this.birthDate = birthDate;  // ← 将传入的值赋给实例的 birthDate
    }
}

var user = new User("1990-01-01");
console.log("出生日期: " + user.birthDate);
// 输出：出生日期: 1990-01-01

// 访问存储的元数据
// birthDate_format 被定义在 User.prototype 上，实例通过原型链继承到
// 使用 (user as any) 绕过类型检查，因为 birthDate_format 不是 User 类型声明的属性
console.log("日期格式: " + (user as any).birthDate_format);
// 输出：日期格式: YYYY-MM-DD

// 原型链结构：
// user 实例:
//   ├── birthDate: "1990-01-01"     ← 实例自身的属性
//   └── [[Prototype]] → User.prototype
//                        ├── constructor: User
//                        └── birthDate_format: "YYYY-MM-DD"  ← 装饰器添加的元数据（不可枚举）
```

### 参数装饰器

参数装饰器应用于类方法中**某个参数**的前面。与方法装饰器和访问器装饰器不同，参数装饰器**没有属性描述符**（descriptor），它只用来记录或验证参数信息，不能修改参数本身。

参数装饰器接收三个参数：

1. **target**：对于静态方法，是类的构造函数；对于实例方法，是类的原型对象。
2. **propertyKey**：方法名称的字符串。
3. **parameterIndex**：被装饰的参数在方法参数列表中的索引位置（从 0 开始）。

```ts
// ────── 参数装饰器（不是工厂，不需要传参） ──────
// 参数装饰器直接写成函数形式，不需要外层工厂包装。
// 以下三个参数由 TypeScript 在类定义完成后自动传入：
function logParameter(
    target: any,               // 参数1：所属类的原型对象
                               //   → Greeter.prototype（因为 greet 是实例方法）
                               //   → 如果是静态方法，则 target 是构造函数本身
    propertyKey: string,       // 参数2：方法名称
                               //   → "greet"
    parameterIndex: number     // 参数3：被装饰的参数在参数列表中的位置（从 0 开始）
                               //   → 0（因为 name 是 greet 的第一个参数）
) {
    // ─── 核心操作 ───
    // 参数装饰器没有 descriptor，无法修改参数或方法的行为。
    // 通常用于副效应，比如记录日志、验证参数、收集元数据等。
    console.log("参数装饰器: " + propertyKey +
        " 第 " + (parameterIndex + 1) + " 个参数");
    // 输出：参数装饰器: greet 第 1 个参数
}

class Greeter {
    greeting: string;

    constructor(greeting: string) {
        this.greeting = greeting;
    }

    // ↓ 在参数前应用装饰器（不需要 @xxx() 调用，直接写 @函数名）
    //    效果等同于在类定义完成后自动执行：
    //      logParameter(Greeter.prototype, "greet", 0)
    greet(@logParameter name: string) {
        return this.greeting + ", " + name;
    }
}

var greeter = new Greeter("Hello");
greeter.greet("RUNOOB");
// 输出：
//   参数装饰器: greet 第 1 个参数   ← 类定义时执行
//   Hello, RUNOOB                   ← 方法调用时执行
```

### 装饰器工厂

装饰器工厂是返回装饰器函数的函数。通过装饰器工厂，可以在应用装饰器时传入自定义参数，实现更灵活的配置。

前面的 `enumerable`、`configurable`、`format` 示例都是装饰器工厂。这里的 `color` 示例是一个方法装饰器工厂，它通过替换 `descriptor.value` 来**增强原始方法**的行为。

```ts
// ────── 装饰器工厂 ──────
// 外层函数 color 是"装饰器工厂"——用来接收调用者传入的颜色代码。
// 写法 @color("34") 时，先调外层传入 "34"，返回内层真正的装饰器函数。
function color(colorCode: string) {
    // colorCode 是 ANSI 转义序列的颜色代码
    // 例如：34 = 蓝色，31 = 红色，32 = 绿色

    // ─── 内层函数才是真正的装饰器 ───
    // 以下三个参数由 TypeScript 在类定义完成后自动传入：
    return function (
        target: any,           // 参数1：所属类的原型对象
                               //   → Logger.prototype（log/error/success 都是实例方法）
        propertyKey: string,    // 参数2：方法名称
                               //   → "log"、"error" 或 "success"
        descriptor: PropertyDescriptor // 参数3：方法的属性描述符
                               //   → { value: 原函数, writable: true,
                               //       enumerable: false, configurable: true }
    ) {
        // ─── 核心操作 ───
        // 保存原始方法的引用，以便在增强后的方法中调用它
        var originalMethod = descriptor.value;

        // 替换 descriptor.value 为一个新的函数（包装原始方法）
        // 这就是"装饰器增强方法"的经典模式：
        //   1. 保存原始方法
        //   2. 用新函数替换它
        //   3. 新函数内部调用原始方法，并在前后添加额外逻辑
        descriptor.value = function (...args: any[]) {
            // ① 调用原始方法，获取返回值
            var result = originalMethod.apply(this, args);

            // ② 在返回值外面包裹 ANSI 颜色转义码
            //    \x1b[34m  → 设置终端文字为蓝色
            //    \x1b[0m   → 重置终端文字颜色
            //    result    → 原始方法的返回值
            // 最终返回："\x1b[34m消息内容\x1b[0m"
            return "\x1b[" + colorCode + "m" + result + "\x1b[0m";
        };
    };
}

class Logger {
    // ↓ 应用装饰器工厂，传入蓝色代码 34
    //    效果等同于在类定义完成后自动执行：
    //      color("34")(Logger.prototype, "log", descriptor)
    //    将 log 方法替换为带蓝色 ANSI 转义码的包装函数
    @color("34")
    log(message: string): string {
        return message;
    }

    @color("31")  // 红色
    error(message: string): string {
        return message;
    }

    @color("32")  // 绿色
    success(message: string): string {
        return message;
    }
}

var logger = new Logger();
console.log(logger.log("这是蓝色日志"));
console.log(logger.error("这是红色错误"));
console.log(logger.success("这是绿色成功"));
// 输出（在终端中会显示对应颜色）：
//   这是蓝色日志    ← 蓝色文字
//   这是红色错误    ← 红色文字
//   这是绿色成功    ← 绿色文字
```

### 装饰器执行顺序

当一个类上有多个装饰器时，执行顺序遵循特定规则。理解执行顺序的关键是区分**两阶段**：

1. **工厂求值阶段**：装饰器工厂的外层函数**从上到下**依次执行，收集返回的装饰器函数
2. **装饰器执行阶段**：收集到的装饰器函数**从下到上**依次执行

此外，不同类型的装饰器也有先后顺序：**参数装饰器 → 方法/属性/访问器装饰器 → 类装饰器**。

```ts
// ────── 装饰器工厂 1 ──────
function first() {
    // 外层函数：工厂求值阶段执行
    console.log("first 工厂求值");
    // 返回内层函数：装饰器执行阶段执行
    return function (target: any) {
        console.log("first 装饰器执行");
    };
}

// ────── 装饰器工厂 2 ──────
function second() {
    // 外层函数：工厂求值阶段执行
    console.log("second 工厂求值");
    // 返回内层函数：装饰器执行阶段执行
    return function (target: any) {
        console.log("second 装饰器执行");
    };
}

// ↓ 多个装饰器叠加在同一个类上
//    执行顺序分析：
//
//    阶段1——工厂求值（从上到下）：
//      ① @first()  → 打印 "first 工厂求值"，返回内层装饰器函数
//      ② @second() → 打印 "second 工厂求值"，返回内层装饰器函数
//
//    阶段2——装饰器执行（从下到上）：
//      ① @second 返回的装饰器函数执行 → 打印 "second 装饰器执行"
//      ② @first 返回的装饰器函数执行   → 打印 "first 装饰器执行"
//      ③ 最后执行类装饰器（这里没有单独的类装饰器）

@first()
@second()
class MyClass {
    name: string;
}

var obj = new MyClass();
// 输出顺序：
//   first 工厂求值         ← 阶段1：从上到下
//   second 工厂求值        ← 阶段1：从上到下
//   second 装饰器执行      ← 阶段2：从下到上
//   first 装饰器执行       ← 阶段2：从下到上
```

### 实际应用场景

#### 日志记录

自动记录方法调用日志。

```ts
// 日志装饰器
function log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    var originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log("调用方法: " + propertyKey + "，参数: " + JSON.stringify(args));
        var result = originalMethod.apply(this, args);
        console.log("方法返回: " + JSON.stringify(result));
        return result;
    };
}

class MathService {
    @log
    add(a: number, b: number): number {
        return a + b;
    }

    @log
    multiply(a: number, b: number): number {
        return a * b;
    }
}

var math = new MathService();
console.log("计算结果: " + math.add(5, 3));
```

#### 权限验证

实现方法级别的权限检查。

```ts
// 模拟当前用户角色
var currentUser = { role: "admin" };

// 权限装饰器
function requireRole(role: string) {
    return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
        var originalMethod = descriptor.value;

        descriptor.value = function (...args: any[]) {
            if (currentUser.role !== role) {
                console.log("权限不足，无法执行 " + propertyKey);
                return null;
            }
            return originalMethod.apply(this, args);
        };
    };
}

class AdminService {
    @requireRole("admin")
    deleteUser(id: number): string {
        return "删除用户 " + id + " 成功";
    }

    @requireRole("admin")
    viewUser(id: number): string {
        return "查看用户 " + id;
    }
}

var admin = new AdminService();
console.log(admin.viewUser(1));
console.log(admin.deleteUser(1));

// 模拟普通用户
currentUser = { role: "user" };
console.log(admin.deleteUser(2));
```
