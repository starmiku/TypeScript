# 接口与类

接口与类是TypeScript中非常重要的概念，它们为对象提供了结构和行为。

## 接口

接口是一系列抽象方法的声明，是一些方法特征的集合，这些方法都应该是抽象的，需要由具体的类去实现。

```ts
interface IPerson { 
    firstName:string, 
    lastName:string, 
    sayHi: ()=>string 
} 
 
var customer:IPerson = { 
    firstName:"Tom",
    lastName:"Hanks", 
    sayHi: ():string =>{return "Hi there"} 
} 
 
console.log("Customer 对象 ") 
console.log(customer.firstName) 
console.log(customer.lastName) 
console.log(customer.sayHi())  
 
var employee:IPerson = { 
    firstName:"Jim",
    lastName:"Blakes", 
    sayHi: ():string =>{return "Hello!!!"} 
} 
 
console.log("Employee  对象 ") 
console.log(employee.firstName) 
console.log(employee.lastName)
```

### 联合类型接口

联合类型接口表示一个值可以是几种类型之一。

```ts
interface RunOptions { 
    program:string; 
    commandline:string[]|string|(()=>string); 
} 
 
// commandline 是字符串
var options:RunOptions = {program:"test1",commandline:"Hello"}; 
console.log(options.commandline)  
 
// commandline 是字符串数组
options = {program:"test1",commandline:["Hello","World"]}; 
console.log(options.commandline[0]); 
console.log(options.commandline[1]);  
 
// commandline 是一个函数表达式
options = {program:"test1",commandline:()=>{return "**Hello World**";}}; 
 
var fn:any = options.commandline; 
console.log(fn());
```

联合类型接口允许接口属性为多种类型的组合（`string | string[] | (() => string)`），使同一个属性可接受不同形态的值，适配灵活的输入场景。

### 接口和数组

接口中可以将数组的索引值和元素设置为不同类型，索引值可以是数字或字符串。

```ts
interface namelist {
   [index:number]:string
}

// 类型一致，正确
var list2:namelist = ["Google","Runoob","Taobao"]
// 错误元素 1 不是 string 类型
// var list2:namelist = ["Runoob",1,"Taobao"]
```

接口可用索引签名约束数组元素类型，如 `[index: number]: string` 表示所有元素必须为 string。

### 接口继承

接口可以通过extends关键字继承其他接口，也可以继承类。

接口继承语法格式：

`interface 子接口 extends 父接口1, 父接口2, ...`

接口继承实例：

```ts
interface IParent1 { 
    v1:number 
} 
 
interface IParent2 { 
    v2:number 
} 
 
interface Child extends IParent1, IParent2 { 
    v3: string
} 

var Iobj:Child = { v1:12, v2:23, v3:"This is child"} 
console.log("value 1: "+Iobj.v1+" value 2: "+Iobj.v2+" value 3: "+Iobj.v3)
```

接口通过 `extends` 支持多继承（`Child extends Parent1, Parent2`），子接口合并所有父接口的成员，实现接口层面的能力组合。

## 类

类是面向对象编程（OOP）的基础，它提供了一种结构化的方式来定义对象和它们的行为。

### 类的定义

一个类可以包含以下成员：

* 字段：类中声明的变量，表示对象的属性。
* 构造函数：类实例化时调用的特殊方法，用于初始化对象。
* 方法：类中定义的函数，表示对象的行为。

类定义的语法格式如下：

```ts
class 类名 {
    字段1: 类型;
    字段2: 类型;
    ...
    构造函数(参数1: 类型, 参数2: 类型, ...): void {
        // 初始化代码
    }
    方法1(参数1: 类型, 参数2: 类型, ...): 返回类型 {
        // 方法实现
    }
    ...
}
```

类的定义包含三大成员：字段（属性）、构造函数（初始化）、方法（行为）。构造函数在 `new` 时自动调用，用于初始化字段。

### 类的字段和构造函数

字段是存储对象数据的地方，构造函数在对象创建时自动调用，用于初始化字段。

```ts
// 定义 Car 类
class Car {
    // 字段：描述汽车的属性
    engine: string;

    // 构造函数：用于接受外来数据并初始化
    // 当所有字段都有默认值或不需要初始化的情况下不需要构造函数
    constructor(engine: string) {
        this.engine = engine;
    }

    // 方法：显示发动机信息
    disp(): void {
        console.log("发动机型号: " + this.engine);
    }
}

// 创建类的实例
var car = new Car("V8 发动机");

// 访问字段
console.log("读取发动机: " + car.engine);

// 调用方法
car.disp();
```

字段存储对象数据，构造函数在实例化时自动调用并初始化字段。若所有字段都有默认值，构造函数可省略。

### 访问控制修饰符

TypeScript提供了三种访问修饰符来控制类成员的可访问性：

* public：公共成员，可以在任何地方访问。
* private：私有成员，只能在类内部访问。
* protected：受保护成员，只能在类内部和子类中访问。

#### public

public 是默认的访问修饰符，表示成员可以在任何地方被访问。无论是类内部、子类还是类的外部，都可以访问 public 成员。如果不写访问修饰符，TypeScript会默认使用public，所以public可以省略。

```ts
class Person {
    public name: string;    // 公有属性
    public age: number;     // 公有属性

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    public introduce(): void {
        console.log("我是 " + this.name + "，今年 " + this.age + " 岁");
    }
}

var person = new Person("Alice", 25);
console.log("姓名: " + person.name);   // 可以访问
person.introduce();                    // 可以访问
```

#### private

private 表示私有成员，只能在定义它的类内部访问。子类和类的外部都不能访问 private 成员。这常用于隐藏类的内部实现细节，保护数据不被意外修改。

```ts
class Person {
    public name: string;
    private secret: string;  // 私有属性，外部无法直接访问

    constructor(name: string, secret: string) {
        this.name = name;
        this.secret = secret;
    }

    // 公有方法可以访问私有属性
    public revealSecret(): void {
        console.log("秘密: " + this.secret);
    }
}

var person = new Person("Alice", "我喜欢编程");

console.log("姓名: " + person.name);   // 可以访问
// console.log(person.secret);        // 错误：'secret' 是私有属性

person.revealSecret();                 // 通过公有方法访问私有属性
```

#### protected

protected 表示受保护成员，可以在类内部和子类中访问。类的外部不能直接访问 protected 成员。这在需要让子类继承父类的某些功能，同时隐藏实现细节时非常有用。

```ts
class Person {
    protected name: string;  // 受保护属性

    constructor(name: string) {
        this.name = name;
    }

    protected sayHello(): void {
        console.log("你好，我是 " + this.name);
    }
}

class Student extends Person {
    private grade: string;

    constructor(name: string, grade: string) {
        super(name);
        this.grade = grade;
    }

    public introduce(): void {
        // 子类可以访问受保护的属性和方法
        console.log("我是 " + this.name + "，年级: " + this.grade);
        this.sayHello();
    }
}

var student = new Student("Bob", "高三");
student.introduce();    // 通过公有方法访问

// console.log(student.name);  // 错误：'name' 是受保护属性
```

#### readonly修饰符

readonly 用于将属性设置为只读。只能在声明时或构造函数中赋值，之后不能修改。这在定义常量或标识符时非常有用。

```ts
// 定义用户类
class User {
    // 使用 readonly 修饰的属性只能在初始化时赋值
    // 用户 ID
    readonly id: number;
    // 用户名
    readonly name: string;

    constructor(id: number, name: string) {
        this.id = id;
        this.name = name;
    }
}

// 创建用户实例
var user = new User(1, "Alice");

console.log("用户: " + user.id + ", " + user.name);

// 错误：不能修改 readonly 属性
// user.id = 2; // 编译错误！
// user.name = "Bob"; // 编译错误！
```

#### 参数属性

参数属性是一种简写方式，可以在构造函数中声明并初始化字段。使用`public`、`private`或`protected`关键字来定义参数属性。

```ts
// 定义点类
class Point {
    // 在构造函数参数上直接使用修饰符
    // 相当于同时声明属性并赋值
    constructor(
        // public 修饰：创建公开属性 x
        public x: number,
        // public 修饰：创建公开属性 y
        public y: number,
        // private 修饰：创建私有属性 z
        private z: number
    ) {
        // 构造函数体可以为空，属性已自动创建
    }

    // 计算三维度总和
    public sum(): number {
        // 可以在类内部访问所有属性
        return this.x + this.y + this.z;
    }
}

// 创建点实例
var point = new Point(1, 2, 3);

// 公开属性可以从外部访问
console.log("x: " + point.x);
console.log("y: " + point.y);

// 调用方法
console.log("总和: " + point.sum());

// 错误：私有属性不能从外部访问
// console.log(point.z); // 编译错误！
```

#### 访问控制修饰符小结

三种修饰符 + 两种特殊修饰符控制类成员的可访问性：

- **`public`**（默认）：任意位置可访问，可省略不写
- **`private`**：仅类内部可访问，子类和外部均不可见
- **`protected`**：类内部 + 子类可访问，外部不可见
- **`readonly`**：只在声明或构造函数中赋值，之后不可改
- **参数属性**：构造函数参数加修饰符（`public x: number`），等同于声明属性并赋值

### 类的继承

继承允许创建一个类（子类），作为另一个类（父类）的扩展。子类可以复用父类的代码，还可以扩展或重写父类的行为。

基本语法如下：

`class 子类 extends 父类 { ... }`

#### 单继承

```ts
// 父类：形状
class Shape {
    area: number;

    constructor(a: number) {
        this.area = a;
    }
}

// 子类：圆，继承自 Shape
class Circle extends Shape {
    disp(): void {
        console.log("圆的面积: " + this.area);
    }
}

var circle = new Circle(223);
circle.disp();
```

#### 多层继承

TypeScript不支持多继承（一个类继承多个类），但支持多层继承（A 继承 B，B 继承 C）。

```ts
// 根类
class Root {
    str: string;
}

// 子类：继承 Root
class Child extends Root {
}

// 叶子类：继承 Child（多层继承）
class Leaf extends Child {
}

var leaf = new Leaf();
leaf.str = "hello";
console.log("str 值: " + leaf.str);
```

#### 类的继承小结

TypeScript 只支持单继承（一个类只能继承一个父类），但支持多层继承（A→B→C）：

- 子类用 `extends` 继承父类所有 public/protected 成员
- 构造函数中必须先用 `super()` 调用父类构造，再使用 `this`
- 多层继承形成继承链，可逐层复用和扩展

### 方法重写

子类可以重写（override）父类的方法，即在子类中定义与父类同名的方法，实现自己的行为。使用`super`关键字可以调用父类的方法。

```ts
// 父类
class PrinterClass {
    doPrint(): void {
        console.log("父类的 doPrint() 方法");
    }
}

// 子类：重写父类方法
class StringPrinter extends PrinterClass {
    doPrint(): void {
        // 调用父类的方法
        super.doPrint();

        // 子类自己的逻辑
        console.log("子类的 doPrint() 方法");
    }
}

var obj = new StringPrinter();
obj.doPrint();
```

子类定义与父类同名方法即可重写（override），用 `super.方法名()` 调用父类版本，实现"增强而非替换"。

### 静态成员

使用`static`关键字定义的成员属于类本身，而不是类的实例。可以直接通过类名访问，不需要创建实例。

```ts
class StaticMem {
    // 静态属性
    static num: number;

    // 静态方法
    static disp(): void {
        console.log("num 值为 " + StaticMem.num);
    }
}

// 直接通过类名访问静态成员
StaticMem.num = 12;
StaticMem.disp();
```

`static` 成员属于类本身而非实例，直接通过 `类名.成员` 访问，无需 `new`。常用于工具函数和全局常量。

### instanceof

`instanceof`运算符用于检查一个对象是否是某个类的实例。

```ts
class Person {
}

var obj = new Person();
var isPerson = obj instanceof Person;
// instanceof 运算符返回一个布尔值

console.log("obj 是 Person 类的实例吗？ " + isPerson);
```

`instanceof` 检查对象是否为某个类的实例（含子类实例），返回布尔值。用于运行时类型判断。

### 类实现接口

类可以使用`implements`关键字实现接口，确保类提供接口中声明的所有属性和方法。

```ts
// 定义接口
interface ILoan {
    interest: number;  // 利率
}

// 类实现接口
class AgriLoan implements ILoan {
    interest: number;
    rebate: number;    // 回扣

    constructor(interest: number, rebate: number) {
        this.interest = interest;
        this.rebate = rebate;
    }
}

var loan = new AgriLoan(10, 1);
console.log("利率: " + loan.interest + "%，回扣: " + loan.rebate);
```

`implements` 强制类提供接口声明的所有成员，一个类可实现多个接口，确保"这个类一定能做某事"。

### 鸭子类型

鸭子类型（Duck typing）是动态类型语言的一种风格，它并不关注对象所属的类，而是关注对象所具有的方法和属性。相较于`implements`的显式和强制，鸭子类型是隐式的、自动的。

```ts
interface Duck {
    quack(): void;
    fly(): void;
}

// ─── 用 implements（显式声明） ───
class Mallard implements Duck {     // 显式声明实现了Duck
    //需要在此处实现接口的方法
    quack() { console.log("嘎"); }
    fly()   { console.log("飞"); }
}

// ─── 用鸭子类型（隐式匹配） ───
class Dog {                          // 隐式匹配了Duck
    quack() { console.log("嘎"); }
    fly()   { console.log("飞"); }
}

// 参数类型为 Duck，可以传入 Mallard 或 Dog
function makeDuck(d: Duck) {
    d.quack();
}

makeDuck(new Mallard()); // implements Duck
makeDuck(new Dog());     // Dog 有 quack 和 fly，自动匹配
```

鸭子类型的关键：不要求implement接口，只要求有接口中定义的方法和属性，就认为符合接口。

```ts
interface Duck {
    quack(): void;
    fly(): void;
}

// implements — 定义类时就报错
class Cat implements Duck {
    // 编译错误，缺少 quack() 和 fly()
}

// 鸭子类型 — 传入时才报错
class Fish {
    swim() { console.log("游"); }
}

function makeDuck(d: Duck) { /* ... */ }

makeDuck(new Fish());
// 编译错误，Fish 缺少 quack 和 fly
```

```ts
interface IPoint { 
    x:number 
    y:number 
} 
function addPoints(p1:IPoint,p2:IPoint):IPoint { 
    var x = p1.x + p2.x 
    var y = p1.y + p2.y 
    return {x:x,y:y} 
} 
 
// 正确
var newPoint = addPoints({x:3,y:4},{x:5,y:1})  
 
// 错误，p1缺少y属性
var newPoint2 = addPoints({x:1},{x:4,y:3})
```

鸭子类型是 TypeScript **结构化类型系统**的核心体现 —— 不关注"你是谁"，只关注"你有什么"：

- 只要对象具备接口要求的属性和方法，TypeScript 就认为它符合该接口，**无需显式 `implements`**
- `implements` 在**定义类时**就检查是否满足接口，不满足则立即报错
- 鸭子类型在**使用（传参）时**才检查，更具灵活性和隐式性
- 多余属性不影响匹配（如 `{ x, y, z }` 可传给 `{ x, y }` 类型的参数），缺少必需属性则报错
- 这是 TypeScript 与 Java/C# 等名义类型系统的关键区别之一

### 抽象类

抽象类不能被实例化，只能作为基类供子类继承。抽象类可以包含抽象方法（没有实现的方法）和具象方法（已有实现的方法），子类必须实现这些抽象方法。

```ts
// 使用 abstract 关键字声明抽象类
// 抽象类不能直接实例化，只能作为基类
abstract class Animal {
    // 动物的名字属性
    name: string;

    // 构造函数
    constructor(name: string) {
        this.name = name;
    }

    // 抽象方法：使用 abstract 修饰，没有方法体
    // 子类必须实现这个方法
    abstract speak(): void;

    // 具象方法：有具体实现的方法
    // 子类可以直接继承使用，不需要重写
    move(): void {
        console.log(this.name + " 在移动");
    }
}

// 尝试实例化抽象类会报错
// var animal = new Animal("动物"); // 错误：不能实例化抽象类

// 定义 Dog 类继承 Animal
class Dog extends Animal {
    // 子类必须实现父类的抽象方法 speak()
    speak(): void {
        console.log(this.name + " 汪汪汪!");
    }
}

// 创建 Dog 实例
var dog = new Dog("旺财");
dog.speak();  // 调用子类实现的方法
dog.move();   // 继承父类的具象方法
```

#### 抽象类作为类型

抽象类可以作为参数类型使用。这意味着函数可以接受任何抽象类的子类实例。

```ts
// 定义抽象类 Animal
abstract class Animal {
    // 抽象方法：所有动物都会发出声音
    abstract speak(): void;
}

// 定义 Cat 类继承 Animal
class Cat extends Animal {
    // 实现抽象方法
    speak(): void {
        console.log("喵喵喵!");
    }
}

// 定义 Dog 类继承 Animal
class Dog extends Animal {
    // 实现抽象方法
    speak(): void {
        console.log("汪汪汪!");
    }
}

// 定义函数参数类型为抽象类 Animal
// 这个函数可以接受任何 Animal 的子类实例
function makeSpeak(animal: Animal): void {
    animal.speak();
}

// 传入不同的子类实例，实现不同的行为（多态）
makeSpeak(new Cat());
makeSpeak(new Dog());

// 抽象类类型数组：可以存储不同子类实例
var animals: Animal[] = [new Cat(), new Dog()];
```

#### 抽象类小结

`abstract class` 介于接口和具体类之间：

- **不能实例化**，只能被继承
- **抽象方法**：子类必须实现；**具象方法**：子类直接继承使用
- 可作为参数类型，实现多态（接受任意子类实例）
- 与接口的区别：抽象类可以有实现代码，接口只有声明

### 综合实例

实例1

```ts
// 接口定义
interface IPrintable {
    print(): void;
}

// 抽象类
abstract class Item {
    protected name: string;
    protected price: number;

    constructor(name: string, price: number) {
        this.name = name;
        this.price = price;
    }

    abstract getDetails(): string;
}

// 具体类
class Product extends Item implements IPrintable {
    // 继承抽象类Item并实现接口IPrintable
    private category: string;

    constructor(name: string, price: number, category: string) {
        super(name, price);
        // 调用父类构造函数传入name和price
        this.category = category;
    }

    // 实现抽象方法
    getDetails(): string {
        return `产品: ${this.name}, 价格: ¥${this.price}, 类别: ${this.category}`;
    }

    // 实现接口方法
    print(): void {
        console.log(this.getDetails());
    }

    // 静态方法
    static create(name: string, price: number): Product {
        return new Product(name, price, "默认类别");
    }
}

// 使用
var product = Product.create("笔记本电脑", 5999);
product.print();

// 折扣方法
class DiscountedProduct extends Product {
    private discount: number;

    constructor(name: string, price: number, category: string, discount: number) {
        super(name, price, category);
        // 调用父类构造函数传入name、price和category
        this.discount = discount;
    }

    getDetails(): string {
        //重写getDetails方法
        var discountedPrice = this.price * (1 - this.discount / 100);
        return `产品: ${this.name}, 原价: ¥${this.price}, 折扣: ${this.discount}%, 现价: ¥${discountedPrice.toFixed(2)}`;
        //toFixed(2)保留两位小数
    }
}

var discountedProduct = new DiscountedProduct("手机", 2999, "电子产品", 20);
discountedProduct.print();
//调用父类print方法，输出重写后getDetails方法的结果
```

```ts
// 定义抽象支付类
abstract class Payment {
    // 抽象方法：处理支付，子类必须实现
    abstract process(amount: number): boolean;

    // 具象方法：验证支付信息
    // 所有子类都可以使用这个方法
    validate(): void {
        console.log("验证支付信息");
    }
}

// 信用卡支付类
class CreditCardPayment extends Payment {
    // 信用卡号码
    cardNumber: string;

    // 构造函数
    constructor(cardNumber: string) {
        super();  // 调用父类构造函数
        // 在子类构造函数中，必须先调用 super()，才能使用 this 关键字
        this.cardNumber = cardNumber;
    }

    // 实现抽象方法：处理信用卡支付
    process(amount: number): boolean {
        console.log("处理信用卡支付: " + amount);
        return true;
    }
}

// PayPal 支付类
class PayPalPayment extends Payment {
    // PayPal 邮箱
    email: string;

    // 构造函数
    constructor(email: string) {
        super();
        this.email = email;
    }

    // 实现抽象方法：处理 PayPal 支付
    process(amount: number): boolean {
        console.log("处理 PayPal 支付: " + amount);
        return true;
    }
}

// 使用多态处理不同的支付方式
var payments: Payment[] = [
    new CreditCardPayment("1234"),
    new PayPalPayment("test@example.com")
];

// 遍历处理每种支付方式
for (var _i = 0, payments_1 = payments; _i < payments_1.length; _i++) {
    var payment = payments_1[_i];
    // 调用继承的验证方法
    payment.validate();
    // 调用子类的处理方法
    payment.process(100);
}
```

综合实例演示了接口 + 抽象类 + 具体类的配合使用：抽象类提供公共逻辑（`validate`），具体类实现特定行为（`process`），多态统一处理不同子类。

### 类型模版

```ts
var sites = {

```

对象也可以作为参数传递给函数

```ts
var sites = { 
    site1:"Runoob", 
    site2:"Google",
```

类型模版指在对象上预留函数类型的"空位"（如 `sayHello: function () {}`），后续再赋予具体实现。对象也可直接作为参数，用内联形状约束类型。

### 混入

Mixin 的核心是一个接收基类、返回扩展类的泛型函数。Constructor 类型约束用于描述"任意可被继承的类"，防止传入普通函数或对象。

#### 基本概念

```ts
// Constructor 类型别名
// 描述"任意可以被 new 实例化的类"。
// new  (...args: any[]) => T 表示：
//   - 可以用 new 调用
//   - 接受任意数量和类型的参数（...args: any[]）
//   - 返回类型为 T 的实例
// T = {} 是默认泛型参数，表示"不关心实例具体长什么样"。
type Constructor<T = {}> = new (...args: any[]) => T;

// Mixin 函数：Timestamped
// 泛型参数 <TBase extends Constructor> 确保：
//   传入的 Base 必须是一个"可以被 extends 的类"，防止误传普通函数或对象。
// 返回值是一个匿名类，extends Base 继承了传入类的所有属性和方法，
// 同时新增一个 createdAt 属性，记录对象创建的时间戳。
function Timestamped<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        // 新增属性：每个实例创建时自动获取当前时间
        createdAt = new Date();
    };
}

// 普通的基类
class User {
    // 参数属性简写：public name 既是构造函数参数，又是实例属性
    constructor(public name: string) {}
}

// 使用 Mixin
// Timestamped(User) 将 User 类"喂"给 Mixin 函数，
// 返回一个融合了 User + Timestamped 功能的全新类。
// 注意：TimestampedUser 是一个类，不是实例。
const TimestampedUser = Timestamped(User);

// 用新类创建实例，传入的 "Alice" 最终会传给 User 的构造函数
const user = new TimestampedUser("Alice");

// 访问基类 User 自带的属性
console.log(user.name);                        // Alice

// 访问 Mixin 新增的属性
console.log(user.createdAt instanceof Date);   // true
```

简单例子：

```ts
// Mixin：给任何类添加 "加个尾巴" 的能力
function addTail(AnimalClass) {
    return class extends AnimalClass {
        tail = true;  // 新增尾巴属性
    };
}

// 原始类
class Dog {
    bark() { console.log("汪汪"); }
}

// 混入后得到新类
const DogWithTail = addTail(Dog);
const dog = new DogWithTail();

dog.bark();   // 汪汪（来自原类）
console.log(dog.tail); // true（来自 mixin）
```

#### 组合多个Mixin

```ts
// Constructor 类型：描述任意"可以用 new 实例化"的类
//  - T 是泛型，默认为 {}，表示不限制实例的具体形状
//  - 用于约束 Mixin 的参数必须是类（不能是普通函数或对象）
type Constructor<T = {}> = new (...args: any[]) => T;

// ────── Mixin 1：给类加入"时间戳"能力 ──────
// 功能：为每个实例自动添加 createdAt 属性，记录创建时间
// 约束：只要求 Base 是一个类，不依赖任何特定方法
function Timestamped<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        createdAt = new Date();       // 实例创建时自动获取当前时间
    };
}

// ────── Mixin 2：给类加入"序列化"能力 ──────
// 功能：为实例添加 serialize() 方法，将自身转为 JSON 字符串
// 约束：只要求 Base 是一个类，不依赖任何特定方法
function Serializable<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        serialize(): string {
            return JSON.stringify(this);   // 将当前实例的所有属性序列化为 JSON
        }
    };
}

// ────── Mixin 3：给类加入"日志输出"能力 ──────
// 功能：让实例拥有 log() 方法，以 "[LOG]" 前缀输出序列化结果
// 依赖：要求 Base 必须已有 serialize() 方法（由 Mixin 2 或手动提供）
//       泛型约束 Constructor<{ serialize(): string }> 保证了类型安全
function Loggable<TBase extends Constructor<{ serialize(): string }>>(Base: TBase) {
    return class extends Base {
        log(): void {
            console.log("[LOG]", this.serialize());  // 调用基类的 serialize() 输出日志
        }
    };
}

// ────── 业务基类：产品 ──────
class Product {
    constructor(public name: string, public price: number) {}
}

// ────── 组合：依次叠加三个 Mixin ──────
// 执行顺序（从内到外）：
//   ① Timestamped(Product)               → Product + createdAt
//   ② Serializable(Timestamped(Product)) → Product + createdAt + serialize()
//   ③ Loggable(②)                        → Product + createdAt + serialize() + log()
// 注意：Loggable 必须在 Serializable 之后，因为它依赖 serialize() 方法
const AdvancedProduct = Loggable(Serializable(Timestamped(Product)));

// ────── 使用 ──────
const p = new AdvancedProduct("Phone", 999);
p.log();                                      // "[LOG]" {"name":"Phone","price":999,"createdAt":"..."}
console.log(p.createdAt instanceof Date);     // true — 时间戳来自 Timestamped
```
