# 异步编程

异步编程是JavaScript的核心特性之一，它允许程序在等待某个操作完成时继续执行其他任务，而不是阻塞等待。

## Promise

### Promise构造函数

Promise代表一个异步操作的最终结果，它是一个对象，有三种状态：pending（进行中）、fulfilled（已成功）和rejected（已失败）。

Promise构造函数的参数：

```ts
new Promise((resolve, reject) => { ... })
```

其中 resolve 用于处理操作成功，接收结果值作为参数；reject 用于处理操作失败，接收错误原因作为参数。

```ts
// 构造Promise函数，使用泛型指定解决值的类型
// Promise<string> 表示成功时返回字符串
var promise = new Promise<string>(function(resolve, reject) {
    var success = true;
    if (success) {
        // 调用 resolve 表示操作成功，传入结果值
        resolve("成功!");
    } else {
        // 调用 reject 表示操作失败，传入错误
        reject(new Error("失败"));
    }
});

// 使用 then 处理成功情况
promise
    .then(function(value) {
        console.log("完成: " + value);
    })
    // 使用 catch 处理失败情况
    .catch(function(error) {
        console.log("错误: " + error.message);
    })
    // 使用 finally 处理完成情况（无论成功或失败）
    .finally(function() {
        console.log("结束");
    });

// 输出：
// 完成: 成功!
// 结束
```

### Promise链式调用

then和catch方法返回新的Promise，可以链式调用。

```ts
// 链式调用：每个 then 返回新的值，被下一个 then 接收
var promise = Promise.resolve(1)
    .then(function(n) {
        // 第一个 then，n = 1
        return n * 2;  // 返回 2
    })
    .then(function(n) {
        // 第二个 then，n = 2
        return n + 10;  // 返回 12
    })
    .then(function(n) {
        // 第三个 then，n = 12
        console.log("最终结果: " + n);
        return n;
    });

console.log("Promise 链: " + promise); 
// + 运算符将promise对象强制转换为字符串，调用toString()方法输出[object Promise]

// 输出：
// Promise 链: [object Promise]
// 最终结果: 12
```

### reject/catch

Promise的reject方法表示异步操作失败，catch方法用于处理失败情况。

```ts
function login(username: string): Promise<string> {
    return new Promise((resolve, reject) => {
        console.log("正在登录: " + username);
        if (username === "admin") {
            resolve("登录成功");
        } else {
            reject("用户名错误");
        }
    });
}

login("admin")
    .then((result) => {
        console.log("成功: " + result);
    })
    .catch((error) => {
        console.log("失败: " + error);
    });

login("guest")
    .then((result) => {
        console.log("成功: " + result);
    })
    .catch((error) => {
        console.log("失败: " + error);
    });

// 输出：
// 正在登录: admin
// 正在登录: guest
// 成功: 登录成功
// 失败: 用户名错误
```

### Promise.all

Promise.all方法用于将多个Promise实例包装成一个新的Promise实例，只有当所有实例都成功时，才返回成功结果，否则返回第一个失败的结果。

```ts
// 创建三个 Promise
var p1 = Promise.resolve(1);
var p2 = Promise.resolve(2);
var p3 = Promise.resolve(3);

// Promise.all 等待所有 Promise 完成
// 返回一个数组，包含所有 Promise 的结果
Promise.all([p1, p2, p3]).then(function(results) {
    // result 数组的顺序与传入的 Promise 顺序一致，与完成先后无关
    console.log("全部完成: " + results);
    // 计算总和
    console.log("总和: " + results.reduce(function(a, b) { return a + b; }, 0));
});
// 输出：
// 全部完成: 1,2,3
// 总和: 6
```

如果任何一个 Promise 失败，Promise.all 会立即 reject，不会等待其他 Promise 完成。

### Promise.race

Promise.race方法同样将多个Promise实例包装成一个新的Promise实例，只要有一个实例完成，就返回结果，无论成功或失败。

```ts
// 创建三个不同延迟的 Promise
var p1 = new Promise(function(resolve) {
    setTimeout(function() { resolve("p1"); }, 100);
});
var p2 = new Promise(function(resolve) {
    setTimeout(function() { resolve("p2"); }, 50);
});
var p3 = new Promise(function(resolve) {
    setTimeout(function() { resolve("p3"); }, 30);
});

// Promise.race 返回最先完成的 Promise 的结果
Promise.race([p1, p2, p3]).then(function(value) {
    console.log("最先完成: " + value);
});
// 输出：最先完成: p3
```

### Promise.allSettled

Promise.allSettled 会等待所有 Promise 完成（无论成功或失败），返回每个 Promise 的结果数组。与 Promise.all 不同的是，它不会因为某个 Promise 失败而 reject，始终 resolve。

```ts
// 创建三个 Promise，其中一个会失败
var p1 = Promise.resolve("成功");
var p2 = Promise.reject(new Error("失败"));
var p3 = Promise.resolve("完成");

// Promise.allSettled 等待所有 Promise 结束
// 返回每个 Promise 的状态和值/reason
Promise.allSettled([p1, p2, p3]).then(function(results) {
    results.forEach(function(result, index) {
        if (result.status === "fulfilled") {
            console.log("Promise " + index + ": " + result.value);
        } else {
            console.log("Promise " + index + ": " + result.reason.message);
        }
    });
});
// 输出：
// Promise 0: 成功
// Promise 1: 失败
// Promise 2: 完成
```

### Promise类型注解

TypeScript的泛型支持让Promise的类型声明变得精确。

```ts
// 定义返回 Promise 的函数
// Promise<{ name: string; age: number }> 指定了返回的用户对象类型
function getUser(): Promise<{ name: string; age: number }> {
    return Promise.resolve({ name: "Alice", age: 25 });
}

// async 函数：隐式返回 Promise
async function main() {
    // await 会自动推断 user 的类型
    var user = await getUser();
    // 使用 JSON.stringify 将对象转换为字符串
    console.log("用户: " + JSON.stringify(user));
}

main();

// 输出：用户: {"name":"Alice","age":25}
```

### Promise小结

Promise 是 JavaScript 异步编程的基石，TypeScript 通过泛型增强其类型安全：

- **三种状态**：pending（进行中）→ fulfilled（成功）/ rejected（失败），不可逆
- **核心方法**：`then`（成功回调）、`catch`（失败回调）、`finally`（无论成败都执行）
- **链式调用**：每个 `then`/`catch` 返回新 Promise，值沿链传递
- **组合 API**：`Promise.all`（全成功才成功）、`Promise.race`（竞速）、`Promise.allSettled`（全部完成，不论成败）
- **类型注解**：`Promise<T>` 指定 resolve 值的类型，`async` 函数隐式返回 `Promise`

## async/await

async/await是Promise的语法糖，使异步代码看起来更像是同步代码。

### async函数

使用async关键字声明异步函数。

```ts
// async 函数自动返回 Promise
async function greet() {
    return "Hello, World!";
}

greet().then(function(result) {
    console.log("结果: " + result);
});
// 输出：结果: Hello, World!

// 异步函数返回 Promise
async function getData() {
    return { name: "Alice", age: 25 };
}

getData().then(function(data) {
    console.log("数据: " + JSON.stringify(data));
});
// 输出：数据: {"name":"Alice","age":25}
```

### await关键字

await等待Promise完成并获取结果。

```ts
// async 函数中使用 await
async function getUser() {
    // 等待 fetchUser 函数返回结果
    var user = await fetchUser();
    console.log("用户: " + JSON.stringify(user));
}

// 模拟 fetchUser 函数
function fetchUser() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve({ name: "Alice", age: 25 });
        }, 1000);
    });
}

getUser();
// 输出：用户: {"name":"Alice","age":25}
```

### try/catch

使用try/catch块捕获异步操作中的错误。

```ts
function mayFail(shouldFail: boolean): Promise<string> {
    return new Promise(function(resolve, reject) {
        if (shouldFail) {
            reject(new Error("操作失败"));
        } else {
            resolve("操作成功");
        }
    });
}

async function handleError() {
    try {
        var result = await mayFail(true);
        //如果这里抛出异常，立即跳转到catch
        console.log("结果: " + result);
    } catch (error) {
        // 只有 try 块中发生异常才执行
        // 在严格模式下 error 是 unknown 类型，需要先判断
        if (error instanceof Error) {
            console.log("捕获错误: " + error.message);
        }
    }
}

handleError();
// 输出：捕获错误: 操作失败
```

`try/catch` 是 async/await 的标准错误处理方式：用 `try` 包裹 `await` 调用，`catch` 捕获抛出的异常。与 Promise 的 `.catch()` 等价，但更符合同步代码的阅读习惯。注意：`catch` 中 `error` 的类型是 `unknown`，需用 `instanceof Error` 判断后再访问 `message`。

### async/await小结

`async/await` 是 Promise 的语法糖，让异步代码以同步风格编写：

- **`async` 函数**：自动将返回值包装为 Promise，函数体内可使用 `await`
- **`await`**：暂停执行等待 Promise 完成，直接获取 resolve 的值而非 Promise 对象
- **错误处理**：用 `try/catch` 包裹 `await` 即可捕获异步错误，无需 `.catch()` 链
- **本质**：async/await 只是 Promise 的语法糖，底层仍是 Promise 和事件循环

## 错误处理

### 自定义错误类型

通过扩展 Error 类创建自定义错误类型，可以携带更多错误信息。这使得错误处理更加精确和结构化。

```ts
// 定义应用程序错误类
// 扩展内置 Error 类，添加错误码
class AppError extends Error {
    // 错误码，用于程序化处理错误
    code: string;

    // 构造函数
    constructor(message: string, code: string) {
        super(message);  // 调用父类构造函数
        this.name = "AppError";  // 设置错误名称
        this.code = code;  // 保存错误码
    }
}

// 安全的除法函数
function divide(a: number, b: number): number {
    // 检查除数是否为零
    if (b === 0) {
        // 抛出自定义错误
        throw new AppError("Cannot divide by zero", "DIVIDE_BY_ZERO");
    }
    return a / b;
}

// 使用 try-catch 捕获错误
try {
    var result = divide(10, 0);
} catch (error) {
    // 检查错误类型
    if (error instanceof AppError) {
        console.log("应用错误: " + error.message + ", 代码: " + error.code);
    } else {
        console.log("未知错误: " + error);
    }
}
// 输出：应用错误: Cannot divide by zero, 代码: DIVIDE_BY_ZERO
```

### 自定义Result类型

另一种错误处理方式是使用 Result 类型。它通过返回值携带错误信息，而不是抛出异常。这种方式在函数式编程中很常见。

```ts
// 定义 Result 类型，使用联合类型
// 成功时包含 ok: true 和值，失败时包含 ok: false 和错误
type Result<T, E = Error> =
    | { ok: true; value: T }
    | { ok: false; error: E };
// 等价于：
// type Result<T, E = Error> = { ok: true, value: T } | { ok: false, error: E };


// 使用 Result 类型的除法函数
function safeDivide(a: number, b: number): Result<number, string> {
    // 检查除数是否为零
    if (b === 0) {
        // 返回错误结果
        return { ok: false, error: "Cannot divide by zero" };
    }
    // 返回成功结果
    return { ok: true, value: a / b };
}

// 调用函数并处理结果
var result = safeDivide(10, 2);

// 根据结果类型进行处理
if (result.ok) {
    console.log("结果: " + result.value);
} else {
    console.log("错误: " + result.error);
}
// 输出：结果: 5
```

### Async函数错误处理

在异步函数中，错误处理尤为重要。可以使用 try-catch 或 Result 类型来处理异步操作中的错误。

```ts
// 定义用户接口
interface User {
    id: number;
    name: string;
}

// 模拟获取用户的异步函数
async function fetchUser(id: number): Promise<Result<User, Error>> {
    try {
        // 模拟网络请求
        var response = await fetch("/api/users/" + id);
        var user = await response.json();
        // 返回成功结果
        return { ok: true, value: user };
    } catch (error) {
        // 返回错误结果
        return { ok: false, error: error as Error };
    }
}

// 主函数
async function main() {
    // 调用异步函数
    var result = await fetchUser(1);

    // 处理结果
    if (result.ok) {
        console.log("用户: " + JSON.stringify(result.value));
    } else {
        console.log("错误: " + result.error.message);
    }
}

// 执行主函数
main();
// 输出（成功时）：用户: {"id":1,"name":"Alice"}
// 输出（失败时）：错误: Network Error
```

### 通用错误处理封装

可以创建一个通用的错误处理函数，简化异步代码的错误处理。

```ts
// 通用错误处理包装函数
// 接受一个异步函数，返回 Result 类型
async function withErrorHandling<T>(
    fn: () => Promise<T>
): Promise<Result<T, Error>> {
    try {
        // 执行传入的异步函数
        var data = await fn();
        // 返回成功结果
        return { ok: true, value: data };
    } catch (error) {
        // 返回错误结果
        return { ok: false, error: error as Error };
    }
}

// 使用通用错误处理
// 模拟获取数据
var result = await withErrorHandling(async function() {
    var response = await fetch("/api/data");
    return response.json();
});

// 根据结果处理
if (result.ok) {
    console.log("数据: " + JSON.stringify(result.value));
} else {
    console.error("错误:", result.error);
}
// 输出（成功时）：数据: {"key":"value"}
// 输出（失败时）：错误: Error: ...
```

### 错误处理小结

错误处理是编写健壮 TypeScript 应用的关键环节，主要有两种范式：

- **异常抛出 + try/catch**：通过 `throw` 抛出自定义错误（如 `AppError`），用 `try/catch` 捕获并按类型分发处理。适合不可恢复的致命错误。
- **自定义Result 类型**：通过返回值携带成功/失败信息（`{ ok: true; value }` 或 `{ ok: false; error }`），避免异常抛出的开销。适合可预期的业务错误。
- **异步错误处理**：`async` 函数中既可用 `try/catch` 包裹 `await`，也可返回 `Result` 类型让调用方显式处理。
- **通用封装**：可抽取 `withErrorHandling` 等工具函数，统一将 Promise 异常转换为 `Result` 类型，减少重复的 try/catch 代码。
